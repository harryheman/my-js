## 1. Основы семантики и валидации

### Правила

- Используйте стандартный HTML5 doctype `<!DOCTYPE html>` во избежание причудливых режимов рендеринга.
- Устанавливайте атрибут `lang` на элементе `<html>` для устройств чтения с экрана и перевода.
- Используйте элемент `<meta name="viewport">` с атрибутом `content`, установленным в значение `"width=device-width, initial-scale=1.0"` для обеспечения отзывчивости страницы.
- Используйте один `<h1>` на страницу/отображение для представления главного заголовка. Исключением являются модальные окна, которые также могут содержать один `<h1>`.
- Поддерживайте последовательную иерархию заголовков без пропусков (`<h2>` -> `<h3>`, но не `<h2>` -> `<h4>`).
- Используйте семантические элементы (`<header>`, `<nav>`, `<main>`, `<aside>`, `<footer>`) для создания навигации для вспомогательных технологий.
- Используйте `<search>` для инкапсуляции разметки поиска и фильтрации (заменяет потребность в `role="search"`).
- Используйте `<button>` для запуска операций (JS, модалки, формы), а `<a>`  строго для навигации с помощью URL. Устанавливайте `type="button"` для кнопок в формах, не предназначенных для их отправки (submit).
- Используйте элементы `<ul>`, `<ol>` и `<dl>` для списков.
- Убедитесь, что все интерактивные элементы, вроде ссылок и кнопок, имеют доступные имена.
- Скрывайте чисто декоративные изображения SVG для вспомогательных технологий с помощью `aria-hidden="true"`. В случае декоративного `<img>`, всегда включайте пустой атрибут `alt` (например, `alt=""`).
- Убедитесь, что информативные SVG, вроде логотипов, визуализации данных или кнопок-иконок имеют доступные имена.

- Не используйте общие `<div>` или `<span>` при наличии подходящего семантического элемента, например, для интерактивных элементов, заголовков или независимого многократно используемого автономного контента.
- Не используйте логические атрибуты с лишними значениями (например, используйте `disabled`, а не `disabled="disabled"`).
- Не используйте общие элементы с добавлением ролей или состояний ARIA при наличии нативных элементов со встроенной семантикой и поведением.
- Не изменяйте нативную семантику элементов с помощью ARIA без крайней необходимости.
- Не используйте `role="presentation"` или `aria-hidden="true"` на фокусируемых элементах или их предках/потомках.
- Не отключайте возможности масштабирования страницы.

### Пример кода

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Панель управления | Платформа</title>
</head>
<body>
  <header>
    <nav>
      <ul>
        <li><a href="#">Главная</a></li>
        <li><a href="#">Контакты</a></li>
      </ul>
    </nav>
  </header>
  <main>
     <h1>Аналитика</h1>
    <search>
      <form action="/filter" method="GET">
        <label for="search-input">Поиск элементов:</label>
        <input type="search" id="search-input" name="q">
        <button type="submit">Поиск</button>
      </form>
    </search>
    <article>
      <h2>Первый пост</h2>
    </article>
  </main>
</body>
</html>
```

## 2. Группировка и атрибуция контента

### Правила

- Используйте `<blockquote>` для расширенных цитат из другого источника и атрибут `cite` для предоставления машиночитаемого URL этого ресурса.
- Используйте `<figure>` для группировки независимого контента (изображения, блоки кода или цитаты), на который имеется ссылка в основном потоке, но который можно переместить в приложение или на боковую панель, не влияя на смысл документа.
- Используйте `<figcaption>` в качестве первого или последнего потомка `<figure>` для предоставления человекочитаемой подписи или атрибуции.
- Используйте элемент `<cite>` внутри подписи или атрибуции в качестве названия работы (например, названия книги или сайта), но не имени автора.
- Используйте элемент `<code>` для коротких фрагментов компьютерного кода (например, имен переменных, путей файлов или встроенных блоков).
- Оборачивайте `<code>` в элемент `<pre>` при отображении блоков кода для сохранения отступов и переносов строк.
- Убедитесь, что блоки кода являются доступными путем добавления `tabindex="0"` к элементу `<pre>`, чтобы он был прокручиваемым, позволяя взаимодействовать с контентом с помощью клавиатуры.

- Не используйте `<blockquote>` для чисто визуального отделения одного текста от другого.
- Не используйте `<figure>` для каждого изображения; используйте его, только когда требуется подпись, или когда контент представляет собой отдельную единицу, на которую есть ссылка.
- Не используйте `<pre>` без `<code>` для блоков кода; `<pre>` позволяет сохранить форматирование, но не сообщает, что контент является компьютерным кодом.

### Пример кода

```html
<!-- Цитата с атрибуцией -->
<figure>
  <blockquote cite="https://html.spec.whatwg.org/">
    <p>The figure element represents some flow content, optionally with a caption, that is self-contained and is typically referenced as a single unit from the main flow of the document.</p>
  </blockquote>
  <figcaption>
    Определение элемента &lt;figure&gt; из <cite>HTML Living Standard</cite>
  </figcaption>
</figure>

<!-- Изображение с подписью -->
<figure>
  <img
    src="architecture-diagram.webp"
    alt="Диаграмма взаимодействия клиентского приложения, шлюза API и микросервисов"
    width="800"
    height="450"
    loading="lazy"
  >
  <figcaption>Рисунок 1: высокоуровневый обзор системной архитектуры.</figcaption>
</figure>

<!-- Блок кода с подсказками, связанными с доступностью и языком -->
<figure>
  <figcaption>Пример настройки:</figcaption>
  <pre tabindex="0"><code class="language-json">
{
  "name": "gemini-cli",
  "version": "1.0.0",
  "private": true
}
  </code></pre>
</figure>

<!-- Встроенный код -->
<p>Для инициализации проекта выполните команду <code>npm install</code>.</p>
```

## 3. Приоритизация ресурсов и производительность

### Правила

- Используйте `fetchpriority="high"` для элемента отрисовки наибольшего контента (LCP) (например, основного изображения) для повышения его сетевого приоритета.
- Используйте `<link rel="preload" as="image">` с `fetchpriority="high"` для фоновых изображений LCP, определенных в CSS.
- Применяйте `loading="lazy"` к изображениям и `iframe`, находящимися за пределами экрана, для экономии пропускной способности.
- Определяйте `width` и `height` для всех тегов `<img>` для сохранения соотношения сторон и предотвращения сдвигов макета (CLS).
- Используйте атрибут `srcset` на `<img>` для определения нескольких версий изображения для разных размеров экрана.
- Используйте элемент `<picture>` с резервным `<img>` для более точного управления изображением, например, переключения между разными форматами, размерами изображения или вырезкой из него фрагментов для экранов разных размеров.

- Не применяйте `loading="lazy"` к видимым при загрузке страницы или основным изображениям. Это ухудшает LCP.
- Не злоупотребляйте `fetchpriority="high"`; приоритизация - это [механизм с нулевой суммой](https://ru.wikipedia.org/wiki/%D0%90%D0%BD%D1%82%D0%B0%D0%B3%D0%BE%D0%BD%D0%B8%D1%81%D1%82%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B0%D1%8F_%D0%B8%D0%B3%D1%80%D0%B0). Используйте `fetchpriority="low"` для понижения приоритета загрузки некритичных трекеров или элементов карусели.

### Пример кода

```html
<!-- Высокоприоритетное основное изображение с разными размерами -->
<img
  src="hero-large.webp"
  srcset="hero-small.webp 480w, hero-medium.webp 800w, hero-large.webp 1200w"
  sizes="(max-width: 600px) 100vw, (max-width: 1200px) 80vw, 70vw"
  alt="Основной вид продукта"
  fetchpriority="high"
  width="1200"
  height="600"
>

<!-- Переключение размеров и форматов изображения с помощью <picture> -->
<picture>
  <!-- Телефон: разное соотношение сторон (квадрат) и формат (AVIF) -->
  <source
    media="(max-width: 600px)"
    srcset="hero-mobile.avif 1x, hero-mobile-2x.avif 2x"
    type="image/avif"
    width="600"
    height="600"
  >
  <source
    media="(max-width: 600px)"
    srcset="hero-mobile.webp 1x, hero-mobile-2x.webp 2x"
    width="600"
    height="600"
  >

  <!-- Десктоп: современный формат для основного макета -->
  <source srcset="hero-desktop.avif" type="image/avif">

  <!-- Резервное изображение определяет дефолтное соотношение сторон (2:1) -->
  <img
    src="hero-desktop.webp"
    alt="Обзор панели управления платформы"
    width="1200"
    height="600"
    loading="lazy"
  >
</picture>

<!-- Низкоприоритетное декоративное изображение в подвале -->
<img
  src="footer-art.png"
  alt=""
  loading="lazy"
  width="200"
  height="100"
>
```

## 4. Нативные оверлеи: диалоги и поповеры

### Правила

- Используйте `<dialog>` для модальных оверлеев (метод JS `.showModal()`) для автоматического захвата фокуса, затенения фона и поддержки отмены (dismiss) с помощью `Esc`. Используйте атрибут `closedby="any"` для включения нативной "легкой отмены" (закрытие при клике по фону) без кастомного JavaScript.
- Используйте Popover API (атрибут `popover`) для немодального UI (меню, всплывающие подсказки), когда не требуется захват фокуса.
- Используйте `::backdrop` для стилизации фона модалки.
- Используйте `<form method="dialog">` для закрытия диалога без обработчика JS. Кнопка c `formmethod="dialog"` записывает значение кнопки в `.returnValue` диалога.

- Не используйте `show()` для модалок, когда ожидается захват фокуса (используйте `showModal()`).
- Не вызывайте `showModal()` у элементов с атрибутом `popover` (это взаимоисключающие программные состояния). Однако, `<dialog popover="auto">` - это валидная декларативная архитектура для соединения семантики диалога с механиками легкой отмены.

### Пример кода

```html
<!-- Поповер (для переключения видимости не нужен JS) -->
<button popovertarget="help-menu">Информация</button>
<div id="help-menu" popover="auto">
  <p>Обычный вспомогательный текст.</p>
</div>

<!-- Модальный диалог с закрытием через форму -->
<button id="show-dialog">Открыть диалог</button>
<dialog id="fav-modal">
  <!-- method="dialog" закрывает диалог нативно и устанавливает returnValue -->
  <form method="dialog">
    <p>Подтвердить операцию?</p>
    <button value="cancel">Отмена</button>
    <button value="confirm">Подтверждение</button>
  </form>
</dialog>

<script>
  const dialog = document.getElementById("fav-modal");
  const openModal = document.getElementById("show-dialog");

  // Отображаем модальный диалог
  openModal.addEventListener('click', () => dialog.showModal());

  // Обрабатываем событие 'close' для извлечения выбора пользователя (returnValue)
  dialog.addEventListener('close', () => {
    console.log(dialog.returnValue); // "confirm" или "cancel"
  });
</script>
```

### Нативный UI оверлеев

| Возможность | Модальность             | Фокус                           | Механизм отмены                   | Случаи использования                     |
|:------------|:------------------------|:--------------------------------|:----------------------------------|:-----------------------------------------|
| `<dialog>`  | Модальный / Немодальный | Автоматический захват (модалка) | Esc / форма / `closedby`          | Критичные операции, настройки            |
| `[popover]` | Немодальный             | Стандартный поток фокуса        | Легкая отмена (клик за пределами) | Меню, всплывающие подсказки, уведомления |
| `<details>` | Встроенное раскрытие    | Стандартный поток фокуса        | Клик по `summary`                 | Аккордеоны, частые вопросы               |

Эвристическое правило: используйте `<dialog>` для прерываний, требующих пользовательского действия, `popover` для незначительной информации и `<details>` для раскрытия контента, отображаемого прямо на странице.

## 5. Раскрытия: `details` и `summary`

### Правила

- Используйте `<details>` и `<summary>` для нативных аккордеонов или раскрываемого контента без JS.
- `<summary>` должен быть первым потомком `<details>`.
- Если внутри `<summary>` должен содержаться заголовок, убедитесь, что он является существенным для понимания или навигации по структуре документа. Если это так, используйте более надежный подход к раскрытию информации, который позволяет обернуть триггер раскрытия заголовком (например, `<h2><button type="button" aria-expanded="false" aria-controls="significant-section-content">Важный раздел</button></h2>`). Это обеспечит сохранение семантики заголовка, а также анонсирование кнопки и ее состояния.
- Используйте атрибут `details[open]` для стилизации раскрытого состояния.
- Используйте `details::details-content` для стилизации контента элемента `<details>`.
- Используйте атрибут `name` с несколькими элементами `<details>` для создания эксклюзивных аккордеонов (одновременно может быть открыт только один `<details>`).

- Не вкладывайте другие интерактивные элементы (ссылки, кнопки) прямо в текст `<summary>`, поскольку он действует как кнопка и ломает фокус.
- Не скрывайте видимые треугольники с помощью `list-style: none` без предоставления явных указателей направления (через псевдоэлементы `::before`/`::after`).
- Не используйте атрибут `title` для создания эффектов всплывающей подсказки.

### Пример кода

```html
<!-- Эксклюзивный аккордеон -->
<details name="faq">
  <summary>Элемент 1</summary>
  <p>Контент...</p>
</details>
<details name="faq">
  <summary>Элемент 2</summary>
  <p>Контент...</p>
</details>
```

## 6. Границы фокуса и видимость

### Правила

- Используйте глобальный атрибут `inert` для больших скрытых блоков (меню за пределами экрана, фона при открытой модалке) для их удаления из потока фокуса и дерева доступности.
- Используйте `[inert]` вместе с CSS (`opacity: 0.5`) для визуального представления инертности.
- Полагайтесь на порядок элементов в DOM для последовательной навигации.

- Не используйте положительные значения `tabindex` (например, `1`, `2`). Используйте `0` для добавления элемента в поток фокуса или `-1` для программного фокуса с помощью JS.
- Не меняйте поток фокуса с помощью свойств CSS (`flex-flow: row-reverse`, `order`) без синхронизации структуры DOM.
- Не используйте `node.focus({ preventScroll: true })` без проверки удобства использования; это может скрыть элемент в фокусе за экраном.

### Пример кода

```html
<!-- Отключение основного контента при открытой боковой панели -->
<main id="app-shell" inert>
  <a href="/">Панель управления</a>
</main>
<aside id="drawer">
  <button>Закрыть</button>
</aside>
```

```css
[inert], [inert] * {
  opacity: 0.5;
  cursor: default;
  user-select: none;
}
```

## 7. HTML API и группировка форм

### Правила

- Используйте атрибут `form="form-id"` для отделения инпутов от физического дерева `<form>`.
- Используйте `<datalist>` с `<input list="id">` для реализации легковесного функционала предложений автозаполнения (обратите внимание: визуально не стилизуется и имеет проблемы с устройствами чтения с экрана).
- Не используйте атрибут `autocomplete="off"` в полях учетных данных, адреса, платежных или контактных данных. Браузеры и менеджеры паролей намеренно игнорируют его. Используйте конкретный токен (`autocomplete="email"`, `"street-address"`, `"cc-number"` и т.д.).
- Не используйте `autocomplete="off"`, пока не потребуется обработка высокочувствительных токенов отслеживания (нарушает стандартные переопределения менеджера паролей). Используйте стандартные инпуты `type="email"`, `type="tel"`.
- Разделяйте `autocomplete="current-password"` (авторизация) и `autocomplete="new-password"` (регистрация / изменение пароля), чтобы менеджеры паролей предлагали правильные вещи.
- Используйте токены `autocomplete` с правильными `inputmode` и `type` (`type="email"` + `inputmode="email"` + `autocomplete="email"`). Они контролируют разные вещи - клавиатуру, валидацию и автозаполнение, соответственно - и усиливают друг друга.

### Пример кода

```html
<form>
  <fieldset>
    <legend>Адрес</legend>
    <label for="city">Город:</label>
    <input type="text" id="city" list="cities" autocomplete="address-level2">
    <datalist id="cities">
      <option value="New York">
      <option value="London">
    </datalist>
  </fieldset>
</form>
```

## 8. Нативные медиа-элементы

### Правила

- Устанавливайте `width` и `height` на элементах `<video>` для предотвращения сдвигов макета (CLS).
- Предоставляйте резервное изображение `poster` для видео.
- Включайте субтитры и подписи с помощью `<track>`.
- Убедитесь, что фоновые видео имеют атрибут `muted` (заглушены), предоставляйте пользователям возможность остановить воспроизведение и используйте `role="none"` или `aria-hidden="true"`. Атрибут `controls` также должен быть опущен, чтобы сделать видео нефокусируемым.

- Не используйте JS для основного управления видео, если нативного атрибута `controls` достаточно.
- Не применяйте `role="none"` или `aria-hidden="true"` к фокусируемым элементам (таким как встраиваемые интерактивные компоненты `<iframe>`). Скрытие элементов от вспомогательных технологий при сохранении их доступности для навигации с помощью клавиатуры нарушает ключевые эвристики доступности. Фоновые видео являются исключением, поскольку отсутствие атрибута `controls` рендерит элемент `<video>` как полностью нефокусируемый.

### Пример кода

```html
<video
  controls
  width="800"
  height="450"
  poster="poster.webp"
>
  <source src="intro.webm" type="video/webm">
  <source src="intro.mp4" type="video/mp4">
  <track src="caps.vtt" kind="captions" srclang="en" label="English">
</video>
```

## 9. Динамические стили и интерактивность

### Правила

- Используйте атрибут `style` для передачи состояния в CSS через кастомные свойства (переменные CSS). Это обеспечивает нахождение визуальной логики в таблице стилей, в то время как JavaScript предоставляет сырые (raw) данные.

- Не используйте встроенные стили для статического дизайна (цвета, внутренние и внешние отступы) - они должны находиться в таблице стилей.
- Не используйте встроенные обработчики событий (например, `onclick`). Запускайте операции с помощью `addEventListener()`.

### Пример кода

```html
<body>
  <!-- Прогресс с цветовыми данными на основе style -->
  <label for="upload-progress">Статус загрузки:</label>
  <progress id="upload-progress" class="loading-bar" value="0" max="100" style="--brand-hue: 200;"></progress>

  <script>
    const updateProgress = (percent, hue) => {
      const bar = document.querySelector('.loading-bar');
      bar.value = percent;

      // Обновляем динамическую переменную CSS
      if (hue) bar.style.setProperty('--brand-hue', hue);
    };

    // Пример: двигаем прогресс на 85% и меняем цвет на зеленый (120)
    setTimeout(() => updateProgress(85, 120), 1000);
  </script>
</body>
```

```css
.loading-bar {
  accent-color: hsl(var(--brand-hue, 200) 80% 50%);
  transition: accent-color 0.3s ease;
}
```
