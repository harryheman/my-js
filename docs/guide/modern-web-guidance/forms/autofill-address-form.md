# Форма адреса, соответствующая лучшим практикам

Создайте форму, которая максимально упростит пользователям ввод адресных данных на настольных компьютерах и мобильных телефонах. Убедитесь, что форма максимально использует встроенные возможности браузера по автозаполнению, валидации и учету ограничений ввода данных.

## Реализация

Ниже перечислены наиболее важные рекомендации по созданию хороших форм для ввода адреса.

### Используйте осмысленный валидный HTML

Получайте максимальную выгоду от встроенных элементов и атрибутов форм:

-   `<form>`, `<input>`, `<label>` и `<button>`
-   `type`, `autocomplete` и `inputmode`

Это включает встроенный браузерный функционал, улучшает доступность и добавляет осмысленности в разметку.

### Используйте элемент `<label>` для добавления подписей к полям формы

Для добавления подписей к `<input>`, `<select>` или `<textarea>` используйте `<label>`. Связывайте подпись с инпутом путем добавления атрибуту `for` подписи и атрибуту `id` инпута одинакового значения.

### Получайте максимальную выгоду от атрибутов HTML

Облегчите пользователям ввод данных путем использования подходящего атрибута `<type>` элемента `<input>` для предоставления правильной клавиатуры на мобильных телефонах и включения базовой встроенной браузерной валидации.

Всегда используйте `type="email"` для адресов электронной почты и `type="tel"` для номеров телефона:

```html
<!-- type="email"/"tel" дает пользователям правильную мобильную клавиатуру и включает встроенную валидацию -->
<input type="email" id="email" name="email" autocomplete="email" required>
<input type="tel" id="phone" name="phone" autocomplete="tel">
```

Каждый элемент `<input>`, `<select>` и `<textarea>` должен содержать подходящий атрибут `autocomplete` для улучшения доступности и помощи пользователям в повторном вводе данных.

### Сделайте кнопки полезными

Используйте `<button>` для кнопок. Вы также можете использовать `<input type="submit">`, но не используйте `div` или другой элемент в качестве кнопки. Элементы `<button>` предоставляют доступное поведение, встроенный функционал отправки форм и легко стилизуются.

Предоставляйте каждой кнопке для отправки формы понятное название. Для каждого шага оформления заказа используйте описательный призыв к действию, который показывает прогресс и делает следующий шаг очевидным. Например, кнопку отправки в форме адреса доставки следует назвать "Перейти к оплате", а не "Продолжить" или "Сохранить".

### По возможности используйте одно поле для ввода имени

Позвольте пользователям вводить свое имя с помощью одного поля, если у вас нет веской причины хранить имена, фамилии, почетные звания или другие части имени отдельно. Использование одного поля ввода имени упрощает заполнение форм, позволяет копировать и вставлять данные, а также упрощает автозаполнение.

Разрешите использование международных имен. Для валидации избегайте использования регулярных выражений, которые соответствуют только латинским символам. Использование только латинских символов исключает пользователей, чьи имена или адреса содержат символы, не входящие в латинский алфавит. Вместо этого разрешите сопоставление с символами юникода — и убедитесь, что ваш бэкэнд поддерживает юникод как на входе, так и на выходе. Юникод в регулярных выражениях хорошо поддерживается современными браузерами.

### Предусмотрите возможность использования различных форматов адресов

При создании формы адреса учитывайте разнообразие форматов адресов, даже в пределах одной страны. Не делайте предположений о "стандартных" адресах.

Используйте один элемент `<textarea>` для названия улицы, если это возможно:

```html
<!-- textarea обрабатывает многострочные международные форматы адресов, чего разделенные инпуты не могут достичь -->
<textarea id="address" name="address" autocomplete="street-address" required></textarea>
```

Это наиболее гибкий вариант, подходящий для различных форматов местных и международных адресов.

### Помогите предотвратить случайное отсутствие данных в полях

Добавляйте атрибут `required` к обязательным полям:

```html
<input type="text" id="city" name="city" autocomplete="address-level2" required>
```

### Поддержка и страховка

`:autofill` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 110+, Edge 110+ (с февраля 2023 года) и Safari 15+ (с сентября 2021 года). Не поддерживается в Firefox.

Автозаполнение - это прогрессивное улучшение. В браузерах без поддержки автозаполнения пользователям просто нужно будет вводить данные вручную. Семантические ограничения HTML (такие как `type`, `inputmode` и `required`) по-прежнему будут корректно функционировать для валидации пользовательского ввода и предоставления правильной виртуальной клавиатуры.
