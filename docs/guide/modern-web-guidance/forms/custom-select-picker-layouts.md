# Кастомные макеты выпадающих список

"Кастомные макеты выпадающих список" (Custom Select Picker Layouts) позволяют разработчикам не довольствоваться традиционными вертикальными списками вариантов для выбора в выпадающем слое `<select>`. Свойство `appearance: base-select` и псевдоэлемент `::picker(select)` позволяют стилизовать список вариантов с помощью современных техник компоновки CSS, таких как Grid или Flexbox. Это отлично подходит для колорпикеров, селекторов эмодзи или вариантов товара, когда визуальное меню эффективнее, чем список.

Свойство `appearance: base-select` разблокирует возможность стилизации внутренних частей элемента `<select>`. С помощью `select::picker(select)` можно применить `display: grid` и расположить варианты в колонках, создавая богатый визуальный опыт без кастомного JavaScript.

## Реализация

Для реализации кастомного макета выпадающего списка:

1. Активируйте базовую стилизацию: примените `appearance: base-select` к элементу `<select>` и его внутреннему псевдоэлементу пикера `select::picker(select)`.
2. Стилизуйте контейнер пикера: в `select::picker(select)` примените `display: grid` (или `display: flex`). Определите колонки и отступы по аналогии с любым другим контейнером.
3. Стилизуйте варианты: стилизуйте элементы `<option>` как элементы сетки или карточки. В них можно добавлять изображения, SVG или сложные макеты.
4. Кастомизируйте триггер (опционально): используйте `<selectedcontent>` внутри `<button>` для рендеринга богатого контента для выбранного значения.

## Пример кода: кастомный пикер в виде сетки

```html
<label for="weather-picker">Выберите погоду</label>
<select class="grid-picker" name="weather" id="weather-picker">
  <button>
    <selectedcontent></selectedcontent>
  </button>
  <option value="sunny">
    <span class="icon">☀️</span>
    <span class="label">Солнечно</span>
  </option>
  <option value="cloudy">
    <span class="icon">☁️</span>
    <span class="label">Облачно</span>
  </option>
  <!-- Другие варианты... -->
</select>
```

```css
/* Активируем кастомизируемое состояние селекта */
.grid-picker,
.grid-picker::picker(select) {
  appearance: base-select;
}

/* Стилизуем выпадающий список как сетку */
.grid-picker::picker(select) {
  display: grid;
  grid-template-columns: repeat(2, 1fr); /* 2 колонки */
  gap: 10px;
  padding: 15px;
  background: white;
  border: 1px solid #ccc;
  border-radius: 8px;
}

/* Стилизуем варианты как карточки сетки */
.grid-picker option {
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  padding: 15px;
  border: 1px solid #eee;
  border-radius: 6px;
}

/* Используем несколько визуальных индикаторов (утолщение границы, изменение цвета фона и веса шрифта) для выбранного состояния */
.grid-picker option:checked {
  border: 2px solid #007bff;
  background-color: #f0f7ff;
  font-weight: 700;
}
```

## Стратегическая реализация и лучшие практики

- Используйте `appearance: base-select`, когда требуется изменить визуальный макет вариантов для выбора от стандартного списка к двумерной сетке или кастомному гибкому (flex) потоку.
- Не рассчитывайте на то, что стили будут одинаково применяться ко всем браузерам; проверяйте поддержку и предоставляйте резерв.
- Не забывайте о доступности: нативный элемент `<select>` поддерживает только одномерную линейную навигацию с помощью стрелок (вверх/вниз). Размещение вариантов в виде двумерной сетки приводит к несоответствию пространственной логики: нажатие стрелки "Влево" или "Вправо" не перемещает фокус горизонтально между соседними колонками. Если для удобства использования необходима полноценная двумерная навигация с помощью клавиатуры, не используйте стандартный элемент `<select>`; вместо этого, реализуйте собственный составной виджет с ARIA-атрибутом `role="listbox"`, обеспечив навигацию стрелками по матрице элементов с помощью JavaScript.
- Используйте `<selectedcontent>` для автоматического отображения в триггере изображения или иконки выбранного варианта.
- Используйте стандартный атрибут `value` в вариантах для того, чтобы отправка формы работала ожидаемо.
- Убедитесь, что `<select>` содержит атрибут `name` и связанный `<label>`. Это гарантирует, что даже с кастомным UI, компонент остается доступным для устройств чтения с экрана и работает правильно при стандартной отправке формы.

## Поддержка и страховка

Кастомизируемый `<select>` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 135+ и Edge 135+ (с апреля 2025 года). Не поддерживается в Firefox и Safari.

В браузерах без поддержки кастомный элемент `<select>` мягко деградирует к стандартному выпадающему слою операционной системы.

- Нетекстовый контент игнорируется: старые браузеры удаляют теги HTML (такие как `<svg>` или `<div>`) внутри тегов `<option>` и рендерят только текстовые узлы. Убедитесь, что текст `<option>` является читаемым и осмысленным сам по себе.
- Обработка структуры HTML: стандартные парсеры могут игнорировать теги `<button>` и `<selectedcontent>` внутри `<select>` или считать их невалидными. При отображении читаемого текста в качестве резерва для прогрессивного улучшения строго не требуются тяжелые полифиллы JavaScript.

```javascript
document.addEventListener("DOMContentLoaded", () => {
  // Проверяем поддержку браузером значения `base-select`
  if (!CSS.supports("appearance", "base-select")) {
    // Кастомизация селекта не поддерживается нативно
  }
});
```
