# Автоматически расширяющееся поле формы

По умолчанию контролы формы, такие как `<input>`, `<textarea>` и `<select>` имеют фиксированные размеры. Это означает, что их размеры остаются постоянными, независимо от контента, введенного или выбранного пользователем.

Для того, чтобы контролы растягивались по ширине/высоте их контента (включая заполнители (placeholders)), используйте CSS-свойство `field-sizing: content`.

## Автомасштабирование контролов формы

Установка `field-sizing: content` на инпутах, селектах или текстовых областях позволяет им динамически масштабироваться по мере ввода или выбора пользователя. Однако, следует учитывать наследуемые стили, дефолтные значения компоновки и ограничения минимума/максимума для обеспечения хорошего пользовательского опыта.

Для предотвращения проблем с макетом рекомендуется устанавливать `min-inline-size` (или `min-width`) и `max-inline-size` (или `max-width`) вместе с `field-sizing: content` на текстовых инпутах. Минимальная ширина предотвращает схлопывание пустого инпута (что делает его некликабельным), а максимальная ширина гарантирует, что инпут не будет растягиваться бесконечно и не сломает макет страницы.

Для текстовых областей включение автоматического изменения размера по горизонтали может привести к неприятному UI (например, текстовая область с длинным заполнителем будет резко сжиматься по горизонтали, когда пользователь введет первый символ). Для предотвращения этого устанавливайте явную ширину (например, `width: 100%`) текстовой области. Это заставляет текстовую область поддерживать стабильную ширину по горизонтали, сохраняя при этом автоматическое изменение размера по вертикали при переносе содержимого.

Рекомендуется устанавливать `max-inline-size` (или `max-width`) на элементах `<select>` вместе с `field-sizing: content` для того, чтобы гарантировать, что очень длинные выбранные варианты не сломают макет страницы.

```css
/* Применяем автоматическое изменение ширины к инпутам */
input {
  /* Указывает элементу изменять размер в соответствии с содержимым */
  field-sizing: content;

  /* Сбрасываем неявную ширину, унаследованную от глобальных стилей */
  width: auto;

  /* Предотвращаем схлопывание и исчезновение пустого инпута */
  min-inline-size: 15ch;

  /* Предотвращаем бесконечное растягивание */
  max-inline-size: 50ch;
}

textarea {
  /* Указывает элементу изменять размер в соответствии с содержимым */
  field-sizing: content;

  /* Сбрасываем неявную высоту, унаследованную от глобальных стилей */
  height: auto;

  /* Используем фиксированную ширину для предотвращения сдвигов по горизонтали при замене длинного заполнителя */
  width: 50ch; /* или 100% в зависимости от вашего макета */

  /* Устанавливаем разумную минимальную высоту (например, 3 строки) для пустой текстовой области */
  min-block-size: 3lh;

  /* Предотвращаем бесконечное растягивание. При достижении этой высоты */
  /* текстовая область перестает растягиваться и появляется вертикальная полоса прокрутки */
  max-block-size: 10lh;
}

select {
  /* Ширина селекта определяется шириной выбранного варианта */
  field-sizing: content;

  /* Предотвращаем бесконечное растягивание во избежание поломки макета */
  max-inline-size: 50ch;
}
```

Важно: явные свойства `width` и `height` перезаписывают `field-sizing: content` всех контролов формы. Если глобальный CSS вашего проекта устанавливает `width: 100%` инпутам, селектам или текстовым областям, необходимо явно сбросить их с помощью `width: auto` (или `width: fit-content`) для элементов с `field-sizing: content` для работы автомасштабирования. Аналогично, необходимо явно устанавливать `height: auto` на текстовых областях, если глобально установлена фиксированная высота.

Важно: компоновки Grid и Flexbox часто неявно растягивают потомков для заполнения доступного пространства. Если контрол формы с `field-sizing: content` "отказывается" сжиматься, проверьте свойства выравнивания его контейнера и примените `align-self: start` или `justify-self: start` к контролу для перезаписи растягивания.

## Поддержка и страховка

`field-sizing` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 123+, Edge 123+ (с марта 2024 года) и Safari 26.2+ (с декабря 2025 года). Не поддерживается в Firefox.

Таким образом, `field-sizing` - это прогрессивное улучшение. В браузерах без поддержки контролы формы мягко деградируют к фиксированным размерам. Пользователи просто видят стандартную прокрутку переполненного контента внутри полей ввода и текстовых областей фиксированного размера.

Если динамическое масштабирование полей является критически важным для старых браузеров, следует использовать определение поддержки и сложные обходные пути. Самый надежный запасной вариант для текстовых областей - трюк CSS Grid, где скрытый псевдоэлемент отражает ввод пользователя в реальном времени, заставляя контейнер сетки (и текстовую область внутри него) расширяться.

```html
<!-- Текстовая область оборачивается в контейнер, который будет отражать ее значение -->
<div class="growable-textarea" data-replicated-value="">
  <textarea></textarea>
</div>
```

```javascript
// Привязываем резервные обработчики событий только при отсутствии поддержки field-sizing
if (!CSS.supports('field-sizing', 'content')) {
  document.querySelectorAll('.growable-textarea > textarea').forEach(textarea => {
    textarea.addEventListener('input', () => {
      textarea.parentNode.dataset.replicatedValue = textarea.value;
    });
  });
}
```

```css
/* Применяем резерв только при отсутствии поддержки field-sizing */
@supports not (field-sizing: content) {
  .growable-textarea {
    display: grid;
  }

  /* Псевдоэлемент и текстовая область должны иметь одинаковые ячейку (cell), шрифт и паддинг */
  .growable-textarea::after,
  .growable-textarea > textarea {
    grid-area: 1 / 1 / 2 / 2;
    font: inherit;
    padding: 0.5rem;
    border: 1px solid #999;
  }

  /* Псевдоэлемент рендерит невидимый текст для растягивания сетки */
  .growable-textarea::after {
    /* Это нужно для рендеринга завершающего пробела */
    content: attr(data-replicated-value) " ";
    white-space: pre-wrap;
    visibility: hidden;
  }

  .growable-textarea > textarea {
    resize: none;
    overflow: hidden;
  }
}
```

Учитывая сложность дублирования стилей и синхронизации состояния между узлами DOM для каждого контрола формы, использование дефолтного резерва (фиксированных размеров) является рекомендуемым подходом для большинства приложений, если только динамическое масштабирование не является критичным для UX.
