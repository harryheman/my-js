# Руководство по работе с формами

## 1. Семантическая структура и элемент формы

### Рекомендации

- Используйте элемент `<form>` для инкапсуляции интерактивных контролов для сбора данных.
- Используйте `method="POST"` для чувствительных данных и мутаций; используйте `method="GET"` для идемпотентных запросов (например, поиск).
- Определяйте атрибут `action` для целевого URL.
- Определяйте атрибут `name` для каждого контрола для определения данных при отправке формы.
- Используйте семантические теги, такие как `<button type="submit">`, `<textarea>` и `<select>`.
- Используйте `<fieldset>` и `<legend>` для группировки связанных контролов.
- Используйте описательные названия кнопок отправки формы (например, "Сохранить изменения").

- Не используйте `GET` для чувствительных данных (данные будут отображаться в истории/логах).
- Не используйте общие `<div>` или `<span>` для контролов формы.
- Не используйте `type="button"` для кнопок отправки формы.
- Не отключайте ресайз `textarea` без предоставления альтернативы.

### Пример кода

```html
<form action="/search" method="GET">
  <fieldset>
    <legend>Настройки поиска</legend>
    <label for="q">Строка запроса:</label>
    <input type="text" id="q" name="q" required>
    <button type="submit">Поиск</button>
  </fieldset>
</form>
```

### Выбор контрола

| Количество вариантов | Тип выбора               | Рекомендуемый элемент              | Логика юзабилити и доступности                                                                                               |
|:---------------------|:-------------------------|:-----------------------------------|:-----------------------------------------------------------------------------------------------------------------------------|
| **1–5**              | Единичный (эксклюзивный) | `<input type="radio">`             | **Сканирование без клика**: все варианты выбора видны сразу. Более быстрое сканирование.                                     |
| **6+**               | Единичный (эксклюзивный) | `<select>`                         | **Экономия пространства**: используйте только в том случае, если вертикального пространства недостаточно или список длинный. |
| **10+ / динамичное** | Единичный (эксклюзивный) | `<input list="id">` (`<datalist>`) | **Нечеткий поиск**: предотвращает утомляемость при прокрутке больших наборов данных (например, стран).                       |
| **Любое**            | Множественный            | `<input type="checkbox">`          | **Стандартная семантика**: встроенные неэксклюзивные переключатели.                                                          |

Ментальная модель в одном предложении: "отображайте взаимоисключающие варианты в виде радио-кнопок, если вариантов меньше шести; используйте `<select>` только тогда, когда пространство ограничено или список длинный".

## 2. Доступные подписи и состояние

### Рекомендации

- Всегда связывайте `<label>` с инпутом с помощью `for` и `id`.
- Размещайте подписи над контролами формы для более быстрого сканирования.
- Используйте видимые подписи; не полагайтесь только на `placeholder`.
- Убедитесь, что вертикальный отступ между подписью и инпутом меньше отступа между группами формы (правило гештальт-близости).
- Используйте `aria-describedby` для связывания инпутов со вспомогательным текстом или сообщениями об ошибках.
- Определяйте атрибут `lang` на `<html>` для правильного перевода.
- Используйте не цветовые визуальные подсказки (иконки, текст) для индикации состояния (не полагайтесь только на цвет).
- Четко указывайте, какие поля являются обязательными.
- Используйте `aria-live` для динамичных сообщений об ошибках.

- Не используйте `placeholder` как замену подписей.
- Не используйте `aria-label` в качестве единственного текстового описания, если требуется перевод.
- Не отключайте кольца фокуса без предоставления высококонтрастной альтернативы.

### Пример кода

```html
<div class="field">
  <label for="username">Имя пользователя:</label>
  <input type="text" id="username" name="username" aria-describedby="user-help" required>
  <span id="user-help" class="hint">3-12 символов.</span>
</div>

<style>
  input:focus-visible {
    outline: 3px solid #0b57d0;
    outline-offset: 2px;
  }
</style>
```

## 3. Автозаполнение и режимы ввода

### Рекомендации

- Используйте атрибут `autocomplete` для определения ожидаемых данных (например, `email`, `tel`, `current-password`, `new-password`).
- Используйте `inputmode` для оптимизации экранной клавиатуры (например, `inputmode="numeric"` для PIN-кода).
- Используйте `enterkeyhint` для установки меток клавиши Enter (например, `дальше`, `готово`).
- Используйте одно поле для сложных чисел (кредитных карт, телефонов) для помощи в автозаполнении.

- Не используйте `type="number"` для кредитных карт или почтовых индексов (вызывает проблемы UI прокрутки и удаляет ведущие нули).

### Пример кода

```html
<label for="zip">Почтовый индекс:</label>
<input type="text" id="zip" name="zip" autocomplete="postal-code" inputmode="numeric" pattern="\d{5}">
```

## 4. Ограничения и валидация

### Рекомендации

- Используйте нативные ограничения: `required`, `minlength`, `maxlength`, `pattern`.
- Используйте псевдоклассы CSS `:user-valid` и `:user-invalid` для ненавязчивой стилизации.
- Используйте ValidityState API (`setCustomValidity`) для кастомных сообщений.

- Не отключайте кнопки отправки формы для блокировки валидации; позволяйте пользователям отправлять форму и видеть ошибки. Однако, отключайте кнопку после валидной отправки во избежание повторной отправки.

### Пример кода

```html
<label for="code">Код активации (4 цифры):</label>
<input type="text" id="code" name="code" required pattern="\d{4}">

<script>
  const input = document.getElementById('code');
  input.addEventListener('invalid', () => {
    input.setCustomValidity('Пожалуйста, введите ровно 4 цифры.');
  });
  input.addEventListener('input', () => {
    input.setCustomValidity('');
  });
</script>
```

### Время валидации

| Событие-триггер         | Стадия            | Разрешенное действие                      | Логика UX / доступности                                                                                       |
|:------------------------|:------------------|:------------------------------------------|:--------------------------------------------------------------------------------------------------------------|
| **`input`**             | Активный ввод     | Только **очистка** существующих ошибок.   | **Ненавязчивая**: не показывайте ошибки до завершения пользовательского ввода.                                |
| **`blur` / `focusout`** | Выход из поля     | **Запуск** проверок и отображение ошибок. | **Контекстуальная валидация**: запускайте валидацию, как только пользователь показал, что "закончил" с полем. |
| **`submit`**            | Финальная попытка | **Блокировка** и установка фокуса.        | **Финальная защита**: перехват плохой полезной нагрузки и принудительная установка фокуса.                    |

Ментальная модель в одном предложении: "выполняйте валидацию при `blur`, чтобы избежать преждевременных предупреждений при вводе, и сбрасывайте ошибки при `input`, как только пользователь приступил к исправлению ошибок".

Безопасность и UX: клиентская валидация предназначена для улучшения пользовательского опыта; серверная валидация - для безопасности. Никогда не рассматривайте ограничения браузера как защиту целостности данных.

## 5. Отзывчивый дизайн и типографика

### Рекомендации

- Используйте одноколоночные макеты для сканирования.
- Устанавливайте `font-size` в значение `1rem` (16px), минимум.
- Расширяйте кликабельные зоны/области касания на мобильных телефонах, например, с помощью паддинга.
- Цели касания должны быть размером `48px`, минимум.
- Используйте корневые (root) единицы измерения (`rem`) и `line-height` без единиц измерения.
- Используйте логические свойства CSS (например, `margin-inline-start`) для поддержки RTL.

### Пример кода

```css
.form-group {
  margin-block-end: 1.5rem;
}

/* Расширяем кликабельную зону без сдвига макета */
label {
  display: inline-block;
  padding: 10px 0;
  margin: -10px 0;
}

input {
  font-size: 1rem;
  padding: 0.75rem;
  min-height: 48px;
  box-sizing: border-box;
}

@media (pointer: coarse) {
  input {
    min-height: 52px;
  }
}
```

## 6. Стилизация контролов формы

### Рекомендации

- Используйте `accent-color` для быстрого брендирования радио-кнопок/чекбоксов.
- Используйте `appearance: none` для кастомных стрелок выпадающих слоев без поломки семантики.
- Убедитесь, что инпуты четко видны и имеют адекватные контрастные границы (например, `#ccc` или темнее на белом фоне).
- Скрывайте инпуты с помощью канонической техники `.visually-hidden` (`clip-path: inset(50%)` с размерами 1px) — не `display: none`, которое удаляет их из дерева доступности.

### Пример кода

```html
<div class="checkbox-container">
  <input type="checkbox" id="sub" name="sub" class="visually-hidden">
  <label for="sub" class="checkbox-label">Подписаться</label>
</div>

<style>
  .visually-hidden {
    position: absolute;
    clip-path: inset(50%);
    overflow: hidden;
    width: 1px;
    height: 1px;
    margin: -1px;
    padding: 0;
    border: 0;
    white-space: nowrap;
  }
  .checkbox-label::before {
    content: "";
    display: inline-block;
    width: 1.25rem;
    height: 1.25rem;
    border: 2px solid #ccc;
  }
  input:focus-visible + .checkbox-label::before {
    outline: 2px solid #0b57d0;
  }
</style>
```

## 7. JavaScript и AJAX

### Рекомендации

- Блокируйте дефолтную навигацию при отправке формы для AJAX (`e.preventDefault()`).
- Используйте интерфейсы `ValidityState` для валидации в реальном времени.
- Используйте `aria-expanded` и `aria-controls` для динамически отображаемого UI.

- Не блокируйте отправку формы при отключении JS; убедитесь в наличии серверного резерва.

### Пример кода

```js
form.addEventListener('submit', (e) => {
  e.preventDefault();
  const data = new FormData(form);
  // fetch('/submit', { method: 'POST', body: data });
});
```

## 8. Идентификация, платежи и повышенная безопасность

### Рекомендации

- Используйте `autocomplete="new-password"` для регистрации и `autocomplete="current-password"` для авторизации.
- Разрешайте вставку текста в поля для паролей.
- Предоставляйте переключатель видимости пароля.
- Указывайте точную цену в кнопках оплаты (например, "Заплатить 1000 рублей").
- Используйте `autocomplete="cc-number"`, `cc-exp`, `cc-csc`.
- Используйте HTTPS для всех страниц.
- Внедрите криптографически безопасные токены анти-CSRF для операций по изменению данных (POST/PUT/DELETE).
- Обезвреживайте пользовательский ввод (например, с помощью DOMPurify) перед его вставкой в DOM для предотвращения XSS.
- Реализуйте защиту от спама (приманки (honeypots) или CAPTCHA) для открытых форм.

- Не используйте HTTP-метод `GET` для конечных точек, выполняющих модификацию данных.
- Не используйте встроенный JavaScript (например, `onclick="..."`) непосредственно в разметке формы для соблюдения строгих политик безопасности контента (CSP).

### Пример кода

```html
<form method="post">
  <input type="hidden" name="csrf_token" value="secure_token_abc123">

  <h1>Регистрация</h1>

  <div class="form-group">
    <label for="name">Полное имя</label>
    <input id="name" name="name" autocomplete="name" required pattern="[\p{L}\.\- ]+">
  </div>

  <div class="form-group">
    <label for="email">Адрес электронной почты</label>
    <input id="email" name="email" type="email" autocomplete="username" required>
  </div>

  <div class="form-group">
    <label for="password">Пароль</label>
    <button id="toggle-password" type="button" aria-pressed="false" aria-label="Show password" aria-describedby="toggle-warning">
      <img class="icon-eye" src="/icons/eye.svg" alt="" width="20" height="20">
      <img class="icon-eye-off" src="/icons/eye-off.svg" alt="" width="20" height="20">
    </button>
    <span id="toggle-warning" class="visually-hidden">Предупреждение: это отобразит ваш пароль на экране.</span>
    <input id="password" name="password" type="password" autocomplete="new-password" minlength="8" aria-describedby="password-constraints" required>
    <div id="password-constraints">Восемь или более символов.</div>
  </div>

  <button id="sign-up">Зарегистрироваться</button>
</form>
```

## 9. Адрес

### Рекомендации

- Используйте одно поле для имени пользователя.
- Используйте `autocomplete="street-address"`.
- Если сайтом пользуются люди из разных стран, используйте элемент `<textarea>` для адресов, чтобы учесть разные форматы адресов в разных географических регионах. Если в форме несколько инпутов для разных частей адреса (например улица, город), используйте `autocomplete` со значениями `address-line1`, `address-line2` и т.д.
- Почтовые индексы должны быть опциональными.

- Не делите имя на переменные (имя, фамилия) для глобальной аудитории.
- Не ограничивайте имена (names) и имена пользователей (usernames) только латинскими символами.

### Пример кода

```html
<!-- Доступная форма адреса с автозаполнением -->
<form action="/save-address" method="POST">
  <div class="form-group">
    <label for="full-name">Полное имя</label>
    <input type="text" id="full-name" name="full_name" maxlength="100" required autocomplete="name">
  </div>

  <div class="form-group">
    <label for="address">Адрес</label>
    <textarea id="address" name="address" required autocomplete="street-address" maxlength="300"></textarea>
  </div>

  <button type="submit">Сохранить</button>
</form>
```

## 10. Тестирование юзабилити и аналитика

### Рекомендации

- Тестируйте формы на разных устройствах, в разных браузерах и на экранах разных размеров.
- Тестируйте навигацию с помощью клавиатуры (с помощью `Tab` и `Shift+Tab`) и проверяйте работу визуального фокуса.
- Эмулируйте разные нарушения (визуальные, двигательные) с помощью инструментов браузера.
- Используйте аналитику для отслеживания показателей заполнения форм и точек отказов.
- Отслеживайте отдельные события (например, фокус, клик), чтобы найти точки микротрения.

- Не полагайтесь только на автоматизированные инструменты (Lighthouse) для юзабилити; тестируйте с помощью реальных пользователей.
- Не отслеживайте конфиденциальные персональные данные в стандартных событиях.

### Пример кода

```html
<form action="/submit" method="POST" id="track-form">
  <label for="postal-code">Почтовый индекс</label>
  <input type="text" id="postal-code" name="postal-code" autocomplete="postal-code" maxlength="20" required>
  <button type="submit" id="submit-btn">Отправить</button>
</form>

<script>
  const trackForm = document.getElementById('track-form');
  const trackBtn = document.getElementById('submit-btn');

  trackBtn.addEventListener('click', () => {
    console.log('Событие аналитики: нажатие кнопки отправки формы');
  });
</script>
```

## 11. Многостраничные формы

### Рекомендации

- Четко показывайте прогресс в многостраничных формах с ясными подписями и индикаторами прогресса.
- У пользователей должна быть возможность свободно переключаться между страницами.
- Используйте контекстно зависимые значения `enterkeyhint` (например, `"назад"`, `"вперед"`) для навигации на экранной клавиатуре.
- Проектируйте дизайн таким образом, чтобы мобильная клавиатура не перекрывала инпуты или кнопки (например, путем их размещения в верхней части области просмотра при фокусировке или с помощью отступа прокрутки (scroll-padding) CSS).

### Пример кода

```html
<nav aria-label="Progress">
  <ol class="progress-tracker">
    <li class="step-done">Шаг 1. Аккаунт</li>
    <li class="step-active" aria-current="step">Шаг 2. Доставка</li>
    <li class="step-todo">Шаг 3. Оплата</li>
  </ol>
</nav>

<button type="button" onclick="history.back()" enterkeyhint="назад">Назад</button>
<button type="submit" enterkeyhint="вперед">Вперед</button>
```
