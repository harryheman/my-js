# Селект, соответствующий бренду

API кастомизируемого селекта (customizable select) предоставляет декларативный, основанный на CSS способ стилизации элементов `<select>`. `appearance: base-select` предоставляет доступ к внутреннему теневому DOM элемента `<select>`, что позволяет стилизовать кнопку, варианты для выбора, иконку стрелки и галочку выбора с помощью стандартных свойств CSS.

Ранее для получения кастомизированного селекта требовалось воссоздавать контрол с нуля с помощью JavaScript, что часто ломало доступность, навигацию с помощью клавиатуры и нативную интеграцию с формами. `appearance: base-select` позволяет кастомизировать внешний вид селекта, а браузер обрабатывает управление фокусом, рендеринг на верхнем уровне и обеспечение доступности.

## Реализация

Для реализации брендированной стилизации селекта:

1. Включаем возможность кастомизации: применяем `appearance: base-select` к элементу `<select>` и псевдоэлементу `::picker(select)` (выпадающий список вариантов для выбора).
2. Добавляем кастомную кнопку (опционально): определяем элемент `<button>` прямо внутри `<select>` для замены дефолтного триггера. Используем элемент `<selectedcontent>` внутри этой кнопки для представления текста или контента выбранного варианта.
3. Стилизуем список вариантов: используем псевдоэлемент `::picker(select)` для применения типографики, цветов фона, границ и теней к выпадающему списку. Браузер рендерит его на верхнем уровне, что устраняет конфликты `z-index`.
4. Стилизуем внутренние иконки:
   - используйте `select::picker-icon` для стилизации или замены иконки стрелки;
   - используйте `option::checkmark` для стилизации галочки рядом с выбранным вариантом.
5. Стилизуйте варианты для выбора: стилизуйте элементы `<option>` для индикации наведения, добавления паддинга и правильной компоновки контента.

## Пример кода

Следующий пример демонстрирует кастомный селект, стилизованный с помощью моноширинного шрифта и пунктирных границ:

```css
/* Включаем кастомизацию для селекта и его пикера */
.brand-select,
.brand-select::picker(select) {
  appearance: base-select;
}

/* Стилизуем видимую кнопку-триггер */
.brand-select {
  font-family: 'Courier New', monospace;
  background-color: #fffaf0;
  color: #8b4513;
  border: 2px dashed #8b4513;
  border-radius: 4px;
  padding: 0.75rem;
  font-size: 1rem;
  cursor: pointer;
}

/* Стилизуем выпадающий список вариантов для выбора */
.brand-select::picker(select) {
  font-family: 'Courier New', monospace;
  background-color: #fffaf0;
  border: 2px dashed #8b4513;
  border-radius: 4px;
  padding: 0.5rem;
}

/* Кастомизируем внутренние цвета для совпадения с текстом */
.brand-select::picker-icon {
  color: #8b4513;
}

.brand-select option::checkmark {
  color: #8b4513;
}

/* Стилизуем отдельные варианты и эффекты наведения */
.brand-select option {
  padding: 0.5rem;
  border-radius: 4px;
  color: #8b4513;
  cursor: pointer;
}

.brand-select option:hover {
  background-color: #fdf5e6;
}
```

```html
<label for="preferences">Выберите способ доставки</label>
<select class="brand-select" id="preferences" name="preferences">
  <button>
    <selectedcontent></selectedcontent>
  </button>
  <option value="standard">Стандартная</option>
  <option value="express" selected>Экспресс</option>
  <option value="overnight">Ночная</option>
</select>
```

## Стратегическая реализация и лучшие практики

- Используйте `appearance: base-select`, когда ваша дизайн-система требует высокой точности, визуальной согласованности всех контролов формы, что не может быть достигнуто с помощью стандартных кросс-браузерных возможностей.
- Не используйте это, если полагаетесь на нативный пикер операционной системы (например, стандартное колесо прокрутки на устройствах iOS). Использование `base-select` отключает нативный мобильный UI контролов ради верхнеуровневых веб-меню.
- Убедитесь, что цветовой контраст удовлетворяет стандартам WCAG. Кастомизируемый пикер позволяет устанавливать любые цвета, но ответственность за то, чтобы текст оставался разборчивым на кастомном фоне, несете вы.
- Тестируйте поведение макета. Установка `appearance: base-select` удаляет дефолтное поведение браузера по масштабированию селекта на основе самого длинного варианта для выбора. Может потребоваться установка фиксированной ширины или ограничений flex/grid для предотвращения сдвигов макета.
- Убедитесь, что `<select>` содержит атрибут `name` и соответствующий `<label>`. Это гарантирует, что даже с кастомным UI, компонент остается доступным для устройств чтения с экрана и работает правильно при стандартной отправке формы.

## Поддержка и страховка

Кастомизируемый `<select>` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 135+ и Edge 135+ (с апреля 2025 года). Не поддерживается в Firefox и Safari.

В браузерах, которые пока не поддерживают `appearance: base-select`, элемент `<select>` мягко деградирует к стандартному выпадающему слою операционной системы.

- Нетекстовый контент игнорируется: старые браузеры удаляют теги HTML (такие как `<svg>` или `<div>`) внутри тегов `<option>` и рендерят только текстовые узлы. Убедитесь, что текст `<option>` читаемый и осмысленный сам по себе.
- Обработка структуры HTML: стандартные парсеры могут игнорировать теги `<button>` и `<selectedcontent>` внутри `<select>` или считать их невалидными. Однако, если в качестве резервного варианта отображается обычный текст, для прогрессивного улучшения тяжелые полифиллы JavaScript использовать не обязательно.

```javascript
document.addEventListener("DOMContentLoaded", () => {
  // Проверяем поддержку значения "base-select"
  if (!CSS.supports("appearance", "base-select")) {
    // Кастомизация селекта не поддерживается
  }
});
```
