# Стилизация на основе контента

Исторически применение разных макетов к компоненту на основе его контента требовало либо JavaScript, либо условной логики в языке шаблонов HTML для внедрения классов-модификаторов (таких как `.card--has-image` или `.card--text-only`).

Псевдокласс `:has()` снижает эту необходимость, выступая в роли родительского селектора. Он позволяет условно стилизовать элемент контейнера на основе наличия или отсутствия определенных дочерних элементов.

С помощью `:has()` можно легко определить разные вариации макета целиком в CSS на основе актуального контента элемента в DOM. Опционально, его можно комбинировать с `:not()` для явной идентификации отсутствия контента для определения дефолтных стилей.

## Реализация

Селектор `:has()` на элементе-контейнере предназначен для определения наличия конкретного дочернего контента.

Для разработки компонента, меняющего макет на основе контента:

1. Определите дефолтные стили: примените базовые стили макета к контейнеру (например, простой одноколоночный стек).
2. Примените перезаписи на основе контента: ищите контейнер с помощью `:has([child-selector])` и примените новые стили макета при наличии контента (например, многоколоночную сетку).

Пример: компонент карточки, переключающийся в строчный макет при наличии изображения.

```css
/* 1. Определяем дефолтное состояние контейнера. */
/* Это применяется при отсутствии изображения */
.article-card {
  display: flex;
  flex-direction: column;
  gap: 1rem;
  padding: 1.5rem;
  border: 1px solid #ccc;
  border-radius: 8px;
}

/* 2. Применяем стили при наличии определенного дочернего элемента. */
/* Используем :has() для проверки потомка */
.article-card:has(img) {
  /* Применяем строчный макет при наличии изображения */
  flex-direction: row;
  align-items: center;
}

/* Опционально: стилизуем само изображение (здесь :has() не нужен) */
.article-card img {
  width: 150px;
  height: auto;
  border-radius: 4px;
}

/* Комбинация :has() и :not() позволяет явно определять отсутствие контента. */
/* Это применяется к карточке, которая не содержит изображения */
.article-card:not(:has(img)) {
  /* Например, применяем другой цвет фона для карточек, содержащих только текст */
  background-color: #f9f9f9;
}
```

```html
<!-- Карточка с изображением (строчный макет) -->
<article class="article-card">
  <img src="thumbnail.jpg" alt="Article thumbnail" />
  <div class="content">
    <h2>Карточка с изображением</h2>
    <p>Контент этой карточки размещается в строке.</p>
  </div>
</article>

<!-- Карточка без изображения (дефолтный колоночный макет) -->
<article class="article-card">
  <div class="content">
    <h2>Текстовая карточка</h2>
    <p>Контент этой карточки размещается в колонке, а цвет ее фона определяется правилом :not(:has()).</p>
  </div>
</article>
```

Заметка о производительности: ограничивайте область видимости селектора `:has()` самым специфичным компонентом (например, `.article-card`). Не применяйте его к очень высокоуровневым элементам, таким как `body:has(img)`, если изменения стилей являются очень локальными, более широкие запросы `:has()` могут вызывать больше повторных вычислений макета.

## Страховка и поддержка

`:has()` поддерживается всеми основными браузерами с 19.12.2023.

Если стилизация макета на основе контента является критичной для пользовательского опыта или дизайна страницы, следует предоставить резерв для браузеров, не поддерживающих селектор `:has()`. Для чисто декоративных целей `:has()` может использоваться как прогрессивное улучшение без резерва.

При реализации резерва для критичных макетов следует использовать `@supports not selector(:has(*))` в CSS для определения традиционного резерва на основе классов (например, `.has-image`).

В отличие от интерактивной стилизации на основе состояния, наличие контента, как правило, известно во время рендеринга. Самым надежным резервом является внедрение серверным движком шаблонов (или генератором статического контента) класса, вроде `.has-image`, в контейнер при наличии дочернего элемента (такого как изображение) в данных.

Если рендеринг выполняется на клиенте, следует использовать небольшой скрипт с `CSS.supports()` для определения поддержки и применять класс при загрузке страницы или после динамического внедрения контента:

```css
/* Резервный CSS для старых браузеров */
@supports not selector(:has(*)) {
  .article-card.has-image {
    flex-direction: row;
    align-items: center;
  }

  .article-card:not(.has-image) {
    background-color: #f9f9f9;
  }
}
```

```javascript
/* Резервный JavaScript для старых браузеров (если не используется SSR для добавления класса) */
if (!CSS.supports('selector(:has(*))')) {
  // Находим все компоненты, требующие проверки
  const cards = document.querySelectorAll('.article-card');

  cards.forEach(card => {
    // Если критичный контент существует, вручную применяем резервный класс
    if (card.querySelector('img')) {
      card.classList.add('has-image');
    }
  });
}
```
