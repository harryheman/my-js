# Поиск скрытого контента

Веб-интерфейсы часто скрывают контент на отображении (view) для улучшения пользовательского опыта, сохранения пространства на экране или улучшения производительности страницы. Традиционные методы, такие как `display: none` или `visibility: hidden`, не только скрывают контент визуально, но также делают его полностью недоступным для устройств чтения с экрана и браузерных возможностей, таких как "Поиск по странице".

Для визуального скрытия контента с сохранением возможности его поиска пользователями и доступности для глубоких ссылок (deep links) через фрагменты URL и ссылки "Прокрутить до фрагмента текста", можно использовать элемент HTML `<details>` или HTML-атрибут `hidden="until-found"`. Рекомендуется использовать элемент `<details>`, поскольку он проще в реализации и поддержке, но существуют случаи, когда `<details>` не подходит и нужно использовать `hidden="until-found"`.

Например:

- Если нужен полный контроль над стилизацией механизма отображения/скрытия.
- Если контролы UI для отображения/скрытия контента находятся в другой части DOM.
- Если контент не должен скрываться после отображения.

## Реализация

Элемент `<details>` по умолчанию содержит доступный для поиска и вспомогательных технологий текст, специальной реализации не требуется. Используйте `<details>`, вместо `hidden="until-found"`, если это возможно.

При использовании `hidden="until-found"` следуйте этим инструкциям:

1. Примените атрибут: добавьте атрибут `hidden="until-found"` прямо к элементам, содержащим контент, который должен быть скрыт.
2. Синхронизируйте состояние UI: если интерфейс содержит связанные состояния, которые зависят от видимости контента (например, обновление атрибутов ARIA, переключение CSS классов открытия/закрытия или вращение иконок аккордеона):
   - Следует добавить обработчик события `beforematch`.
   - Зарегистрировать обработчик события `beforematch` прямо на элементе, содержащем атрибут `hidden="until-found"`. Поскольку событие всплывает (bubbles), можно использовать делегирование событий путем регистрации одного обработчика на родительском элемента (таком как контейнер вкладки) для управления несколькими скрытыми разделами одновременно.
   - Внутри обработчика событий должна содержаться логика синхронизации связанных элементов UI (например, закрытие других открытых вкладок или изменение состояния кнопки-переключателя).

## Пример кода

Цель: отрендерить скрытый контейнер, в котором контент является невидимым для пользователя, но остается доступным для поиска.

### Использование элемента `<details>`

```html
<details>
  <summary>Нажмите для раскрытия</summary>
  <p>Этот контент визуально скрыт.</p>
</details>
```

#### Взаимоисключающие раскрытия информации

Для обработки взаимоисключающий (mutually exclusive) областей контента, таких как эксклюзивный аккордеон, следует использовать нативный элемент HTML `<details>` с общим атрибутом `name`:

```html
<div class="accordion-group">
  <!-- Атрибут `name` создает эксклюзивную раскрывающуюся группу -->
  <details class="disclosure" name="my-accordion">
    <summary>Раздел 1</summary>
    <p>Контент раздела 1</p>
  </details>

  <details class="disclosure" name="my-accordion" open>
    <summary>Раздел 2</summary>
    <p>Контент раздела 2</p>
  </details>

  <details class="disclosure" name="my-accordion">
    <summary>Раздел 3</summary>
    <p>Контент раздела 3</p>
  </details>
</div>
```

### Использование атрибута `hidden="until-found"`

```html
<!-- Браузер автоматически удаляет `hidden="until-found"` при совпадении поиска -->
<div class="hidden-container" hidden="until-found">
  <p>Этот контент визуально скрыт.</p>
</div>
```

#### Кастомные взаимоисключающие раскрытия информации

При обработке кастомных взаимоисключающих областей, управляемых внешними кнопками, убедитесь, что только совпавшая панель является видимой. В обработчике события `beforematch` необходимо синхронизировать связанные состояния ARIA, такие как установка `aria-expanded="true"` на кнопке управления:

```html
<div class="custom-accordion">
  <div class="controls">
    <button aria-expanded="true" aria-controls="panel-1" id="btn-1">Раздел 1</button>
    <button aria-expanded="false" aria-controls="panel-2" id="btn-2">Раздел 2</button>
  </div>

  <div id="panel-1" class="panel">
    <p>Контент раздела 1 (видимый)</p>
  </div>

  <div id="panel-2" class="panel" hidden="until-found">
    <p>Контент раздела 2 (скрытый)</p>
  </div>
</div>
```

```javascript
const accordion = document.querySelector('.custom-accordion');

accordion.addEventListener('beforematch', (e) => {
  // Скрываем все панели и синхронизируем состояние кнопок перед тем, как браузер покажет соответствующую панель
  accordion.querySelectorAll('.panel').forEach((panel) => {
    if (panel !== e.target) {
      panel.hidden = 'until-found';
    }
  });
  accordion.querySelectorAll('button').forEach((btn) => {
    const controls = btn.getAttribute('aria-controls');
    btn.setAttribute('aria-expanded', controls === e.target.id ? 'true' : 'false');
  });
});
```

## Лучшие практики для `hidden="until-found"`

- Применяйте границы, паддинги и фоны к вложенным оберткам потомков, а не к самому элементу с атрибутом `hidden="until-found"`. Это предотвращает случайные сдвиги макета или визуальные артефакты, когда элемент скрыт.
- Не применяйте `display: none`, `visibility: hidden` или аналогичные свойства CSS к элементам с атрибутом `hidden="until-found"`. Это ломает нативный функционал и скрывает контент от поиска.
- Не используйте `hidden="until-found"` для чувствительной информации, внутренних токенов данных или данных, которые не должны быть доступными через поиск.
- Не используйте `hidden="until-found"` как замену для текста "только для устройств чтения с экрана" (`.sr-only`).

## Поддержка и страховка

Элемент `<details>` поддерживается всеми основными браузерами, поэтому страховка не требуется.

Атрибут `hidden="until-found"` пока не поддерживается всеми основными браузерами, но его можно безопасно использовать с резервом в браузерах без поддержки. Не игнорируйте `hidden="until-found"` из-за отсутствия поддержки, поскольку предоставляемые им выгоды для доступности перевешивают стоимость реализации резерва.

#### Резерв `hidden="until-found"`

Для стандартных элементов UI, таких как аккордеоны или разделы "Читать больше", используйте JavaScript для определения поддержки и отображения всего контента при ее отсутствии:

```javascript
if (!('onbeforematch' in HTMLElement.prototype)) {
  // Раскрываем весь скрытый контент в браузерах без поддержки
  document.querySelectorAll('[hidden="until-found"]').forEach((el) => {
    el.removeAttribute('hidden');
    // Также обновляем любые ссылки ARIA на этот элемент
  });
}
```
