# Группировка переходов элементов

При добавлении, удалении или упорядочении элементов списка, переходы (transitions) могут помочь пользователям в понимании происходящего. Переходы отображения позволяют выполнять переходы между двумя состояниями элемента путем присвоения элементу уникального `view-transition-name`. Когда несколько элементов на странице используют одну логику перехода, `view-transition-class` позволяет определить эту логику один раз в CSS, а не повторять ее для каждого уникального `view-transition-name`. Это делает таблицы стилей более поддерживаемыми, обеспечивая согласованность анимации группы элементов.

## Реализация

1. Присвоение уникальных названий и общего класса.

Каждый элемент, который должен отдельно отслеживаться в процессе перехода, должен иметь уникальное `view-transition-name`:

```html
<li style="view-transition-name: item-1" class="item">Item 1</li>
<li style="view-transition-name: item-2" class="item">Item 2</li>
```

Для применения общих стилей также присваиваем `view-transition-class`:

```css
.item {
  view-transition-class: list-item;
}
```

2. Определение общей логики перехода.

Используем псевдокласс `::view-transition-group()` с селектором класса для применения стилей ко всем членам этой группы:

```css
/* Находит любую группу перехода отображения, которая содержит класс `list-item` */
::view-transition-group(.list-item) {
  animation-duration: 0.5s;
  animation-timing-function: ease-in-out;
}

/* Обрабатываем доступность путем учета предпочтений о движении */
@media (prefers-reduced-motion: reduce) {
  /* Отключаем все групповые переходы, включая дефолтную группу `root` */
  ::view-transition-group(*),
  ::view-transition-old(*),
  ::view-transition-new(*) {
    animation: none !important;
  }
}
```

3. Опционально: определение анимации входа и выхода.

Используем селектор `:only-child` для добавления специфичных переходов к добавляемым или удаляемым элементам. Псевдоэлементы `::view-transition-new()` и `::view-transition-old()` являются потомками псевдоэлемента `::view-transition-image-pair()`, что позволяет определить, является ли добавляемый/удаляемый элемент единственным потомком.

```css
/* Элемент `::view-transition-new()` является единственным потомком, если его не было до перехода отображения, т.е. он был добавлен в список */
::view-transition-new(.list-item):only-child {
  animation-name: slide-in;
  animation-duration: 1s;
}
/* Элемент `::view-transition-old()` является единственным потомком, если отсутствует после перехода отображения, т.е. он был удален из списка */
::view-transition-old(.list-item):only-child {
  animation-name: slide-out;
  animation-duration: 1s;
}

@keyframes slide-in {
  from {
    translate: -100vw 0;
  }
}
@keyframes slide-out {
  to {
    translate: -100vw 0;
  }
}
```

4. Запуск перехода.

Оборачиваем обновление DOM в `document.startViewTransition()`. Браузер захватит старое состояние, выполнит обновление и переход к новому состоянию.

```javascript
function updateList(newData) {
  document.startViewTransition(() => {
    // Все изменения DOM здесь будут анимироваться
    render(newData);
  });
}
```

5. Поддержка интерактивности элементов без переходов.

Переходы отображения работают за счет наложения снимков (snapshots) элементов DOM и их последующей смены. Это означает, что в процессе перехода элементы не интерактивны. Элементы без переходов можно сделать интерактивными путем отключения событий касания:

```css
::view-transition {
  /* Элементы без переходов будут интерактивными */
  pointer-events: none;
}
```

Кроме того, по умолчанию, элемент `:root` имеет переход отображения `root`, который включает дефолтные полностраничные переходы. Если корневой элемент не изменился, будет выполнен переход между двумя идентичными снимками, которые не интерактивны. Для выполнения переходов только отдельных элементов, а не всего экрана, можно отключить переход `root`:

```css
:root {
  view-transition-name: none;
}
```

## Поддержка и страховка

Свойство `view-transition-class` поддерживается всеми основными браузерами с 14.10.2025.

Переходы отображения являются прогрессивным улучшением. Если браузер не поддерживает `document.startViewTransition()`, обновление DOM произойдет мгновенно, обеспечивая функциональный опыт, но без анимации:

```javascript
if (document.startViewTransition) {
  document.startViewTransition(() => updateDOM());
} else {
  // Резерв: выполняем обновление без анимации
  updateDOM();
}
```

В CSS браузеры без поддержки не распознают свойство `view-transition-class` или псевдоэлемент `::view-transition-group()` и просто их игнорируют, анимация не применяется.
