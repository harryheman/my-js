# Выделение привязанных элементов карусели

Запросы состояния прокрутки контейнера (scroll-state container queries) позволяют стилизовать элементы на основе их текущего состояния прокрутки, например, на основе того, является ли элемент "прилипшим" (stuck) (из-за липкого (sticky) позиционирования) или "привязанным" (snap) (через привязку прокрутки (scroll snapping)). Это позволяет визуально выделить активный элемент карусели или галереи без помощи наблюдателей пересечения (intersection observers) или обработчиков прокрутки JavaScript.

## Реализация

Для подсветки привязанных элементов нужно создать контейнер с привязанной прокруткой (scroll-snap container), определить цели привязки (snap targets) как контейнеры с состоянием прокрутки (scroll-state containers) и затем запрашивать это состояние для стилизации потомков.

### 1. Создаем контейнер с привязанной прокруткой

Родительский контейнер должен содержать свойство `scroll-snap-type`.

```html
<div class="carousel">
  <div class="carousel-item">
    <div class="card">Контент продукта 1</div>
  </div>
  <div class="carousel-item">
    <div class="card">Контент продукта 2</div>
  </div>
</div>
```

```css
.carousel {
  display: flex;
  overflow-x: auto;
  /* Включаем привязку прокрутки на контейнере */
  scroll-snap-type: x mandatory;
}
```

### 2. Определяем цели привязки как контейнеры с состоянием прокрутки

Каждый элемент карусели, прокрутка которого должна отслеживаться, должен определяться как контейнер `scroll-state`.

```css
.carousel-item {
  /* Определяем, как элемент привязывается внутри контейнера */
  scroll-snap-align: center;

  /* Делаем элемент контейнером с состоянием прокрутки */
  container-type: scroll-state;
}
```

### 3. Запрашиваем состояние `snapped`

Поскольку запросы к контейнеру стилизуют потомков, стили подсветки должны применяться к элементу внутри цели привязки. Поскольку прокручиваемый контейнер переполняется по оси x, следует использовать запрос `scroll-state(snapped: x)`.

```css
/* Определяем переход за пределами запросов, чтобы он применялся, независимо от состояния  */
.card {
  transition:
    scale 0.4s cubic-bezier(0.25, 0.8, 0.25, 1),
    background-color 0.4s,
    color 0.4s,
    box-shadow 0.4s;
}

/*
Показываем эффект только пользователям, не запрашивавшим снижение движения. Отключаем эффект полностью, включая изменение цвета, поскольку это вызывает вспышку, которая может стать проблемой
*/
@media (prefers-reduced-motion: no-preference) {
  /* Стилизуем контент, когда его предок .carousel-item привязан к оси x */
  @container scroll-state(snapped: x) {
    .card {
      background: #007bff;
      color: white;
      scale: 1.15;
      box-shadow: 0 10px 25px rgba(0, 123, 255, 0.3);
    }
  }
}

/* Отключаем эффекты для чувствительных к движению пользователей */
@media (prefers-reduced-motion: reduce) {
  .card {
    transition: none !important;
    scale: 1 !important;
  }
}
```

Дескриптор `snapped` может запрашивать определенные оси: `x`, `y`, `inline`, `block` или `both`.


## Поддержка и страховка

Запросы состояния прокрутки контейнера имеют ограниченную поддержку браузеров. Поддерживаются в Chrome 133+ и Edge 133+ (с февраля 2025 года). Не поддерживаются в Firefox и Safari.

В браузерах без поддержки необходимо предоставить функциональный базовый опыт, когда все элементы четко различимы, даже без "активной" подсветки.

### Определение поддержки

Для предоставления продвинутого функционала только в браузерах с поддержкой можно использовать `@supports`:

```css
@supports (container-type: scroll-state) {
  /* Продвинутые стили */
}
```

### Резерв JavaScript

Если подсветка является критически важной для пользовательского опыта, используйте `IntersectionObserver` для определения привязанного элемента. Определите наблюдаемую область как тонкий срез (slice) в центре карусели, указав `rootMargin` с отрицательным значением. Например, чтобы считать элемент пересекающимся, если он находится в середине 2% карусели, установите `rootMargin` в значение `"0px -49%"`.

```javascript
// Определяем поддержку и применяем основанный на JS резерв
if (!CSS.supports('container-type', 'scroll-state')) {
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      // Переключаем класс на основе пересечения
      entry.target.classList.toggle('is-snapped', entry.isIntersecting);
    });
  }, {
    root: document.querySelector(".carousel"),
    // Элемент карусели считается пересекающимся, если любая его часть находится в середине 2% карусели
    rootMargin: "0px -49%"
  });

  document.querySelectorAll('.carousel-item').forEach(item => {
    observer.observe(item);
  });
}
```
