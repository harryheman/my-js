# Создание эффектов смены слайдов карусели

Эффекты смены слайдов - отличный способ добавить визуальной привлекательности карусели. При прокрутке слайдов пользователем, можно анимировать вход каждого слайда, его центрирование и выход из области прокрутки (scrollport). Например, слайды могут появляться и исчезать, вращаться или масштабироваться. Это создает динамичный и вовлекающий опыт пользователя. В отличие от простой анимации входа/выхода, в этом эффекте используется одна продолжающаяся анимация для управления прохождением слайдов через всю область прокрутки.

## Реализация

Для создания эффектов смены слайдов карусели:

1. Создаем "прокручиваемый контейнер" (scroller): этот элемент является контейнером для слайдов карусели. В этом примере используется `overflow-x: scroll` для горизонтальной прокрутки.

```html
<ul class="scroller">
  <li class="entry">1</li>
  <li class="entry">2</li>
  <li class="entry">3</li>
  <!-- ... -->
</ul>
```

```css
.scroller {
  overflow-x: scroll;
}
```

2. Определяем анимацию: создаем анимацию CSS, которая определяет различные состояния слайда по мере его прохождения через область прокрутки. Можно определить ключевые кадры (keyframes) для любой части анимации. Например, можно определить состояние, когда слайд находится в середине области прокрутки, с помощью ключевого кадра `50%`. В этом примере свойство `scale` заставляет слайды увеличиваться по мере приближения к центру и уменьшаться по мере удаления от него.

```css
@keyframes animate {
  0% {
    scale: 0.5;
  }
  50% {
    scale: 1;
  }
  100% {
    scale: 0.5;
  }
}
```

3. Применяем анимацию и временную шкалу отображения (`view-timeline`): добавляем анимацию к слайдам карусели и привязываем ее к `view-timeline`, которое отслеживает элемент по мере его прохождения через контейнер.

```css
.scroller > * {
  animation: animate auto linear both;
  animation-timeline: view(inline);
}
```

По умолчанию, `view()` отслеживает элемент по оси `block` (вертикальная ось). Для отслеживания по оси `inline` (горизонтальная ось), можно использовать `view(inline)`.

## Пример кода

Анимация прокрутки слайдов горизонтальной карусели с помощью анонимной `view-timeline`:

```css
@keyframes animate {
  0% {
    scale: 0.5;
  }

  50% {
    scale: 1;
  }

  100% {
    scale: 0.5;
  }
}

.scroller > * {
  /* Применяем анимацию продолжительностью `auto` */
  animation: animate auto linear both;
  /* Применяем анонимную временную шкалу отображения,
  отслеживающую прохождение элемента через область прокрутки
  по горизонтальной оси, к временной шкале анимации */
  animation-timeline: view(inline);
}
```

Анимация прокрутки слайдов горизонтальной карусели с помощью именованной `view-timeline`:

```css
@keyframes animate {
  0% {
    scale: 0.5;
  }

  50% {
    scale: 1;
  }

  100% {
    scale: 0.5;
  }
}

/*Это создает именованную `view-timeline` для каждого элемента карусели.
  Временная шкала используется для управления анимацией, которая
  применяется к тому же элементу. */
.scroller > * {
  /* Применяем анимацию продолжительностью `auto` */
  animation: animate auto linear both;
  /* Определяем именованную временную шкалу отображения,
  отслеживающую прохождение элемента через область прокрутки
  по горизонтальной оси */
  view-timeline: --item inline;
  /* Применяем временную шкалу отображения к временной шкале анимации */
  animation-timeline: --item;
}
```

## Лучшие практики

При использовании анимации, управляемой прокруткой (scroll-driven animations), важно следовать нескольким лучшим практикам для обеспечения плавного и доступного опыта:

- Используйте определение поддержки: не все браузеры поддерживают анимацию, управляемую прокруткой. Используйте `@supports ((animation-timeline: view()) and (animation-range: entry))` для определения поддержки и предоставляйте резерв для браузеров без поддержки.
  - Проверка `(animation-range: entry)` предназначена для исключения браузеров с частичной поддержкой.
  - Не используйте пакет `scroll-timeline-polyfill` в качестве резерва, поскольку в нем реализован не весь функционал, и он содержит много известных проблем.
  - Если анимация является чисто декоративной, резерв можно не предоставлять.
- Учитывайте предпочтения пользователей: некоторые пользователи предпочитают минимум движений в вебе. Используйте медиа-запрос `prefers-reduced-motion` для отключения или уменьшения количества анимаций для таких пользователей.
- Старайтесь анимировать только "производительные" свойства CSS: для плавной анимации старайтесь анимировать только те свойства, которые могут обрабатываться потоком компоновщика браузера, такие как `transform` и `opacity`. Анимация таких свойств, как `width` или `height` может привести к проблемам с производительностью.
- Используйте правильный порядок объявлений: объявляйте `animation-timeline` и `animation-range` после стандартного сокращения `animation` во избежание сброса временной шкалы сокращением.

Используйте именованную `view-timeline`, когда несколько элементов DOM должны анимироваться на основе одной временной шкалы или когда нужно анимировать потомков элемента с `view-timeline`, определенным на нем. Если анимируемый элемент - это также элемент с `view-timeline`, можно использовать анонимную временную шкалу отображения с помощью `view()`.

При использовании функции `view()` для создания анимации, управляемой прокруткой:

- Опционально явно указывайте ось для отслеживания: по умолчанию отслеживается ось `block` (вертикальная), но можно всегда явно указывать ось для отслеживания с помощью `view(block)` или `view(inline)`.

При использовании свойства `view-timeline` для создания анимации, управляемой прокруткой:

- Используйте `<dashed-ident>` для названия (например, `view-timeline: --my-custom-name`).
- Опционально явно указывайте ось для отслеживания: по умолчанию отслеживается ось `block` (вертикальная), но можно всегда явно указывать ось для отслеживания с помощью `view-timeline-axis`.
- Убедитесь, что область поиска работает: когда элемент с `view-timeline` не является предком плоского дерева анимируемого элемента, повысьте видимость именованной `view-timeline` с помощью `timeline-scope` на общем предке.

## Поддержка и страховка

Анимация, управляемая прокруткой, имеет ограниченную поддержку браузеров. Поддерживается в Chrome 115+, Edge 115+ (с июля 2023 года) и Safari 26+ (с сентября 2025 года). Не поддерживается в Firefox.

Для браузеров без поддержки можно использовать резерв для воссоздания визуальных эффектов. Резерв, как правило, создается с помощью обработчика прокрутки (для эффектов ScrollTimeline) или IntersectionObserver API (для эффектов ViewTimeline).

В браузерах с поддержкой всегда используйте нативную реализацию CSS, поскольку она гораздо более производительна.

Обратите внимание, что не все эффекты можно воссоздать с помощью резерва.

Следующий скрипт применяет резерв для браузеров, не поддерживающих анимацию, управляемую прокруткой. В нем используется Web Animations API (`Element.animate()`) для создания приостанавливаемой анимации для каждого элемента карусели. В обработчике события `scroll` прокручиваемого контейнера обновляется `currentTime` каждой анимации на основе прогресса прокрутки элемента.

```js
if (!CSS.supports('(animation-timeline: view()) and (animation-range: entry)')) {
  const scroller = document.querySelector('.scroller');
  const entries = document.querySelectorAll('.entry');

  // Создаем карту для хранения анимаций
  const animations = new Map();

  entries.forEach(entry => {
    const animation = entry.animate(
      {
        scale: ['0.5', '1', '0.5']
      },
      {
        duration: 1, // Мы сами управляем временем
        fill: 'both'
      }
    );
    animation.pause();
    animations.set(entry, animation);
  });

  // Обновляем анимации при прокрутке
  const tick = () => {
    const scrollerRect = scroller.getBoundingClientRect();

    entries.forEach(entry => {
      const animation = animations.get(entry);
      if (!animation) return;

      const entryRect = entry.getBoundingClientRect();
      const progress = (entryRect.left + entryRect.width / 2 - scrollerRect.left) / scrollerRect.width;

      animation.currentTime = progress;
    });
  };

  scroller.addEventListener('scroll', tick);
  tick();
}
```
