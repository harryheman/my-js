# Определение цветовой схемы компонента

Хотя обычно свойство `color-scheme` устанавливается на корневом элементе (root), оно также может устанавливаться на отдельных элементах для использования ими цветовой схемы, отличной от схемы остальной страницы. Это может быть полезным для компонентов, которые должны всегда отображаться в определенной цветовой схеме (например, всегда в темном или светлом режиме).

Примеры случаев использования:

- элементы, всегда находящиеся в темном режиме, даже на страницах со светлым режимом, по эстетическим соображениям, например, блоки кода, медиа-проигрыватели, галереи фотографий;
- области, содержащие медиа, предназначенные для светлого фона (например, изображения, видео, иллюстрации, превью версий для печати) могут всегда находиться в светлом режиме, даже если остальная страница находится в темном режиме;
- элементы, цветовая схема которых устанавливается настройкой пользователя, такие как превью компонентов;
- встраивания (embeddings), которые не поддерживают оба режима;
- инструменты дизайна, карты, визуализации, игры и т.д.

## Когда менять цвета, а когда форсировать `color-scheme`?

Не каждому элементу, который содержит светлый текст на темном фоне или темный текст на светлом фоне, нужна принудительная `color-scheme`. Например, основная кнопка может рендериться синей с белым текстом в светлом режиме, но это может быть не так при `color-scheme: dark`.

Как правило, элементы с принудительной `color-scheme` представляют собой сложные компоненты, создающие собственный визуальный контекст, а не простые контейнеры.

При решении вопроса об использовании другой `color-scheme` на элементе, спросите себя:

- Должен ли встроенный UI браузера, который не кастомизирован иным образом (например, контролы формы, полосы прокрутки и т.д.), использовать эту цветовую схему или он должен адаптироваться к цветовой схеме страницы? -> Если первое, не используйте `color-scheme`.
- Должны ли цвета `light-dark()` разрешаться (resolve) таким же образом, что и на остальной странице, или они должны разрешаться на основе перезаписи (override)? -> Если первое, не используйте `color-scheme`.
- Должны ли потомки быть в этой `color-scheme`? Если нет, не используйте `color-scheme`.

## Реализация

Специфичные для компонента перезаписи, обычно (но не обязательно), используются на страницах, также поддерживающих несколько цветовых схем через глобальную `color-scheme`.

Когда `color-scheme` всей страницы на месте, и используются чувствительные к ней токены (например, через `light-dark()`), можно просто установить `color-scheme` на отдельные компоненты для перезаписи цветового режима для определенных поддеревьев элементов:

```css
pre, code, .dark {
  color-scheme: dark;
}
```

Обратите внимание, что некоторые браузеры все равно адаптируют компоненты к другой цветовой схеме. Для использования цветовой схемы во всех случаях, используйте `only`, т.е. `color-scheme: only dark;`, вместо `color-scheme: dark;`.

### Адаптация не цветовых значений

`light-dark()` в настоящее время работает только для цветов.

## Лучшие практики

- Не устанавливайте `color-scheme` на элементы без фона, поскольку существует риск смешивания пар цветов фона и текста из разных схем, что может привести к нечитаемому тексту.
- Опционально: хотя можно использовать те же пары цветов, что и на всей странице, можно определять другие пары цветов для отдельных компонентов. Например, мы можем захотеть, чтобы компонент темного режима, используемый на странице светлого режима, был немного менее темным, чем когда тот же компонент темного режима используется на странице, которая в целом находится в темном режиме.

## Известные проблемы

### Наследование цветов `light-dark()`

`light-dark()` разрешается в вычисленное значение.

Это означает, что любое унаследованное свойство `<color>`, установленное в `light-dark()`, передаст потомкам один из двух цветов, а не само выражение `light-dark()`.

Это включает:

- встроенные свойства цвета, которые наследуются, такие как `color`, `accent-color`, `fill`, `stroke`, `text-shadow`, `caret-color`;
- любые наследуемые кастомные свойства, определенные с помощью `syntax: <color>` и `inherits: true`;
- любое другое свойство `<color>`, установленное в значение `inherit`.

Это означает, что мы должны:

- не регистрировать кастомные свойства, содержащие дизайн-токены (например, `--surface-color`) как `<color>`. Токены должны содержать "живое" выражение `light-dark()`, чтобы потомки могли повторно разрешать цвета при другой `color-scheme`;
- при установке `color-scheme` на элементе, переопределять любые унаследованные свойства `<color>`, которым могли быть присвоены значения `light-dark()` (напрямую или через токены дизайна), даже если используется один и тот же дизайн-токен;
- не использовать `inherit` в свойствах `<color>` на элементах с перезаписью `color-scheme` (можно использовать на их потомках);
- использовать зарегистрированные свойства `<color>` для противоположного варианта использования: когда мы намеренно хотим сделать снимок (snapshot) разрешенного цвета предка и предотвратить его повторное разрешение в `color-scheme` потомка. Например, в случае захвата фона страницы для использования в другом месте;
- если нужно анимировать цвет, используйте отдельную директиву регистрации `@property` свойства `<color>` на анимируемом элементе (для интерполяции цвета требуется регистрация) - это не токен дизайна, а цель анимации на уровне элемента, так что это не противоречит предыдущему правилу.

Пример:

```css
:root {
	--accent-color: light-dark(blue, skyblue);
	--surface-color: light-dark(white, #222);
	--text-color: light-dark(#111, white);

	color-scheme: light dark;
	accent-color: var(--accent-color);
	color: var(--text-color);
}

body {
	/* Значение --surface-color динамически переключается, несмотря на то, что было унаследовано, поскольку переменная --surface-color не зарегистрирована */
	background: var(--surface-color);
}

pre, code {
	color-scheme: dark;
	background: var(--surface-color);

	/* Без этого accent-color будет blue, а не skyblue! */
	accent-color: var(--accent-color);

	/* Без этого text-color будет #111, а не white! */
	color: var(--text-color);
}
```

### Проблемы использования `color-scheme`

- Chrome и Firefox учитывают `color-scheme` в `iframe`: они рендерят встроенные страницы в правильной цветовой схеме и настраивают медиа-запрос `prefers-color-scheme` встроенной страницы так, чтобы он отражал `color-scheme` контекста внедрения. Safari этого не делает и разрешает `prefers-color-scheme` в соответствии с системными настройками, даже внутри `iframe`.

## Поддержка и страховка

### Поддержка и страховка `color-scheme`

Свойство `color-scheme` поддерживается всеми основными браузерами с 03.02.2022.

Свойство `color-scheme` - это прогрессивное улучшение. Браузеры без поддержки игнорируют его и используют дефолтный светлый UI.

Для учета предпочтений пользователя в старых браузерах, используйте медиа-запросы `prefers-color-scheme` для предоставления других цветов, когда выбран темный режим.

- Используйте медиа-запрос для переключения кастомных свойств на `:root` или `html`.
- Избегайте использования медиа-запроса на отдельных компонентах, пока компоненту не потребуется очень специфичный тип кастомизации темного режима (больше чем кастомизация цветов).

```css
:root {
  /* Определяем цвета бренда для каждого режима */
  --color-brand-light: #0056b3;
  --color-brand-dark: #00e5ff;
  --color-brand: var(--color-brand-light);

  /* Резерв для браузеров без поддержки `color-scheme` */
  @media (prefers-color-scheme: dark) {
    --color-brand: var(--color-brand-dark);
  }

  /* Игнорируется старыми браузерами */
  color-scheme: light dark;
}

button.primary {
	background-color: var(--color-brand);
}
```

### Поддержка и страховка `light-dark()`

`light-dark()` поддерживается всеми основными браузерами с 13.05.2024.

В браузерах, поддерживающих `color-scheme`, но не поддерживающих `light-dark()`, светлые и темные версии цветов должны сначала определяться как кастомные свойства, а для установки цветов в определенный режим должен использоваться медиа-запрос `prefers-color-scheme`:

```css
:root {
  /* Определяем акцентный цвет браузерного UI для каждого режима */
  --brand-accent-light: #0056b3;
  --brand-accent-dark: #00e5ff;
  --accent-color: var(--brand-accent-light);

  /* Резерв для браузеров без поддержки */
  @media (prefers-color-scheme: dark) {
    --accent-color: var(--brand-accent-dark);
  }

  /* Опционально: используем light-dark() для большего контроля цветов встроенного UI */
  @supports (color: light-dark(white, black)) {
    --accent-color: light-dark(var(--brand-accent-light), var(--brand-accent-dark));
  }

  /* Автоматически адаптируем нативный UI к системным предпочтениям пользователя */
  color-scheme: light dark;

  /* Пример унаследованного свойства цвета */
  accent-color: var(--accent-color);
}

pre, code {
  color-scheme: dark;

  /* Любое унаследованное значение свойства цвета должно устанавливаться повторно, даже если устанавливается тот же дизайн-токен */
  accent-color: var(--accent-color);
}
```
