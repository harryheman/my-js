# Гибкое масштабирование

## Обзор

Гибкое масштабирование (fluid scaling) позволяет компонентам корректировать свои внутренние пропорции (например, размеры шрифта и интервалы) в зависимости от их текущих размеров. Это создает более связный дизайн, чем переход между фиксированными контрольными точками (breakpoints).

Хотя гибкое масштабирование исторически достигалось за счет использования единиц области просмотра (масштабирование на основе размера экрана), современные единицы запросов к контейнеру (container queries) позволяют компонентам масштабироваться относительно их родительского контейнера. Это обеспечивает хороший внешний вид компонентов, независимо от их положения в макете, обеспечивая лучшую изоляцию и повторное использование компонентов.

## Реализация

### 1. Определение контейнера

Для использования единиц запросов к контейнеру, сначала нужно определить контекст сдерживания (containment context) на родительском элементе:

```css
.component-wrapper {
  /* Определяем тип контейнера. Используем 'inline-size' для масштабирования на основе ширины. */
  /* Также можно использовать 'size' для ширины и высоты, но это требует явного определения размеров */
  container-type: inline-size;

  /* Опционально: именуем контейнер для специфичных сценариев */
  container-name: fluid-card;
}
```

### 2. Используем единицы запросов к контейнеру

Используем единицы запросов к контейнеру (`cqi`, `cqb` и др.) для установки размеров относительно размеров контейнера.

*   `cqi`: 1% строчного (inline) размера контейнера (ширина в режиме горизонтального письма).
*   `cqb`: 1% блочного (block) размера контейнера (высота в режиме горизонтального письма).

Единицы контейнера могут использоваться напрямую в любом свойстве без правила `@container`. Они автоматически разрешаются на основе ближайшего предка с определенным `container-type`.

```css
.component-title {
  /* Увеличиваем размер шрифта на основе ширины контейнера. */
  /* 10cqi означает 10% ширины контейнера */
  font-size: 10cqi;
}

.component-body {
  /* Увеличиваем отступ на основе ширины контейнера */
  padding: 5cqi;
}
```

### 3. Ограничение значений с помощью `clamp()`

Для предотвращения того, что размеры станут слишком маленькими или слишком большими, используйте функцию CSS `clamp()`. Это влияет на возможность пользователя масштабировать или регулировать базовый размер шрифта. Чтобы текст соответствовал правилам доступности, максимальный размер не должен превышать минимальный более чем в 2,5 раза.

```css
.component-title {
  /* Ограничиваем размер шрифта между 1 и 2.5rem с масштабированием в 5% ширины контейнера */
  font-size: clamp(1rem, 5cqi, 2.5rem);
}
```

### Поддержка и страховка

Запросы к контейнеру поддерживаются всеми основными браузерами с 14.02.2023.

В качестве резервного варианта можно использовать единицы области просмотра и стандартные медиа-запросы:

```css
.component-title {
  /* Резерв для старых браузеров */
  font-size: clamp(1rem, 5vw, 2.5rem);
}

@supports (font-size: 1cqi) {
  .component-title {
    /* Используем запросы к контейнеру при наличии поддержки */
    font-size: clamp(1rem, 5cqi, 2.5rem);
  }
}
```

Текст по-прежнему масштабируется, но не на основе ширины компонента, а на основе ширины экрана.
