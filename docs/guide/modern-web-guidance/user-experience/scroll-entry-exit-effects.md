# Добавление эффектов входа и выхода элементов при прокрутке

Эффекты входа и выхода - это анимация, которая запускается, когда элемент появляется в или покидает область просмотра. Это может использоваться для создания вовлекающего и динамичного пользовательского опыта. Например, эффект входа может использоваться для появления (fade in) элемента при его прокрутке в область просмотра, а эффект выхода - для его уменьшения (scale down) при прокрутке из области просмотра.

## Реализация

Для добавления эффектов входа и выхода элемента нужно использовать сочетание нескольких свойств CSS. Вот пошаговое руководство:

1. Создаем отдельные `@keyframes` для анимации входа и анимации выхода. Анимация входа будет применяться при появлении элемента в области просмотра, а анимация выхода - при покидании элементом области просмотра:

```css
@keyframes slide-in {
  from { transform: translateX(-100%); }
}
@keyframes slide-out {
  to { transform: translateX(100%); }
}
```

2. Привязываем ключевые кадры входа и выхода к элементу. Это можно сделать путем определения нескольких анимаций в свойстве `animation`.

    - Укажите `backwards` в `animation-fill-mode` анимации входа, чтобы она находилась в начальном состоянии перед запуском.
    - Укажите `forwards` в `animation-fill-mode` анимации выхода, чтобы она поддерживала конечное состояния после завершения.

```css
.animated-element {
  animation:
    slide-in 1s linear backwards,
    slide-out 1s linear forwards;
}
```

3. Создайте временную шкалу отображения (View Timeline) и привяжите ее к анимации. View Timeline - это тип временной шкалы, который привязывается к видимости элемента в области просмотра. Ее можно создать с помощью функции `view()` и затем применить к анимации с помощью свойства `animation-timeline`:

```css
.animated-element {
  animation-timeline: view();
}
```

По умолчанию, `view()` отслеживает элемент по оси `block` (вертикальная ось). Для отслеживания оси `inline` (горизонтальная ось) следует использовать `view(inline)`.

4. Огранивайте анимацию диапазонами `entry` и `exit`. Свойство `animation-range` позволяет определять, на каких частях временной шкалы должна запускаться анимация.

    - Диапазон `entry` охватывает время с начала входа элемента в область просмотра до его полной видимости.
    - Диапазон `exit` охватывает время с начала выхода элемента из области просмотра до его полного скрытия.

```css
.animated-element {
  animation-range: entry, exit;
}
```

## Пример кода

Этот код анимирует прямых потомков прокручиваемого контейнера при прокрутке с помощью анонимной временной шкалы отображения:

```css
@media (prefers-reduced-motion: no-preference) {
  @supports ((animation-timeline: view()) and (animation-range: entry)) {
    @keyframes grow {
      from {
        scale: 0.5;
      }
    }
    @keyframes shrink {
      to {
        scale: 0.5;
      }
    }

    .scroller > * {
      animation:
        grow auto linear backwards,
        shrink auto linear forwards;
      animation-timeline: view(inline);
      animation-range: entry, exit;
    }
  }
}
```

Когда элемент входит в область прокрутки (scrollport), запускает анимация `grow`, а когда он выходит из области просмотра, запускается анимация `shrink`.

Аналогичный пример, реализованный с помощью именованной временной шкалы отображения:

```css
@media (prefers-reduced-motion: no-preference) {
  @supports ((animation-timeline: view()) and (animation-range: entry)) {
    @keyframes grow {
      from {
        scale: 0.5;
      }
    }
    @keyframes shrink {
      to {
        scale: 0.5;
      }
    }

    .scroller > * {
      view-timeline: --tl inline;
      animation:
        grow auto linear backwards,
        shrink auto linear forwards;
      animation-timeline: --tl;
      animation-range: entry, exit;
    }
  }
}
```

## Лучшие практики

При использовании анимации, управляемой прокруткой (scroll-driven animations), важно следовать нескольким лучшим практикам для обеспечения плавного и доступного опыта:

- Используйте определение поддержки: не все браузеры поддерживают анимацию, управляемую прокруткой. Используйте `@supports ((animation-timeline: view()) and (animation-range: entry))` для определения поддержки и предоставляйте резерв для браузеров без поддержки.
  - Проверка `(animation-range: entry)` предназначена для исключения браузеров с частичной поддержкой.
  - Не используйте пакет `scroll-timeline-polyfill` в качестве резерва, поскольку в нем реализован не весь функционал, и он содержит много известных проблем.
  - Если анимация является чисто декоративной, резерв можно не предоставлять.
- Учитывайте предпочтения пользователей: некоторые пользователи предпочитают минимум движений в вебе. Используйте медиа-запрос `prefers-reduced-motion` для отключения или уменьшения количества анимаций для таких пользователей.
- Старайтесь анимировать только "производительные" свойства CSS: для плавной анимации старайтесь анимировать только те свойства, которые могут обрабатываться потоком компоновщика браузера, такие как `transform` и `opacity`. Анимация таких свойств, как `width` или `height` может привести к проблемам с производительностью.
- Используйте правильный порядок объявлений: объявляйте `animation-timeline` и `animation-range` после стандартного сокращения `animation` во избежание сброса временной шкалы сокращением.

При использовании функции `view()` для создания анимации, управляемой прокруткой:

- Опционально явно указывайте ось для отслеживания: по умолчанию отслеживается ось `block` (вертикальная), но можно всегда явно указывать ось для отслеживания с помощью `view(block)` или `view(inline)`.
- Когда анимация не применяется к самому отслеживаемому элементу, используйте именованную временную шкалу отображения.

При использовании свойства `view-timeline` для создания анимации, управляемой прокруткой:

- Используйте `<dashed-ident>` для названия (например, `view-timeline: --my-custom-name`).
- Опционально явно указывайте ось для отслеживания: по умолчанию отслеживается ось `block` (вертикальная), но можно всегда явно указывать ось для отслеживания с помощью `view-timeline-axis`.
- Убедитесь, что область поиска работает: когда элемент с `view-timeline` не является предком плоского дерева анимируемого элемента, повысьте видимость именованной `view-timeline` с помощью `timeline-scope` на общем предке.

Используйте именованную `view-timeline`, когда нужно анимировать несколько элементов или потомков отслеживаемого элемента.

## Поддержка и страховка

Анимация, управляемая прокруткой, имеет ограниченную поддержку браузеров. Поддерживается в Chrome 115+, Edge 115+ (с июля 2023 года) и Safari 26+ (с сентября 2025 года). Не поддерживается в Firefox.

Для браузеров без поддержки можно использовать резерв для воссоздания визуальных эффектов. Резерв, как правило, создается с помощью обработчика прокрутки (для эффектов ScrollTimeline) или IntersectionObserver API (для эффектов ViewTimeline).

В браузерах с поддержкой всегда используйте нативную реализацию CSS, поскольку она гораздо более производительна.

Обратите внимание, что не все эффекты можно воссоздать с помощью резерва.
