# Темный режим

Свойство `color-scheme` определяет, какие цветовые схемы (такие как светлая или темная) поддерживает страница. Это сообщает браузеру, что он может автоматически стилизовать нативные элементы UI, такие как полосы прокрутки, контролы формы и дефолтный фон холста, для обеспечения соответствия с дизайном сайта и помочь минимизировать белые вспышки во время начальной загрузки.

## Реализация

### 1. Определение поддерживаемых схем в HTML

Во избежание "вспышки нестилизованного контента" (FOUC), поместите тег `<meta>` в `<head>` для того, чтобы браузер понимал, какие темы поддерживаются, перед началом рендеринга. Хотя этот тег помогает избежать FOUC путем ранней установки начального цвета холста (canvas), он может не полностью устранить вспышки во всех браузерах или условиях загрузки.

```html
<!-- Определяем поддержку для светлой и темной темы -->
<meta name="color-scheme" content="light dark">
```

### 2. Применение цветовой схемы к :root или html

Примените свойство `color-scheme` к элементу `html` или псевдоклассу `:root`. Браузер специально "смотрит" на корневой элемент для определения темы для всей области просмотра, включая полосы прокрутки и начальный фон холста. При применении к `body`, эти глобальные компоненты UI могут остаться в светлом режиме, поскольку `body` не управляет контекстом рендеринга окна.

```css
/* Применяем `color-scheme` к :root или html для тематизации всей области просмотра */
:root {
  /* Автоматически адаптируем нативный UI к системным предпочтениям пользователя */
  color-scheme: light dark;
}
```

### 3. Определение светлых и темных цветовых токенов

Для определения токенов цвета, которые автоматически адаптируются к разным значениям `color-scheme`, можно использовать функцию `light-dark()`.

Рекомендуется также определять сырые (raw) значения цвета в отдельных кастомных свойствах, что облегчает их комбинирование разными способами (и, при необходимости, облегчает реализацию резервного варианта).

Для большего контроля цветов встроенного UI, таких как `accent-color` или `scrollbar-color`, авторы могут опционально добавлять собственные динамические цвета с помощью кастомных свойств и/или функции `light-dark()`. Эта функция автоматически берет правильный цвет на основе вычисленной `color-scheme` элемента и снижает необходимость в лишних медиа-запросах, но не требуется для базовой реализации.

```css
:root {
  --color-brand-light: oklch(45% 0.23 270);
  --color-brand-dark: oklch(85% 0.15 210);
  --color-brand-text-light: white;
  --color-brand-text-dark: oklch(40% 0.23 270);

  --color-brand: light-dark(var(--color-brand-light), var(--color-brand-dark));
  --color-brand-text: light-dark(var(--color-brand-text-light), var(--color-brand-text-dark));

  /* Автоматически адаптируем нативный UI к системным предпочтениям пользователя */
  color-scheme: light dark;
}

button.primary {
  /* Это автоматически адаптируется к цветовой схеме */
  background-color: var(--color-brand);
  color: var(--color-brand-text);
}
```

Опционально: доступно некоторое количество системных цветов, которые также автоматически адаптируются к цветовой схеме пользователя (и другим цветовым режимам, например, принудительным (forced) цветам), такие как `canvas`, `canvastext`, `accentcolor` (проверяйте поддержку) , `buttonborder` и т.д.  Обычно, они слишком ограничены для того, чтобы быть полезными, кроме очень специфичных случаев, когда требуется точное совпадение с дефолтным браузерным UI или резервные/дефолтные варианты.

#### Опционально: адаптация цветовых пар к контексту

Даже при перезаписи системных значений по умолчанию, может быть полезным использовать медиа-запрос `prefers-color-scheme` для определения других пар цветов, которые учитывают цвета браузера и ОС на странице (или внешней страницы, когда страница рендерится в `iframe`).

Например, используйте немного более тусклую светлую тему при системной настройке `dark` или более контрастную темную тему при системной настройке `light`, чтобы страница не была визуально перегружена окружающим UI.

## Детальная кастомизация браузерного UI

Установка `color-scheme` уже адаптирует браузерный UI к используемой цветовой схеме, но использует дефолтные цвета ОС (системные цвета), которые могут не соответствовать дизайну сайта. Современные браузеры предоставляют несколько хуков (hooks) для детальной кастомизации. Не спешите воссоздавать нативные контролы, не исчерпав возможности этих хуков.

### Установка акцентного цвета

Некоторые элементы браузерного UI (например, выбранные чекбоксы или слайдеры) используют акцентный (accent) цвет. Этот цвет устанавливается ОС, но свойство `accent-color` позволяет установить цвет, который лучше соответствует странице, такой как цвет бренда.

```css
html {
  accent-color: light-dark(var(--color-accent-light), var(--color-accent-dark));
}
```

### Проблемы использования `accent-color`

- При помещении визуальных элементов над акцентным цветом (например, галочка чекбокса), Chrome и Safari автоматически выбирают контрастный цвет, но Safari также модифицирует акцентный цвет и может не поддерживать адекватный контраст.

### Цвета полосы прокрутки

Используйте `scrollbar-color` в сочетании с `light-dark()` для установки кастомных цветов, адаптирующихся к используемой цветовой схеме:

```css
:root {
  --color-scrollbar-track: light-dark(#eee, #222);
  --color-scrollbar-thumb: light-dark(#999, #666);
  scrollbar-color: var(--color-scrollbar-thumb) var(--color-scrollbar-track);
}
```

### Проблемы использования `scrollbar-color`

- Не анимируйте `scrollbar-color`. [Баг WebKit](https://bugs.webkit.org/show_bug.cgi?id=311752) вызывает мерцание скроллбара при каждом изменении `scrollbar-color`.
- На macOS свойства `scrollbar-color` (стандарт) и `::-webkit-scrollbar` (легаси) по умолчанию игнорируются, поскольку там используются нативные "накладываемые" (overlay) скроллбары. Кастомные цвета должны использоваться совместно с `scrollbar-width` (например, `thin` или `auto`) для их принудительного рендеринга.
- Даже с примененным `scrollbar-width`, macOS рендерит дорожку прозрачной по умолчанию. Если дизайн требует видимый фоновый цвет дорожки на macOS, необходимо применить `scrollbar-gutter: stable;` к прокручиваемому контейнеру, но обратите внимание, что он появляется только при наведении пользователем курсора на скроллбар.
- Даже с `scrollbar-gutter: stable` дорожка может оставаться прозрачной на macOS. Поэтому видимость ползунка не должна зависеть от цвета дорожки.

### Дальнейшая кастомизация

Большинство браузеров предоставляет следующие псевдоэлементы для кастомизации нативного UI:

- `::placeholder`
- `::spelling-error`
- `::grammar-error`
- `::selection`
- `::search-text`
- `::target-text`
- `::file-selector-button`

В любом из них можно использовать цвета `light-dark()` для адаптации к используемой цветовой схеме.

## Опционально: реализация переключателя цветовой схемы

Не устанавливайте `color-scheme: light` или `color-scheme: dark` на корневом элементе по умолчанию. Дефолтная цветовая схема должна определяться настройкой системы пользователя, что происходит автоматически при установке `color-scheme` в значение `light dark`.

Для специфичной для сайта кастомизации можно предоставить пользователю переключатель, позволяющий выбирать между светлым, темным и системным режимами.

Переключатель должен:

- Обновлять элемент `<meta name="color-scheme">` для отражения выбранной темы (`light dark` для системной, `light` для светлой и `dark` для темной).
- Если тема меняет не только цвета, установите класс на `<html>` для соответствия теме и используйте селекторы потомков. Хотя селектор `:root:has(> head > meta[name="color-scheme"][content="dark"])` работает, он медленнее и не дает никаких выгод, поскольку мы уже используем JS для обновления элемента `<meta>`.
- Сохранять выбор пользователя в `localStorage`.
- Важно: дефолтным значением темы должна быть системная настройка, с перезаписями (overrides) для пользовательских схем. Таким образом, если JS не будет выполнен, сайт будет использовать системную цветовую схему.
- Системная тема может измениться в любое время. При использовании JS для чтения `matchMedia("(prefers-color-scheme: dark)").matches`, следует также использовать `addEventListener("change", fn)` для обработки изменений. CSS автоматически адаптируется к изменениям.
- Важно: во избежание вспышки нестилизованного контента (FOUC) для пользователей, ранее выбравших схему, отличающуюся от системной, используйте встроенный скрипт (без `type=module` и `defer`) для установки схемы при загрузке страницы:

```html
<meta name="color-scheme" content="light dark">
<script>
{
  const colorScheme = localStorage.getItem("color-scheme");
  if (colorScheme) {
    document.querySelector('meta[name="color-scheme"]').content = colorScheme;
  }
}
</script>
```

## Перезаписи на уровне компонента

Глобальная тема может быть перезаписана для определенного элемента путем установки `color-scheme` на нем. Это полезно для всегда темных разделов на странице, таких как блоки кода или медиа-проигрыватели.

```css
pre, code {
  /* Заставляет элемент и его потомков использовать темную тему */
  color-scheme: dark;
}
```

## Известные проблемы

### Проблемы использования color-scheme

- Chrome и Firefox учитывают `color-scheme` в `iframe`: они рендерят встроенные страницы в правильной цветовой схеме и настраивают медиа-запрос `prefers-color-scheme` встроенной страницы так, чтобы он отражал `color-scheme` контекста внедрения. Safari этого не делает и разрешает `prefers-color-scheme` в соответствии с системными настройками, даже внутри `iframe`.

## Поддержка и страховка

### Поддержка и страховка `color-scheme`

Свойство `color-scheme` поддерживается всеми основными браузерами с 03.02.2022.

Свойство `color-scheme` - это прогрессивное улучшение. Браузеры без поддержки игнорируют его и используют дефолтный светлый UI.

Для учета предпочтений пользователя в старых браузерах, используйте медиа-запросы `prefers-color-scheme` для предоставления других цветов, когда выбран темный режим.

- Используйте медиа-запрос для переключения кастомных свойств на `:root` или `html`.
- Избегайте использования медиа-запроса на отдельных компонентах, пока компоненту не потребуется очень специфичный тип кастомизации темного режима (больше чем кастомизация цветов).

```css
:root {
  /* Определяем цвета бренда для каждого режима */
  --color-brand-light: #0056b3;
  --color-brand-dark: #00e5ff;
  --color-brand: var(--color-brand-light);

  /* Резерв для браузеров без поддержки `color-scheme` */
  @media (prefers-color-scheme: dark) {
    --color-brand: var(--color-brand-dark);
  }

  /* Игнорируется старыми браузерами */
  color-scheme: light dark;
}

button.primary {
	background-color: var(--color-brand);
}
```

### Поддержка и страховка `light-dark()`

`light-dark()` поддерживается всеми основными браузерами с 13.05.2024.

В браузерах, поддерживающих `color-scheme`, но не поддерживающих `light-dark()`, светлые и темные версии цветов должны сначала определяться как кастомные свойства, а для установки цветов в определенный режим должен использоваться медиа-запрос `prefers-color-scheme`:

```css
:root {
  /* Определяем акцентный цвет браузерного UI для каждого режима */
  --brand-accent-light: #0056b3;
  --brand-accent-dark: #00e5ff;
  --accent-color: var(--brand-accent-light);

  /* Резерв для браузеров без поддержки */
  @media (prefers-color-scheme: dark) {
    --accent-color: var(--brand-accent-dark);
  }

  /* Опционально: используем light-dark() для большего контроля цветов встроенного UI */
  @supports (color: light-dark(white, black)) {
    --accent-color: light-dark(var(--brand-accent-light), var(--brand-accent-dark));
  }

  /* Автоматически адаптируем нативный UI к системным предпочтениям пользователя */
  color-scheme: light dark;

  /* Пример унаследованного свойства цвета */
  accent-color: var(--accent-color);
}

pre, code {
  color-scheme: dark;

  /* Любое унаследованное значение свойства цвета должно устанавливаться повторно, даже если устанавливается тот же дизайн-токен */
  accent-color: var(--accent-color);
}
```

### Поддержка и страховка `scrollbar-color`

`scrollbar-color` поддерживается всеми основными браузерами c 12.12.2025.

Это свойство является прогрессивным улучшением и не всегда требует страховки. В качестве таковой следует использовать нестандартные псевдоэлементы `::-webkit-scrollbar`.

Оборачивайте страховку в блок `@supports not (scrollbar-color: auto)` для предотвращения конфликтов между стандартными свойствами и устаревшими селекторами WebKit в современных браузерах.

Пользовательские свойства цвета автоматически передаются устаревшим селекторам WebKit. Дублировать их не нужно.

```css
@supports not (scrollbar-color: auto) {
  .scroller::-webkit-scrollbar {
    /* Необходимо определить базовый размер, чтобы кастомные цвета были видимыми */
    width: 12px;
    height: 12px;
  }

  .scroller::-webkit-scrollbar-thumb {
    background: var(--scrollbar-thumb);
  }

  .scroller::-webkit-scrollbar-track {
    background: var(--scrollbar-track);
  }
}
```

### Поддержка и страховка `accent-color`

Свойство `accent-color` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 93+, Edge 93+ (с сентября 2021 года) и Firefox 92+ (с сентября 2021 года). Не поддерживается в Safari.

Свойство `accent-color` - это прогрессивное улучшение. Браузеры без поддержки игнорируют его и используют свои дефолтные цвета UI.
