# Индикатор прогресса прокрутки

Индикатор прогресса прокрутки - это популярный шаблон пользовательского интерфейса, который визуально отражает прогресс пользователя в прокручиваемом документе или контейнере. По мере прокрутки пользователем, визуальный элемент обновляется для отражения количества просмотренного и оставшегося контента.

## Реализация

Для создания индикатора прогресса прокрутки нужны 2 вещи:

1. Элемент-индикатор прогресса. Этот элемент, как правило, содержит `position: fixed` или `position: absolute`, поэтому остается видимым во время прокрутки.
2. Анимация, привязанная к положению прокрутки.

Вот как можно этого достичь:

- Сначала создаем элемент, который будет выступать в роли индикатора прогресса.
- Затем в CSS определяем анимацию `@keyframes`, которая масштабирует индикатор прогресса. Распространенным подходом является масштабирование элемента от `scaleX(0)` к `scaleX(1)`.
- Наконец, применяем эту анимацию к индикатору прогресса и привязываем его `animation-timeline` (временная шкала анимации) к `scroll-timeline` (временная шкала прокрутки). Это указывает браузеру устанавливать прогресс анимации на основе положения прокрутки ближайшего прокручиваемого предка.

## Пример кода

Этот код увеличивает элемент `#progress` при прокрутке с помощью анонимной временной шкалы прокрутки, созданной с помощью функции `scroll()`:

```css
@media (prefers-reduced-motion: no-preference) {
  @supports ((animation-timeline: scroll())) {
    @keyframes grow-progress {
      from { transform: scaleX(0); }
      to { transform: scaleX(1); }
    }

    #progress {
      position: fixed;
      left: 0; top: 0;
      width: 100%; height: 1em;
      background: red;

      transform-origin: 0 50%;
      animation: grow-progress auto linear;
      animation-timeline: scroll();
    }
  }
}
```

Из-за своего положения в DOM, функция `scroll()` отслеживает ближайшего прокручиваемого предка в направлении `block` - корневой элемент:

```html
<body>
  <!-- Чисто декоративные визуальные индикаторы прогресса должны содержать `aria-hidden="true"` для удаления пустого элемента из дерева доступности -->
  <div id="progress" aria-hidden="true"></div>
</body>
```

Этот код увеличивает элемент `#progress` при прокрутке с помощью именованной временной шкалы прокрутки, созданной свойством `scroll-timeline`:

```css
@media (prefers-reduced-motion: no-preference) {
  @supports ((animation-timeline: scroll())) {
    @keyframes grow-progress {
      from { transform: scaleX(0); }
      to { transform: scaleX(1); }
    }

    :root {
      scroll-timeline: --tl block;
    }

    #progress {
      position: fixed;
      left: 0; top: 0;
      width: 100%; height: 1em;
      background: red;

      transform-origin: 0 50%;
      animation: grow-progress auto linear;
      animation-timeline: --tl;
    }
  }
}
```

## Лучшие практики

При использовании анимации, управляемой прокруткой, важно следовать нескольким лучшим практикам для обеспечения плавного и доступного (accessible) опыта:

- Используйте определение поддержки: не все браузеры поддерживают анимацию, управляемую прокруткой. Используйте `@supports ((animation-timeline: view()) and (animation-range: entry))` для определения поддержки и предоставляйте резерв для браузеров без поддержки.
  - Проверка `(animation-range: entry)` предназначена для исключения браузеров с частичной поддержкой.
  - Не используйте пакет `scroll-timeline-polyfill` в качестве резерва, поскольку в нем реализован не весь функционал, и он содержит много известных проблем.
  - Если анимация является чисто декоративной, резерв можно не предоставлять.
- Учитывайте предпочтения пользователей: некоторые пользователи предпочитают минимум движений в вебе. Используйте медиа-запрос `prefers-reduced-motion` для отключения или уменьшения количества анимаций для таких пользователей.
- Старайтесь анимировать только "производительные" свойства CSS: для плавной анимации старайтесь анимировать только те свойства, которые могут обрабатываться потоком компоновщика браузера, такие как `transform` и `opacity`. Анимация таких свойств, как `width` или `height` может привести к проблемам с производительностью.
- Используйте правильный порядок объявлений: объявляйте `animation-timeline` и `animation-range` после стандартного сокращения `animation` во избежание сброса временной шкалы сокращением.

При использовании функции `scroll()` для создания анимации, управляемой прокруткой:

- Опционально: явно указывайте прокручиваемого предка для отслеживания с помощью `scroll(root)` или `scroll(self)`.
  - Если `root`, `nearest` или `self` не подходят, используйте именованное `scroll-timeline`.
- Опционально: явно определяйте ось для отслеживания с помощью `scroll(block)` или `scroll(inline)`.

При использовании свойства `scroll-timeline` для создания анимации, управляемой прокруткой:

- Используйте `<dashed-ident>` для названия.
- Опционально: явно определяйте ось для отслеживания с помощью `scroll-timeline-axis`.
- Убедитесь, что область поиска (scope of the lookup) работает: если элемент, объявляющий `scroll-timeline`, не является предком плоского дерева анимированного элемента, увеличьте видимость названия `scroll-timeline`, используя `timeline-scope` на общем предке.

## Поддержка и страховка

Анимация, управляемая прокруткой, имеет ограниченную поддержку браузеров. Поддерживается в Chrome 115+, Edge 115+ (с июля 2023 года) и Safari 26+ (с сентября 2025 года). Не поддерживается в Firefox.

Для браузеров без поддержки можно использовать резерв для воссоздания визуальных эффектов. Резерв, как правило, создается с помощью обработчика прокрутки (для эффектов ScrollTimeline) или IntersectionObserver API (для эффектов ViewTimeline).

В браузерах с поддержкой всегда используйте нативную реализацию CSS, поскольку она гораздо более производительна.

Обратите внимание, что не все эффекты можно воссоздать с помощью резерва.

Следующий скрипт применяет резерв для браузеров, которые не поддерживают анимацию, управляемую прокруткой. В нем используется обработчик прокрутки для отслеживания положения прокрутки корневого элемента и соответствующим образом обновляется свойство `transform` индикатора прогресса:

```html
<script>
  if (!CSS.supports('animation-timeline', 'scroll()')) {
    const progress = document.querySelector('#progress');

    window.addEventListener('scroll', () => {
      const scrollable = document.documentElement.scrollHeight - window.innerHeight;
      const scrolled = window.scrollY;
      const progressPercentage = (scrolled / scrollable);

      progress.style.transform = `scaleX(${progressPercentage})`;
    });
  }
</script>
```
