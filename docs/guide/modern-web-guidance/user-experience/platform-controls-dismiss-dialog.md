# Нативное закрытие модального диалога

Когда модальный диалог открыт, пользователь ожидает использования привычных контролов для его закрытия: нажатие клавиши `Esc` на клавиатуре, использование кнопки "Назад" или соответствующего жеста на мобильных платформах или во вспомогательных технологиях.

Когда элемент `<dialog>` был впервые представлен, он мог закрываться/отменяться (dismiss) с помощью клавиши `Esc`, но не с помощью других контролов платформы, таких как кнопка/жест "Назад". Атрибут `closedby` расширяет реакцию элементов `<dialog>`, открытых в виде модальных окон (например, с помощью метода JavaScript `showModal()` или декларативно с помощью команды вызывающего (invoker) `show-modal`), на контролы платформы для запросов закрытия (close requests). Поэтому разработчикам не нужно вносить какие-то специфические изменения, если они уже используют элемент `<dialog>`.

```html
<!-- Диалог должен быть открыт либо с помощью метода `showModal()`, либо с помощью команды `show-modal` для реакции на запросы закрытия, включая платформенные контролы -->
<dialog aria-labelledby="example">
  <h1 id="example">Пример</h1>
  <p>Модалка, которая может закрываться с помощью запросов закрытия.</p>
</dialog>
```

При открытии в виде модалки, диалог без атрибута `closedby`  отвечает на запросы закрытия также, как в случае явной установки атрибута `closedby` в значение `closerequest`:

```html
<!-- Это необязательно, поскольку это дефолтное поведение для модальных диалогов -->
<dialog closedby="closerequest" aria-labelledby="example">
  <h1 id="example">Пример</h1>
  <p>Модалка, которая может закрываться с помощью запросов закрытия.</p>
</dialog>
```

Если требуется "легкая отмена" (закрытие при клике за пределами модалки), нужно установить атрибут `closedby` в значение `any`:

```html
<dialog closedby="any" aria-labelledby="example">
  <h1 id="example">Пример</h1>
  <p>Модалка, которая может закрываться с помощью запросов закрытия и легкой отмены.</p>
</dialog>
```

## Поддержка и страховка

`<dialog closedby>` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 134+, Edge 134+ (с марта 2025 года) и Firefox 141+ (с июля 2025 года). Не поддерживается в Safari.

Элементы `<dialog>`, открытые в виде модалок, уже могут закрываться с помощью `Esc`, поэтому резерв не требуется. Не существует хорошего способа реализовать запросы закрытия для мобильных кнопок/жестов "Назад", поэтому проще рассматривать этот функционал как прогрессивное улучшение - особенно с учетом того, что существуют и другие, инклюзивные способы закрыть модалку. Аналогично, легкая отмена для элемента `<dialog>` с помощью `closedby="any"` может считаться прогрессивным улучшением.
