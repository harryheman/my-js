# Индивидуальные свойства преобразования

Свойство `transform` позволяет применять несколько преобразований в указанном порядке, но любое изменение отдельного преобразования требует повторного определения всей цепочки преобразований. Это усложняет анимацию или переход (transition) отдельного преобразования.

Индивидуальные свойства преобразования CSS (`translate`, `rotate` и `scale`) позволяют применять преобразования независимо от свойства `transform`. Это облегчает перезапись отдельного преобразования, например, при `:hover`.

## Реализация

Индивидуальные свойства преобразования всегда применяются в фиксированном порядке, независимо от порядка их определения в CSS:

1. `translate`
2. `rotate`
3. `scale`
4. `transform` (применяется последним)

Если требуется другой порядок (например, масштабирование перед вращением), придется довольствоваться `transform`.

`transform` не перезаписывает индивидуальные свойства преобразования. Другими словами, `scale: 2; transform: scale(3);` приведет сначала к двукратному масштабированию, потом еще к трехкратному, т.е. элемент будет масштабирован в 6 раз.

## Предотвращение неожиданных изменений в стековых контекстах и содержащих блоках

Свойство `transform` и индивидуальные свойства преобразования влияют на компоновку и рендеринг страницы и могут приводить к неожиданному поведению с `z-index` или якорным позиционированием (anchor positioning). Если к элементу может быть применено преобразование как часть изменения состояния, например, `:hover`, перехода или анимации, примените аналогичное преобразование к базовому элементу. Это гарантирует, что контекст стека и вложенность (containment) элемента не изменятся при преобразовании:

```css
.element{
  /* Применяем аналогичные преобразования для свойств, которые
     изменятся при изменении состояния (например, при `:hover`). Это предотвращает неожиданную компоновку или смещения `z-index`,
     вызванные созданием нового контекста стека только при наведении курсора */
  translate: 0px;
  rotate: 0deg;
  scale: 1;
}
.element:hover{
  translate: 10px 10px;
  rotate: 20deg;
  scale: 0.8;
}
```

## Независимая анимация и преобразования

Основным преимуществом является возможность без конфликтов определять перекрывающиеся анимации или переходы, нацеленные на разные свойства:

```css
.card {
  /* Определяем независимые анимации, которая не перезаписывают друг друга */
  animation: float 3s infinite ease-in-out;

  /* Выполняем переход только `scale` при наведении курсора */
  transition: scale 0.3s ease;

  /* Устанавливаем базовый масштаб, чтобы предотвратить внезапное смещение стекового контекста в процессе перехода при наведении курсора */
  scale: 1;
}

.card:hover {
  /* Меняется только масштаб; анимация "float" (смещение) продолжается без прерываний */
  scale: 1.05;
}

@keyframes float {
  0%, 100% { translate: 0 0; }
  50% { translate: 0 -10px; }
}
```

## Поддержка и страховка

Индивидуальные свойства преобразования поддерживаются всеми основными браузерами с 05.08.2022.

В старых браузерах используйте стандартное свойство `transform`:

```css
.element {
  /* Базовое преобразование */
  transform: translate(100px, 0) rotate(45deg);
  /* Базовое индивидуальное свойство преобразования */
  scale: 1;
}

@supports not (translate: 0px) {
  .element:hover {
    /* Резерв: повторяем `translate` и `rotate`, несмотря на то, что изменилось только `scale` */
    transform: translate(100px, 0) rotate(45deg) scale(1.1);
  }
}

@supports (translate: 0px) {
  .element:hover {
    /* Определяем только изменение */
    scale: 1.1;
  }
}
```
