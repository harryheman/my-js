# Анимация внутренних размеров

Анимация динамических размеров элементов, таких как `block-size: auto` или `inline-size: max-content`, исторически требовала JavaScript или хрупких хаков с `max-height`. Свойство `interpolate-size` и функция `calc-size()` позволяют браузеру нативно выполнять интерполяцию между фиксированными размерами и ключевыми словами внутренних (intrinsic) размеров.

## Реализация

1. Включите интерполяцию по ключевым словам: примените `interpolate-size: allow-keywords` к родительскому элементу (как правило, `:root`) для включения переходов свойств размеров, в которых используются ключевые слова.
2. Определите переход: установите `transition` для свойства размера (например, `block-size`, `inline-size`) на целевом элементе.
3. Используйте ключевые слова внутренних размеров: измените свойство размера на поддерживаемое ключевое слово внутреннего размера - `auto`, `min-content`, `max-content`, `fit-content` или (для `flex-basis`) `content` - в процессе взаимодействия (например, `:hover` или класс состояния).
4. Произведите вычисления (опционально): используйте `calc-size()`, если нужно выполнить расчет внутреннего размера (например, `auto + 2rem`). `calc-size()` также поддерживает ключевое слово `any` для не зависящих от базиса вычислений.

## Пример: общий шаблон расширения

Этот шаблон можно применять к любому контейнеру (такому как раздел "Показать больше" или меню навигации) для перехода между ограниченной высотой и натуральным размером элемента:

```css
/* Включаем глобально для всех потомков */
:root {
  /* Переходы к ключевым словам размеров отключены по умолчанию для совместимости */
  interpolate-size: allow-keywords;
}

.expandable-container {
  /* 1. Определяем фиксированный начальный размер (или 0) и скрываем переполнение */
  block-size: 100px;
  overflow: hidden;

  /* 2. Включаем переход свойства размера */
  transition: block-size 0.4s ease-out;
}

.expandable-container.is-expanded {
  /* 3. Плавно анимируем к натуральной внутренней высоте */
  block-size: auto;
}
```

## Пример: вычисление внутреннего `inline-size`

```css
.badge {
  inline-size: 40px;
  overflow: hidden;
  white-space: nowrap;
  transition: inline-size 0.3s ease;
}

.badge:hover {
  /* calc-size(базис, вычисление) */
  /* 'size' представляет вычисленный базис (`max-content` в данном случае) */
  inline-size: calc-size(max-content, size + 20px);
}
```

## Пример: переход от внутреннего к фиксированному размеру

Также допускается анимация в противоположном направлении - от натурального размера к определенной высоте. Это полезно для "пропускаемых" компонентов.

```css
.collapsible-alert {
  /* 1. Начинаем с натуральной высоты контента */
  block-size: auto;
  overflow: hidden;
  transition: block-size 0.5s ease-in-out, opacity 0.5s ease;
}

.collapsible-alert.is-dismissed {
  /* 2. Плавно схлопываем к нулю */
  block-size: 0;
  opacity: 0;
  pointer-events: none;
}

/* Отключаем анимацию для чувствительных пользователей */
@media (prefers-reduced-motion: reduce) {
  .expandable-container,
  .badge,
  .collapsible-alert {
    transition: none !important;
  }
}
```

```javascript
// Синхронизация доступности: удаляем схлопнутые элементы из дерева доступности и синхронизируем состояния `aria-expanded` на триггерах
const alertElement = document.querySelector('.collapsible-alert');
alertElement.addEventListener('transitionend', (e) => {
  if (e.propertyName === 'block-size' && alertElement.classList.contains('is-dismissed')) {
    alertElement.hidden = true;
  }
});

// Пример синхронизации триггера
const triggerBtn = document.querySelector('.accordion-trigger');
triggerBtn?.addEventListener('click', () => {
  const isExpanded = triggerBtn.getAttribute('aria-expanded') === 'true';
  triggerBtn.setAttribute('aria-expanded', !isExpanded);
});
```

## Основные ограничения

* Ограничение "ключевое слово-ключевое слово": нельзя анимировать от одного ключевого слова к другому напрямую (например, от `min-content` к `max-content`). Одним концом перехода должен быть фиксированный размер или процент (например, от `0` к `auto`).
* Синтаксис `calc-size()`: внутри `calc-size()` нельзя смешивать разные ключевые слова размеров в одном выражении. Первый аргумент (базис) определяет, что представляет `size`.
* Дефолтное значение: переходы к ключевым словам размеров отключены по умолчанию (`numeric-only`) для обеспечения обратной совместимости. `interpolate-size: allow-keywords` следует применять явно к элементу или его предку. `calc-size()` действует как переопределение для каждого свойства, автоматически включая интерполяцию при использовании.

## Поддержка и страховка

`interpolate-size` и `calc-size()` имеют ограниченную поддержку браузеров. Поддерживаются в Chrome 129+ и Edge 129+ (с сентября 2024 года). Не поддерживаются в Firefox и Safari.

`interpolate-size` и `calc-size()` являются прогрессивными улучшениями. Браузеры без поддержки выполняют мгновенный прыжок к целевому размеру.

* Мягкая деградация: для простых переходов `block-size: auto`, браузер просто переключает размер мгновенно, что функционально, но не так приятно.
* Резерв для ключевого слова: при использовании `calc-size()`, всегда предоставляйте резерв ключевого слова для старых браузеров, поскольку они отклоняют все объявление `calc-size()`.

```css
.card {
  block-size: auto; /* Резерв для старых браузеров */
  block-size: calc-size(auto, size); /* Современные браузеры будут использовать это объявление */
  transition: block-size 0.3s ease;
}
```
