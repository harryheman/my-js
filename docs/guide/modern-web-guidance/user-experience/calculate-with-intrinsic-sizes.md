# Вычисления на основе внутренних размеров

`calc-size()` - это функция CSS для выполнения математических операций с ключевыми словами внутренних (intrinsic) размеров, таких как `auto`, `min-content` и `fit-content`. Важно: `calc-size()` следует использовать, только когда нужно модифицировать внутренний размер элемента путем какого-либо вычисления или ограничения; для простой анимации на основе ключевых слов (например, от `0` к `auto`) следует использовать `interpolate-size: allow-keywords`.

## Реализация

1. Определите внутренний базис: определите, какое ключевое слово (`auto`, `min-content` и т.д.) должно быть базой вычисления.
2. Определите ограничения: используйте математические функции CSS, такие как `clamp()`, `min()` или `max()`, во втором аргументе для применения ограничений дизайна к внутреннему размеру.
3. Предоставьте резерв: всегда определяйте стандартное ключевое слово размера или фиксированный размер перед свойством, в котором используется `calc-size()`, для того, чтобы макет оставался функциональным в старых браузерах.
4. Применяйте логические свойства: используйте логические свойства, такие как `inline-size` или `block-size`, для учета направления письма.
5. Опциональное прогрессивное улучшение: оборачивайте сложную логику компоновки или анимации в блоки `@supports (inline-size: calc-size(auto, size + 0px))` для доставки продвинутого функционала только в браузеры, которые его поддерживают.

## Базовый синтаксис

```css
/* calc-size(<calc-size-basis>, <calc-sum>) - математические операции с ключевыми словами размеров */
.element {
  /* Всегда предоставляйте резерв для браузеров, которые не поддерживают `calc-size()` */
  inline-size: min-content;

  /* Используйте `calc-size()` для модификации внутреннего базиса с помощью вычисления или функции */
  inline-size: calc-size(min-content, size + 2rem);
}
```

### Базис (`<calc-size-basis>`)

Первый аргумент определяет "базовый" размер для вычисления.

Стандартные ключевые слова:

- `auto`: дефолтный размер элемента.
- `min-content`: минимальный размер элемента без переполнения.
- `max-content`: размер, необходимый для размещения всего контента элемента в одной строке.
- `fit-content`: эквивалент `clamp(min-content, auto, max-content)`.
- `content`: валидно только при использовании `calc-size()` в свойстве `flex-basis`.

Специальные аргументы:

- `any`: общий базис, который используется, когда внутренний тип неизвестен или при выполнении вложенных вычислений.
- Вложенный `calc-size()`: позволяет выполнять многоэтапные или условные вычисления.
- `<calc-sum>`: фиксированное значение, проценты или математическое выражение (например, `100px` или `20%`). При использовании фиксированного значения в качестве базиса, ключевое слово `size` по-прежнему доступно (но только во втором аргументе) и представляет разрешенное (resolved) значение этого базиса.

Важно: ключевое слово `size` нельзя использовать в первом аргументе (`<calc-size-basis>`). Это локальная переменная, которая ссылается на базис во втором аргументе (`<calc-sum>`).

### Аргументы вычисления (`<calc-sum>`)

Вторым аргументом является математическое выражение.

- В нем, как правило, используется ключевое слово `size` в качестве ссылки на значение базиса.
- Хотя ключевое слово `size` технически является опциональным, при его отсутствии вычисление разрешается в фиксированное значение, а базис полностью игнорируется.
- Оно может включать стандартные математические операторы (`+`, `-`, `*`, `/`).
- Оно может включать математические функции CSS, такие как `clamp()`, `min()`, `max()` и `round()`.
- Важно: `calc-size()` допускает использование только одного значения внутреннего размера (базиса) в каждом вычислении. Нельзя смешивать разные ключевые слова размеров в одном вызове `calc-size()`.

## Случаи использования

### Анимация к и от внутренних размеров

По умолчанию, браузеры не умеют выполнять интерполяции между фиксированными размерами (например, `0px`) и ключевыми словами внутренних размеров (например, `auto`). Оборачивание ключевого слова в `calc-size()` делает его "интерполируемым" значением.

#### Выбор правильного инструмента для анимации

- Используйте `interpolate-size: allow-keywords` для простой анимации к или от внутренних размеров (например, от `height: 0` к `height: auto`) без каких-либо математических модификаций. Это рекомендуемый подход для простой интерполяции ключевых слов, и в идеале его следует применять глобально через `:root`.

```css
:root {
  /* Включаем интерполяцию ключевых слов глобально */
  interpolate-size: allow-keywords;
}

.item {
  height: 0;
  transition: height 0.3s ease;
}

.item.open {
  /* Простая интерполяция от 0 к auto теперь работает без `calc-size()` */
  height: auto;
}
```

- Используйте `calc-size()` только при необходимости выполнения математических вычислений внутреннего размера в процессе перехода (например, добавление паддинга или ограничение (clamping) размера).

```css
.accordion-content {
  display: block;
  overflow: hidden;
  /* Резервное значение для закрытого состояния */
  block-size: 0;
  transition: block-size 0.3s ease-out;
}

.accordion-content.open {
  /* Резервное значение для открытого состояния */
  block-size: auto;

  /*
    Используем `calc-size(auto, ...)` для включения анимации от 0 к внутреннему размеру
    элемента при выполнении вычисления (в данном случае, добавлении пространства в 2rem).
  */
  block-size: calc-size(auto, size + 2rem);
}
```

Важно: нельзя напрямую интерполировать два ключевых слова внутренних размеров. Одним концом перехода должно быть фиксированное значение или проценты.

### Применение ограничений к внутренним размерам

`calc-size()` можно использовать с любой математической функцией CSS, такой как `min()`, `max()`, `clamp()` или `round()`, для ограничения внутреннего размера рамками дизайна.

```css
.dynamic-container {
  /* Всегда предоставляем резерв для браузеров, которые не поддерживают `calc-size()` */
  inline-size: fit-content;

  /*
    Устанавливаем динамический размер на основе контента и двух условий:
    1. Соблюдение границ, установленных с помощью математической функции CSS (min, clamp и т.д.).
    2. Модификация внутреннего размера с помощью фиксированных или относительных отступов.
  */
  inline-size: calc-size(fit-content, min(size + var(--extra-space), var(--max-allowed)));
}
```

## Важные замечания

- Ловушки процентов: проценты внутри `<calc-sum>` разрешаются относительно размера контейнера, а не ключевого слова `size`. Например, `calc-size(auto, size + 10%)` добавляет 10% родительской ширины к ширине `auto` элемента, что может привести к неожиданному результату или переполнению.
- Требования вычислений: используйте `interpolate-size: allow-keywords`, вместо `calc-size()`, для простой анимации (например, от `0` к `auto`). `calc-size()` должна использоваться, только когда макет требует динамических математических корректировок внутреннего базиса.
- Заметка о производительности: анимация свойств блочной модели, таких как `inline-size` или `block-size`, запускает повторные вычисления макета, что может быть дорогим с точки зрения производительности. Используйте анимацию `calc-size()` для элементов, критичных для макета, где альтернатив, не связанных с макетом, недостаточно.

## Поддержка и страховка

`calc-size()` и `interpolate-size` имеют ограниченную поддержку браузеров. Поддерживаются в Chrome 129+ и Edge 129+ (с сентября 2024 года). Не поддерживаются в Firefox и Safari.

`calc-size()` и `interpolate-size` являются прогрессивными улучшениями. В старых браузерах эти свойства игнорируются, переход к внутреннему размеру выполняется мгновенно. Всегда предоставляйте стандартное ключевое слово или фиксированное значение в качестве резерва.

```css
.element {
  /* Резерв для браузеров без поддержки `calc-size()` */
  inline-size: fit-content;
  /* Современные браузеры перезаписывают резерв */
  inline-size: calc-size(fit-content, size + 2rem);
}
```

### Резервы для анимации и перехода

В браузерах без поддержки `calc-size()` или `interpolate-size` переходы с ключевыми словами внутренних размеров не интерполируются.

- Мягкая деградация: дефолтный резерв - это "мгновенный прыжок" между состояниями (например, от `0` к `auto`). Это часто является приемлемым, поскольку макет остается функциональным.
- Улучшенный опыт: используйте `@supports` для применения сложной логики компоновки или дополнительной стилизации, только когда возможны плавные переходы внутренних размеров.
- Не выполняйте измерений с помощью JS: хотя можно использовать JavaScript для измерения элементов и ручной анимации их размеров, это часто является лишним и может приводить к засорению макета. Нативный "мгновенный прыжок" является рекомендуемым резервом для современных веб-приложений.

Определение поддержки с помощью CSS или JavaScript:

```css
/* Определение поддержки в CSS */
@supports (inline-size: calc-size(auto, size + 0px)) {
  .element {
    /* Продвинутая логика */
  }
}
```

```javascript
/* Определение поддержки в JavaScript */
if (CSS.supports('inline-size', 'calc-size(auto, size + 0px)')) {
  // Продвинутая логика
}
```
