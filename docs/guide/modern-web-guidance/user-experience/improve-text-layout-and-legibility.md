# Улучшение компоновки и читаемости текста

Компоновка текста, в частности, концы строк и концы параграфов, могут влиять на читаемость и эстетическую привлекательность страницы. CSS предоставляет несколько настроек переноса текста, которые могут пригодиться в конкретных случаях.

Для коротких блоков текста, таких как заголовки, используйте `text-wrap: balance`. Это свойство указывает браузеру распределять текст как можно более равномерно по строкам, создавая более симметричный вид.

CSS-свойство `text-wrap: pretty` позволяет улучшить качество типографики основного текста, включив более сложный алгоритм переноса. Он специально предназначен для предотвращения появления "сирот" (orphans) (отдельных слов в последней строке параграфа) и создания более приятного внешнего вида длинных блоков текста.

## Реализация

### 1. Определение текстовых элементов

Для `text-wrap: balance` выбирайте короткие текстовые блоки, такие как заголовки и названия колонок таблицы. Не трогайте элементы с визуальными блоками, такими как границы или фон, поскольку в макете могут образоваться неожиданные пустые области.

Для `text-wrap: pretty` выбирайте элементы, которые потенциально могут содержать длинные фрагменты текста, где сильно заметны слова-сироты или плохие переносы строк. Это включает следующие элементы:
  - `<p>`
  - `<blockquote>`
  - `<li>`
  - любой другой элемент, который может содержать длинный текст.

#### Выбор правильного метода переноса

| Критерий                     | `text-wrap: balance`                  | `text-wrap: pretty`                   | `text-wrap: wrap` (Default)            |
|:-----------------------------|:--------------------------------------|:--------------------------------------|:---------------------------------------|
| Хорошо подходит для          | Короткие блоки (заголовки, названия)  | Длинные блоки (параграфы, списки)     | Критичный к производительности контент |
| Визуальная цель              | Симметричная длина строк              | Отсутствие слов-сирот                 | Быстрый стандартный перенос            |
| Ограничение количества строк | До 6-10 строк (ограничение алгоритма) | Больше 3 строк                        | Нет ограничений                        |
| Производительность           | Низкая: алгоритм бинарного поиска     | Средняя: алгоритм обратного просмотра | Высокая: стандартный жадный алгоритм   |

### 2. Применение выбранного метода

Применяем `text-wrap: balance` для коротких многострочных элементов, таких как заголовки (`h1`-`h6`), подзаголовки или цитаты:

```css
/* Находим заголовки для сбалансированного переноса */
h1, h2, h3, h4, h5, h6 {
  /* Включает логику построчно-сбалансированного переноса */
  text-wrap: balance;
}
```

Используем `text-wrap: pretty` для включения оптимизированного алгоритма, оценивающего последние несколько строк параграфа для определения лучшей точки переноса:

```css
/* Применяем к многострочным блокам текста для предотвращения появления слов-сирот */
p, blockquote, li, .pretty-text {
  /* Включает оптимизированный перенос для основного текста */
  text-wrap: pretty;
}
```

### Критичные ограничения и производительность

#### text-wrap: balance

* Ограничение количества строк: браузеры накладывают ограничение на количество строк, которые они пытаются сбалансировать в целях производительности (6 строк в Chromium и 10 строк в Firefox). Если строк больше, браузер использует стандартный перенос `wrap`. Не используйте `text-wrap: balance` на блоках текста с потенциально большим количеством строк.
* Целевое применение: не применяйте `text-wrap: balance` глобально (например, `* { text-wrap: balance; }`). Итеративный алгоритм "Двоичный поиск", используемый браузером, является дорогим с точки зрения производительности. Ограничивайте его использование определенными короткими элементами.
* Взаимодействие с шириной: `text-wrap: balance` не меняет ширину контейнера (`inline-size`). Оно влияет только на то, как текст переносится в пределах этой ширины. Это может привести к образованию пустого пространства в конце контейнера, что может влиять на макеты, полагающиеся на блоки текста во всю ширину.

#### text-wrap: pretty

* Производительность и качество: свойство `text-wrap: pretty` использует менее производительный, чем дефолтное свойство `wrap` (жадный) алгоритм, поскольку оно оценивает несколько строк (обычно, последние четыре) для оптимизации точек переноса. Не используйте его глобально для каждого элемента, если на странице очень много текста.
* Лучше подходит для многострочного текста: выгоды `pretty` наиболее очевидны в параграфах, состоящих из 3 и более строк. На коротких текстах из пары-тройки строк эффект почти незаметен.
* Зависящее от браузера поведение: имейте ввиду, что детали реализации отличаются. Браузеры на основе Chromium, как правило, оценивают 4 последние строки, другие движки могут оценивать весь параграф.

### Поддержка и страховка

`text-wrap: balance` поддерживается всеми основными браузерами с 13.05.2024. `text-wrap: pretty` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 117+, Edge 117+ (с сентября 2023 года) и Safari 26+ (с сентября 2025 года). Не поддерживается в Firefox.

Браузеры, не поддерживающие `text-wrap: balance` или `text-wrap: pretty`, просто их игнорируют, а текст переносится с помощью дефолтной логики `wrap`. Это прогрессивное улучшение, которое мягко деградирует к стандартной типографике. Таким образом, контент остается полностью читаемым во всех браузерах, а браузеры с поддержкой предоставляют улучшенный опыт пользователя.

Для критичных макетов с обязательной продвинутой компоновкой текста, используйте библиотеку JavaScript, но имейте ввиду, что это сопряжено со снижением производительности.
