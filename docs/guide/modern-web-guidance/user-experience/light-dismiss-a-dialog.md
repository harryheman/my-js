# Легкая отмена модального диалога

Современные модальные диалоги часто поддерживают "легкую отмену" (light dismiss), позволяя пользователям закрывать диалог путем клика или касания наложения (backdrop) (область за пределами диалога). Атрибут `closedby` предоставляет декларативный способ включить это поведение без кастомного JavaScript.

## Реализация

Для включения легкой отмены:

1. Добавьте `closedby="any"` к элементу `<dialog>`.
2. Откройте диалог с помощью `dialog.showModal()`.

### Значения атрибута

- `any`: включает легкую отмену (клик по наложению), "запросы закрытия" (close requests) (клавиша `Esc`) и программные механизмы (например, `dialog.close()`).
- `closerequest`: включает только "запросы закрытия" и программные механизмы. Это дефолтное поведение модальных диалогов.
- `none`: диалог можно закрыть только программно.

### Стилизация наложения

Когда диалог открывается как модальное окно с помощью `showModal()`, браузер генерирует псевдоэлемент `::backdrop`. Это наложение покрывает всю область просмотра и располагается прямо за диалогом:

```css
/* Стилизуем наложение */
dialog::backdrop {
  background-color: rgba(0, 0, 0, 0.5);
  backdrop-filter: blur(2px); /* Опционально: добавляем размытие для современных браузеров */
}
```

## Пример

```html
<!-- Используем `closedby="any"` для включения поведения легкой отмены -->
<dialog id="myDialog" closedby="any" aria-labelledby="dialogTitle">
  <form method="dialog">
    <h2 id="dialogTitle">Обратная связь</h2>
    <p>Кликнете за пределами этого блока или нажмите Esc для закрытия диалога.</p>
    <button type="submit">Закрыть</button>
  </form>
</dialog>

<button onclick="document.getElementById('myDialog').showModal()">Открыть диалог</button>
```

## Ограничения и доступность

- Используйте `closedby="any"` для включения декларативной легкой отмены.
- Всегда открывайте модальные диалоги с помощью `showModal()`. Это обеспечивает рендеринг диалога на верхнем уровне (top layer), захват фокуса и обработку нажатия клавиши `Esc`.
- Используйте `aria-labelledby` или `aria-label` для предоставления доступного (accessible) названия для диалога.
- Не используйте `closedby` для немодальных диалогов (открытых с помощью `show()`), поскольку у них нет наложения, и они не запускают легкую отмену.
- Не используйте событие `click` для критичной логики, которая должна выполняться перед закрытием диалога; вместо этого, обрабатывайте события `close` или `cancel`.

## Поддержка и страховка

`<dialog closedby>` имеет ограниченную поддержку. Поддерживается в Chrome 134+, Edge 134+ (с марта 2025 года) и Firefox 141+ (с июля 2025 года). Не поддерживается в Safari.

Для браузеров без поддержки следует реализовать резервный вариант легкой отмены путем определения клика за пределами границ контента диалога:

```javascript
const dialog = document.querySelector('dialog');

// Резерв для браузеров без поддержки атрибута `closedby`
if (!('closedBy' in HTMLDialogElement.prototype)) {
  dialog.addEventListener('click', (event) => {
    // 1. При клике по наложению, целью события является сам диалог.
    // Игнорируем клики, когда целью является потомок диалога
    if (event.target !== dialog) return;

    // 2. Определяем, что клик произошел в пределах границ контента диалога.
    // Это отделяет клик по наложению от клика по фону/паддингу диалога
    const rect = dialog.getBoundingClientRect();
    const isDialogContent = (
      rect.top <= event.clientY &&
      event.clientY <= rect.top + rect.height &&
      rect.left <= event.clientX &&
      event.clientX <= rect.left + rect.width
    );
    if (isDialogContent) return;

    // 3. Поскольку клик произошел за пределами диалога, закрываем его
    dialog.close();
  });
}
```
