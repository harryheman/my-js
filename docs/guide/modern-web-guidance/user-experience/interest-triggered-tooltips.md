# Отображение всплывающей подсказки при наведении курсора

Пользователи хотят видеть дополнительную информацию без потери контекста. Отображение всплывающей подсказки (tooltip), когда пользователь заинтересован в получении дополнительной информации, может быть полезным для предоставления определения термина, пояснения операции, которая будет запущена кнопкой-иконкой или предоставления дополнительной инструкции по заполнению поля формы.

## Создание тултипа

Поповер с требуемым поведением можно создать путем добавления атрибута `popover="hint"` к `<div>` или другому семантически подходящему элементу. Когда пользователь открывает тултип, это скрывает другие тултипы с `popover="hint"`, но не скрывает тултипы с `auto` или `manual`. Это также обрабатывает вложенные тултипы.

Кроме того, это предоставляет функционал легкой отмены (light dismiss), поэтому когда пользователь кликает или другим способом фокусируется за пределами поповера, поповер закрывается.

Элемент тултипа должен содержать атрибут `id` с уникальным значением:

```html
<!-- Контейнер тултипа `<div>` должен содержать атрибут `popover`.
     Значение `"hint"` обеспечивает его "легкую отмену" -->
<div popover="hint" id="tooltip">Контент тултипа</div>
```

Пользователь выражает интерес в получении дополнительной информации путем наведения курсора или фокусировки на элементе `<a>` или `<button>`. Элемент должен содержать атрибут `interestfor`, совпадающий с атрибутом `id` тултипа:

```html
<!-- Атрибут `interestfor` может применяться к элементу `<button>` -->
<button interestfor="tooltip">Триггер тултипа</button>

<!-- Атрибут `interestfor` также может применяться к элементу `<a>` -->
<a interestfor="tooltip" href="">Триггер тултипа</a>
```

Триггер должен иметь визуальный индикатор возможности получения дополнительной информации.

### Доступность, встроенная в `interestfor`

`interestfor` сам заботится о вспомогательных технологиях, поэтому обычно добавлять атрибуты ARIA вручную не нужно:

- Цель с `popover="hint"` получает неявную минимальную роль `tooltip`. Не устанавливайте `role="tooltip"` самостоятельно.
- Браузер неявно ассоциирует этот элемент-источник с целью с помощью `aria-describedby`, когда цель - обычный текст, или с помощью `aria-details`, когда цель содержит интерактивный контент. Не добавляйте `aria-describedby` или `aria-details` к триггеру.
- Поскольку цель переключается в `aria-details` при необходимости, цель может содержать интерактивный контент (например, ссылку).

### Ограничения доступности (WCAG 1.4.13)

Даже с обработкой семантики `interestfor`, описанной выше, наша реализация должна удовлетворять WCAG 1.4.13 (контент при наведении или фокусировке):
- отменяемость: пользователи должны иметь возможность закрывать тултип без движения указателя (pointer) или смены клавиатурного фокуса (например, путем нажатия клавиши `Escape`). Нативный атрибут `popover` управляет этой привязкой автоматически;
- "наводимость" курсора: тултип должен оставаться открытым при перемещении указателя по его контенту. Это позволяет пользователям с лупой безопасно читать текст;
- постоянство: тултип должен оставаться видимым до удаления триггера наведения или фокуса, явной отмены пользователем или невалидности контента.

### Позиционирование тултипа

Тултип может позиционироваться с помощью якорного позиционирования (anchor positioning). Когда тултип открывается с помощью `interestfor`, триггер становится неявным якорем для тултипа, т.е. нам не нужно добавлять свойства CSS `anchor-name` или `position-anchor`. Однако, для браузеров, не поддерживающих якорное позиционирование, нужно использовать полифилл, который имеет несколько ограничений для поповеров. Неявные якоря не поддерживаются полифиллом, поэтому нужно явно устанавливать `anchor-name` на триггере и `position-anchor` на поповере:

```css
/* Используем явное название якоря для совместимости с полифиллом */
button[interestfor="tooltip-dom"] {
  anchor-name: --tooltip-dom;
}
#tooltip-dom {
  position-anchor: --tooltip-dom;
}
```

Также полифилл не поддерживает `position-area` на поповерах, поэтому позиционирование следует выполнять с помощью функций `anchor()`  и включать резерв `position-try` (например, `flip-block` или `flip-inline`):

```css
[popover]{
  /* Используем функции `anchor()` и резерв `position-try` для полифилла */
  top: anchor(bottom);
  left: anchor(center);
  position-try: flip-block;
  margin: unset;
}
```

### Поддержка и страховка

Interest Invokers API имеет ограниченную поддержку браузеров. Поддерживается в Chrome 142+ и Edge 142+ (с октября 2025 года). Не поддерживается в Firefox и Safari.

Рекомендуемым полифиллом этого API является пакет `interestfor` из NPM. Предпочитайте включение полифилла в сборку, чем его запрос из CDN.

```html
<script type="module">
  if(!HTMLButtonElement.prototype.hasOwnProperty("interestForElement")){
    // Ссылка на CDN используется только для примера, лучше включать полифилл в сборку
    await import("https://unpkg.com/interestfor@latest");
  }
</script>
```

Popover API поддерживается всеми основными браузерами с 27.01.2025.`popover="hint"` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 133+, Edge 133+ (с февраля 2025 года) и Firefox 149+ (с марта 2026 года). Не поддерживается в Safari.

Рекомендуемым полифиллом является `@oddbird/popover-polyfill`. Поведение `"hint"` не предоставляется полифиллом в браузерах, поддерживающих `popover`, но не поддерживающих `popover="hint"`. В таких браузерах тултип, открытый через фокусировку, останется открытым при открытии второго тултипа через наведение курсора.

```html
<script type="module">
  if(!HTMLElement.prototype.hasOwnProperty("popover")){
    await import("https://unpkg.com/@oddbird/popover-polyfill@latest");
  }
</script>
```

Anchor Positioning API поддерживается всеми основными браузерами (в Safari пока только в качестве экспериментальной возможности).

Для старых браузеров рекомендуется использовать полифилл `@oddbird/css-anchor-positioning`. Он не поддерживает неявные якоря, поэтому следует явно добавлять названия якорей к триггерам. `position-area` также не поддерживается полифиллом, поэтому для установки сторон прикрепления поповера следует использовать `anchor()`:

```html
<!-- Условно загружаем полифилл якорного позиционирования -->
<script type="module">
  if (!("anchorName" in document.documentElement.style)) {
    await import("https://unpkg.com/@oddbird/css-anchor-positioning");
  }
</script>
```

```css
button[interestfor="tooltip-attrs"] {
  /* Каждая пара триггер-поповер должна иметь уникальное название якоря в `anchor-name` на триггере и `position-anchor` на поповере */
  anchor-name: --tooltip-attrs;
}
#tooltip-attrs {
  position-anchor: --tooltip-attrs;
  /* При использовании полифилла используем функции `anchor()`, вместо свойства `position-area` */
  top: anchor(bottom);
  left: anchor(right);
  margin: unset;
}
```
