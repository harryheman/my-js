# Получение не зависящих от локации данных с помощью Temporal

Запись хронологических данных, которые должны быть одинаковыми, независимо от локации пользователя (таких как даты рождения, повторяющиеся сигналы или национальные праздники), с помощью объекта `Date` исторически была подвержена ошибкам. Поскольку объект `Date` всегда представляет определенный момент времени и привязан к временной зоне, сохранение даты, например, "1990-01-01", может привести к тому, что пользователи в другой временной зоне увидят "1989-12-31".

`Temporal` API предоставляет "простые" (plain) типы, такие как `Temporal.PlainDate` и `Temporal.PlainTime`, которые не зависят от временной зоны. Эти типы представляют дату и время, которые не зависят от календаря или часового пояса, что делает их идеальными для "локационно-независимых" данных.

## Реализация

Для получения и отображения не зависящих от локации данных:

1. Используйте `Temporal.PlainDate` для дат: для таких данных, как даты рождения или праздники, используйте `Temporal.PlainDate.from()` для создания экземпляра из строки ISO 8601 или объекта.
2. Используйте `Temporal.PlainTime` для времени: для таких данных, как ежедневные сигналы или предпочтительное время обеда, используйте `Temporal.PlainTime.from()`.
3. Отображение без преобразования: поскольку эти объекты не зависят от временной зоны, они отображают одинаковые значения, независимо от локальной временной зоны пользователя.

### Пример: получение даты рождения

```javascript
// 1. Парсим строку даты из инпута (например, "1990-01-01")
const birthdateStr = "1990-01-01";
const plainDate = Temporal.PlainDate.from(birthdateStr);

// 2. Отображаем дату.
// Результатом будет "01/01/1990" (или эквивалент) в любой временной зоне
console.log(plainDate.toLocaleString('en-GB'));

// 3. Сравниваем со стандартным `Date` (который может смещаться)
const dateObj = new Date("1990-01-01T00:00:00Z");
// Во временной зоне UTC-5 можно получить "31/12/1989"
console.log(new Intl.DateTimeFormat('en-GB', { timeZone: 'America/New_York' }).format(dateObj));
```

## Стратегическая реализация и лучшие практики

- Используйте `Temporal.PlainDate` для "календарных дат", таких как даты рождения, годовщины и праздники, когда календарь или временная зона неважны.
- Используйте `Temporal.PlainTime` для "настенных часов", таких как ежедневные напоминания в 9:00, когда время должно быть 9:00 в любой временной зоне, в какой бы ни оказался пользователь.
- Не используйте простые типы для представления определенного момента в физическом времени ("мгновения" (instant)). Используйте `Temporal.Instant` или `Temporal.ZonedDateTime` для логов, временных меток событий и всего, что зависит от временной зоны.
- Помните, что объекты `Temporal` иммутабельны. Такие методы, как `add()` или `with()` возвращают новый экземпляр, а не модифицируют оригинал.

## Поддержка и страховка

`Temporal` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 144+, Edge 144+ (с января 2026 года) и Firefox 139+ (с мая 2025 года). Не поддерживается в Safari.

Для старых браузеров используйте определение поддержки и полифилл. Один из соответствующих стандартам полифиллов - `@js-temporal/polyfill`.

Обратите внимание, что полифилл не добавляет объект `Temporal` в глобальную область видимости автоматически во избежание конфликтов. При необходимости, это нужно делать вручную.

```javascript
// Проверяем поддержку Temporal
(async () => {
  if (typeof Temporal === 'undefined') {
    // Загружаем полифилл условно
    const module = await import("https://esm.sh/@js-temporal/polyfill");
    globalThis.Temporal = module.Temporal;
    // Расширяем `Date.prototype` при необходимости
    Date.prototype.toTemporalInstant = module.toTemporalInstant;
    initializeApp();
  }
})();
```
