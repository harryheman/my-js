# Стилизация на основе состояния потомка

Исторически селекторы CSS позволяли стилизовать потомков на основе их предка, но не предка на основе его потомков. Псевдокласс `:has()` меняет правила игры, позволяя условно стилизовать элемент контейнера, в зависимости от наличия или состояния его потомков.

Сочетая `:has()` с псевдоклассами состояний (такими как `:checked`, `:focus`, `:valid`, `:invalid` или `:not()`), можно создавать сложные интерактивные компоненты UI только с помощью CSS, без использования JavaScript для переключения классов "модификаторов" (таких как `.is-active` или `.has-error`) на родительских элементах.

Это очень полезно для компонентов, которые должны реагировать на внутренние взаимодействия, такие как локализованный переключатель темы, реагирующий на   чекбокс (`:checked`), группа формы, подсвечивающая ошибку (`:invalid`), или карточка, поднимающаяся над фоном, когда дочерняя ссылка находится в фокусе (`:focus-within`).

## Реализация

Используйте селектор `:has()` на контейнере для определения определенного состояния (например, `:checked`, `:focus`, `:invalid`) его интерактивных дочерних элементов.

Для разработки компонента, который стилизуется на основе состояния потомка:

1. Определите дефолтные стили: установите переменные CSS на контейнере для определения его базового состояния.
2. Примените перезаписи на основе состояния: найдите контейнер с помощью `:has([child-selector]:[state])` и переопределите переменные CSS для активного/неактивного состояния.

Пример: компонент, меняющий тему на основе дочернего переключателя.

```css
/* 1. Определяем дефолтное состояние контейнера */
.theme-card {
  /* Используем кастомные свойства, чтобы сделать переключение состояния чище */
  --card-bg: #ffffff;
  --card-text: #333333;
  --card-border: #cccccc;

  background-color: var(--card-bg);
  color: var(--card-text);
  border: 1px solid var(--card-border);

  /* Используем переход для плавных изменений состояния */
  transition: background-color 0.3s, color 0.3s;
}

/* 2. Применяем стили, когда потомок входит в определенное состояние. */
/* Используйте селектор контейнера и :has() для проверки состояния потомка */
.theme-card:has(.theme-toggle:checked) {
  /* Перезаписываем свойства для активного/неактивного состояния */
  --card-bg: #222222;
  --card-text: #f0f0f0;
  --card-border: #555555;
}
/* Можно комбинировать :has() и :not() для определения негативных состояний. */
/* Это выбирает карточку, которая не содержит переключатель в выбранном состоянии */
.theme-card:not(:has(.theme-toggle:checked)) {
  /* Опционально: явная стили неактивного состояния */
}
```

```html
<!-- Стилизуемый контейнер -->
<div class="theme-card">
  <!-- Дочерний элемент, состояние которого определяет стили предка -->
  <label>
    <input type="checkbox" class="theme-toggle">
    Включить темный режим
  </label>

  <h2>Заголовок карточки</h2>
  <p>Стили всей карточки управляются чекбоксом выше.</p>
</div>
```

Заметка о производительности: при использовании `:has()`, ограничивайте область действия этого селектора наиболее специфичным контейнером (таким как `.theme-card`). Избегайте его привязки к очень высокоуровневым элементам (например, `body:has(...)`), если изменение стилей является локальным, поскольку чем шире запрос `:has()`, тем больше пересчетов макета выполняется.

## Поддержка и страховка

`:has()` поддерживается всеми основными браузерами с 19.12.2023.

Если стилизация на основе состояния является критичной для пользовательского опыта или макета страницы, нужно предусмотреть резерв для браузеров без поддержки селектора `:has()`. Для чисто декоративных эффектов, `:has()` может использоваться как прогрессивное улучшение без резерва.

При реализации резерва для критичного функционала, следует использовать `@supports not selector(:has(*))` для определения традиционного основанного на классах резерва. Если критичное изменение состояния зависит от действия пользователя, следует также использовать небольшой встроенный скрипт с `CSS.supports()` для переключения класса на основе эквивалентного события JavaScript (например, `change`, `focus`, `blur`), представляющего изменение состояния.

```css
/* Резервный CSS для старых браузеров. */
/* Проверяем, что браузер не поддерживает селектор :has() */
@supports not selector(:has(*)) {
  /* Определяем традиционный класс-модификатор, который применяет точно такие же перезаписи */
  .theme-card.is-active {
    --card-bg: #222222;
    --card-text: #f0f0f0;
    --card-border: #555555;
  }
}
```

```javascript
/* Резерв JavaScript для старых браузеров. */
/* Проверяем поддержку перед запуском скрипта во избежание лишней работы в современных браузерах */
if (!CSS.supports('selector(:has(*))')) {
  const toggle = document.querySelector('.theme-toggle');
  const card = document.querySelector('.theme-card');

  if (toggle && card) {
    // Вручную переключаем резервный класс при изменении состояния инпута
    toggle.addEventListener('change', (e) => {
      card.classList.toggle('is-active', e.target.checked);
    });
  }
}
```
