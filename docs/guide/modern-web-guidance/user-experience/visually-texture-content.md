# Визуальное текстурирование контента

## Обзор

Для применения реалистичных паттернов износа или текстур (таких как гранж, шум или текстура бумаги) к элементу, используйте маскирование (masking) CSS (`mask-image`) с повторяющимся изображением текстуры. Это позволяет текстурировать контент, сделав его части полупрозрачными, а не просто накладывая текстуру поверх. Это создает более реалистичный эффект физического материала.

## Реализация

Применить текстурный паттерн можно следующим образом:

### Метод 1: использование повторяющегося растрового изображения (рекомендуется для реалистичных текстур)

Это самый популярный метод для реалистичных структур:

```css
.weathered-element {
  /* Используем вендорный префикс для поддержки в старых браузерах */
  -webkit-mask-image: url('grunge-pattern.png');
  -webkit-mask-repeat: repeat; /* Повторяем паттерн для заполнения области */
  -webkit-mask-size: 300px; /* Управляем масштабом текстуры */

  /* Стандартные свойства для современных браузеров */
  mask-image: url('grunge-pattern.png');
  mask-repeat: repeat;
  mask-size: 300px;
}
```

### Метод 2: использование градиентов CSS для геометрических паттернов

Паттерн можно генерировать с помощью градиентов CSS. Это автономное решение, не требующее внешних файлов изображений:

```css
.patterned-element {
  --checkerboard-gradient:
    linear-gradient(45deg, #000 25%, transparent 25%),
    linear-gradient(-45deg, #000 25%, transparent 25%),
    linear-gradient(45deg, transparent 75%, #000 75%),
    linear-gradient(-45deg, transparent 75%, #000 75%);

  /* Применяем шахматный паттерн в качестве маски */
  -webkit-mask-image: var(--checkerboard-gradient);
  -webkit-mask-size: 20px 20px;
  -webkit-mask-position: 0 0, 0 10px, 10px -10px, -10px 0px;

  mask-image: var(--checkerboard-gradient);
  mask-size: 20px 20px;
  mask-position: 0 0, 0 10px, 10px -10px, -10px 0px;
}
```

### Режимы маскирования: альфа-канал и яркость

По умолчанию, маски CSS используют `mask-mode: match-source`. Это означает, что браузер сам решает, использовать альфа-канал (прозрачность) или светимость (яркость) маски на основе предоставленного источника:

| Тип источника маски                                 | Дефолтный режим маски | Маскирование                                                                                                                     |
|:----------------------------------------------------|:----------------------|:---------------------------------------------------------------------------------------------------------------------------------|
| Инлайн элемент SVG `<mask>`                         | `luminance`           | Прозрачность определяется яркостью цветов. Белый цвет показывает контент, черный - скрывает, а серый - делает его полупрозрачным |
| Внешний файл изображения  (например, файл PNG, SVG) | `alpha`               | Прозрачность (opacity) определяется прозрачностью (transparency). Непрозрачные части показывают контент, прозрачные - скрывают   |
| Градиент CSS                                        | `alpha`               | Прозрачность определяется прозрачностью. Непрозрачные цвета (такие как `black`) показывают контент, прозрачные - скрывают        |

Дефолтный режим маскирования можно перезаписать с помощью свойства `mask-mode` (например, `mask-mode: luminance;` или `mask-mode: alpha;`).

## Поддержка и страховка

Маски поддерживаются всеми основными браузерами с 07.12.2023.

Если браузер не поддерживает `mask-image` или версии с вендорным префиксом:

- Элемент будет отображаться без текстуры (чистая заливка).
- Убедитесь, что контент остается читаемым без текстуры (прогрессивное улучшение).
- В качестве резерва можно использовать фоновое изображение или наложение (overlay) для симуляции текстуры, хотя это не будет влиять на прозрачность самого контента:

```css
/* Используем фоновое изображение для браузеров без поддержки масок */
@supports (not (mask-image: url(x))) and (not (-webkit-mask-image: url(x))) {
  .weathered-element {
    background-image: url('grunge-pattern.svg');
    background-color: #fff;
  }
}
```
