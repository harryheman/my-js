# Запросы к якорному контейнеру

При разработке всплывающих подсказок или поповеров с помощью якорного позиционирования CSS (Anchor Positioning), браузер может автоматически менять положение элемента на резервное при выходе элемента за пределы области просмотра. Когда это происходит, может потребоваться изменить стили позиционируемого контента, например, изменить положение стрелки, указывающей от позиционируемого контента к якорю.

Запросы к якорному контейнеру (Anchored Container Queries) решают это, позволяя запрашивать активное состояние позиционирования элемента и применять соответствующие стили.

## Проблема

Представьте тултип, по умолчанию отображаемый над якорем. У него внизу есть стрелка, указывающая вниз. Пользователь прокручивает страницу, и туплтип перемещается под якорь. Теперь стрелка указывает в неправильном направлении и располагается на неправильной стороне тултипа.

## Решение: запросы к якорному контейнеру

Установка `container-type: anchored` на позиционируемом элементе превращает его в запрашиваемый контейнер (query container), который "знает" о своем состоянии якорного позиционирования. Затем можно использовать запрос `@container anchored()` для обновления его потомков или псевдоэлементов.

### 1. Создание тултипа и триггера

Используем Popover API для создания тултипа. Это создает неявную якорную связь, которую можно использовать для позиционирования:

```html
<button popovertarget="tooltip" id="anchor" aria-describedby="tooltip">якорь</button>
<div id="tooltip" popover role="tooltip"></div>
```

Сбрасываем стили положения и отступов поповера для использования с якорным позиционированием, но только если оно поддерживается:

```css
@supports (anchor-name: --my-anchor) {
  [popover] {
    inset: auto;
    margin: unset;
  }
}
```

### 2. Установка контейнера

Применяем `container-type: anchored` к позиционируемому элементу. Этот элемент также должен содержать `position-try-fallbacks` для автоматической смены положения:

```css
#tooltip {
  position: fixed;
  position-area: block-start;
  position-try-fallbacks: flip-block;

  /* Включаем запросы к якорному контейнеру */
  container-type: anchored;
}
```

### 3. Стилизация на основе резерва

Используем `@container anchored(fallback: <value>)` для применения стилей определенного резерва.

Как и все запросы к контейнеру, `@container` может стилизовать только потомков контейнера. Обычно, стрелки создаются с помощью псевдоэлементов `::before` и `::after`, которые считаются потомками и могут стилизоваться напрямую. Однако, для стилизации самого тултипа (как показано в шаге 4), нужно добавить потомка в тултип и создать стрелку в его псевдоэлементе `::before`:

```html
<div id="tooltip" popover role="tooltip">
  <div class="tooltip-content">Тултип</div>
</div>
```

```css
.tooltip-content::before {
  /* Дефолтная "нижняя" стрелка для позиции 'top' */
  content: "▼";
  position: absolute;
  inset-block-end: 0;
  inset-inline-start: 1rem;
}

/* Обновляем на "верхнюю" стрелку, когда активен резерв 'flip-block' */
@container anchored(fallback: flip-block) {
  .tooltip-content::before {
    content: "▲";
    inset-block-start: 0;
    inset-block-end: auto;
  }
}
```

## 4. Стилизация самого контейнера

Для изменения свойств самого контейнера (таких как `margin` или `background-color`) при смене положения можно использовать внутренний элемент-обертку.

1. Применяем `container-type: anchored` к внешнему позиционируемому элементу.
2. Стилизуем внутренний элемент в блоке `@container`.

```css
@container anchored(fallback: flip-block) {
  .tooltip-content {
    border-radius: 0 0 .5rem .5rem;
    margin-block-start: 0.25rem;
  }
}
```

## Лучшие практики

- Используйте логические резервы: используйте такие ключевые слова, как `flip-block` и `flip-inline`, в `position-try-fallbacks` для более простых запросов, обрабатывающих RTL и разные режимы письма автоматически.
- Используйте псевдоэлементы для стрелок: стрелки тултипов - чисто декоративные элементы, поэтому являются идеальными кандидатами для `::before` или `::after`, которые могут стилизоваться с помощью запросов к якорному контейнеру без дополнительных узлов DOM.

## Поддержка и страховка

Запросы к якорному контейнеру имеют ограниченную поддержку браузеров. Поддерживаются в Chrome 143+ и Edge 143+ (с декабря 2025 года). Не поддерживаются в Firefox и Safari.

Позиционирование стрелки на основе примененного резерва - это прогрессивное улучшение, другого способа обработки применения резерва не существует. Поэтому стрелку следует рендерить только в браузерах с поддержкой с помощью блока `@supports (container-type: anchored)`:

```css
@supports (container-type: anchored) {
  .tooltip-content::before {
    content: "▼";
  }
}
```

### Поповер

Popover API поддерживается всеми основными браузерами с 27.01.2025.

Для поддержки атрибута `popover` в старых браузерах рекомендуется использовать пакет `@oddbird/popover-polyfill`.

Определяйте поддержку путем проверки свойства `popover` на прототипе `HTMLElement`. Условно инициализируйте полифилл только при отсутствии поддержки.

**Вариант 1: использование сборщика модулей**

Устанавливаем пакет (`npm install @oddbird/popover-polyfill`).

```javascript
if (!('popover' in HTMLElement.prototype)) {
  import('@oddbird/popover-polyfill/fn').then(({ apply }) => {
    apply();
  });
}
```

**Вариант 2: использование CDN**

Можно динамически импортировать полифилл из CDN (такого как unpkg) внутри `<script type="module">`:

```html
<script type="module">
  if (!('popover' in HTMLElement.prototype)) {
    import('https://unpkg.com/@oddbird/popover-polyfill@latest/dist/popover-fn.js').then(({ apply }) => {
      apply();
    });
  }
</script>
```

Браузеры без поддержки Popover API также не поддерживают якорное позиционирование, поэтому тултип появится в центре экрана.
