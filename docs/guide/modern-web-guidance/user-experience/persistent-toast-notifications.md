# Создание постоянных всплывающих уведомлений

Всплывающие уведомления - это временные сообщения о статусе. В отличие от меню, они не должны закрываться, когда пользователь взаимодействует с другими частями страницы. Состояние `popover="manual"` идеально подходит для этого случая, поскольку оно отключает "легкую отмену" (light dismiss) и позволяет нескольким уведомлениям быть открытыми одновременно.

### Реализация

* Используйте `popover="manual"`, чтобы уведомление оставалось открытым до явного закрытия или истечения времени.
* Используйте контейнер для управления стеком уведомлений. Поскольку поповеры на верхнем слое (Top Layer) игнорируют родительский `z-index`, они должны позиционироваться индивидуально или внутри общей группы макета.
* Используйте `sibling-index()` для добавления отступа между уведомлениями, чтобы нижние элементы оставались видимыми.
* Предоставляйте явную кнопку "Закрыть" в уведомлении с `popovertargetaction="hide"`.
* Используйте JavaScript для таймеров автоматического закрытия уведомлений (например, путем вызова `hidePopover()` через 3000 мс после открытия).
* Используйте `transition-behavior: allow-discrete` для включения анимации входа и выхода элементов.

### Страховка

#### Поповер

* Используйте [этот полифилл](https://github.com/oddbird/popover-polyfill). В качестве альтернативы можно использовать `div` с фиксированной позицией и высоким `z-index`.

#### sibling-index()

* Если `sibling-index()` не поддерживается, используйте оператор `+` для добавления отступа вручную, например, `popover + popover { margin-top: 1rem }`

#### Якорное позиционирование

* Используйте [этот полифилл](https://github.com/oddbird/css-anchor-positioning). В качестве альтернативы можно прикреплять всплывающую подсказку к нижней части области просмотра с помощью `@supports not (anchor-name: --foo)`.

#### transition-behavior

* Если `transition-behavior` не поддерживается, используйте JavaScript для добавления анимации входа/выхода через классы CSS.
