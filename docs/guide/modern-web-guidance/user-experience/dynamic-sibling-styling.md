# Стилизация сиблингов на основе их количества и индекса

Исторически применение уникальных стилей с каждому сиблингу в списке требовало сложных циклов `:nth-child` или JavaScript для внедрения встроенных стилей. Современный CSS предоставляет `sibling-index()` и `sibling-count()` для выполнения этих вычислений прямо в таблице стилей, что позволяет создавать динамические макеты и цветовые системы, которые автоматически адаптируются к добавлению или удалению элементов.

## Динамическая цветовая система

Можно создать цветовой спектр для группы сиблингов, рассчитав уникальный оттенок (hue) или значение яркости (lightness) для каждого потомка. Это обеспечивает постоянный эффект градиента независимо от количества элементов.

```css
.swatch {
  /* Вычисляем оттенок путем деления полного круга (360deg) на количество сиблингов */
  /* и умножения на индекс текущего элемента, начинающийся с 1 */
  background-color: hsl(
    calc(360deg / sibling-count() * sibling-index()),
    70%,
    50%
  );
}
```

## Симметричная компоновка и эффекты веера

Для создания симметричных эффектов (таких как "веер" или расположение элементов вокруг центра), используйте общее количество для нахождения средней точки списка:

```css
.card {
  /* Находим индекс центрального элемента (например, 3, если у нас 5 сиблингов) */
  --center: calc((sibling-count() + 1) / 2);

  /* Вращаем элементы от центра: отрицательные значения для вращения влево, положительные для вращения вправо. */
  /* Центральный элемент вращается на 0deg */
  transform: rotate(calc(10deg * (sibling-index() - var(--center))));
}
```

## Круговое и сложное позиционирование

Комбинируя эти функции с тригонометрией CSS (`sin()`, `cos()`), можно расположить элементы по кругу без ручного вычисления координат:

```css
.orb {
  /* Вычисляем угол этого элемента на круге (360deg) */
  --angle: calc(360deg / sibling-count() * sibling-index());
  --radius: 150px;

  /* Устанавливаем предварительно преобразованное (pre-transformed) положение для центрирования всех элементов */
  position: absolute;
  place-self: center;

  /* Позиционируем каждый элемент вокруг родительского центра */
  transform: translate(
    calc(cos(var(--angle)) * var(--radius)),
    calc(sin(var(--angle)) * var(--radius))
  );
}
```

### Поддержка и страховка

`sibling-count()` и `sibling-index()` имеют ограниченную поддержку браузеров. Поддерживаются в Chrome 138+, Edge 138+ (с июня 2025 года) и Safari 26.2+ (с декабря 2025 года). Не поддерживаются в Firefox.

Если `sibling-index()` и `sibling-count()` не поддерживаются, предоставьте резерв путем внедрения кастомных свойств CSS через JavaScript. Используйте определение поддержки с помощью `CSS.supports()` для условного запуска скрипта.

```js
/* Проверяем поддержку перед применением резерва */
if (!CSS.supports('top: calc(sibling-index() * 1px)')) {
  const items = document.querySelectorAll('.item');
  items.forEach((item, index) => {
    /* Индекс должен начинаться с 1 для соответствия нативному функционалу */
    item.style.setProperty('--sibling-index', index + 1);
    item.style.setProperty('--sibling-count', items.length);
  });
}
```

В CSS используйте эти переменные как основу и перезаписывайте их нативными функциями внутри блока `@supports`. Нативный функционал должен быть обернут в `@supports` для того, чтобы переменные оставались валидными в старых браузерах.

```css
.item {
  /* 1. Устанавливаем базовые значения с помощью переменных (из резерва JS) */
  --index: var(--sibling-index);
  --count: var(--sibling-count);

  /* 2. Используем вычисленные переменные */
  background-color: hsl(calc(360deg / var(--count) * var(--index)), 70%, 50%);
}

@supports (top: calc(sibling-index() * 1px)) {
  .item {
    /* 3. Перезаписываем переменные только при наличии поддержки */
    --index: sibling-index();
    --count: sibling-count();
  }
}
```
