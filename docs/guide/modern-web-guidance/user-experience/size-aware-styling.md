# Стилизация на основе размеров контейнера

## Обзор

Стилизация на основе размеров контейнера позволяет компонентам менять их макет или внешний вид на основе доступного им пространства, а не на основе размеров всего экрана. Это полезно для таких компонентов, как карточки или панели навигации, которые могут помещаться в разные части макета (такие как узкая боковая панель или широкая основная область).

Запросы к контейнеру (container queries) позволяют сделать компоненты по-настоящему модульными. Не нужно знать, где компонент будет "жить" или писать сложные медиа-запросы для обработки каждого возможного макета.

## Реализация

### 1. Определение контейнера

Сначала надо сообщить браузеру, какой элемент является контейнером для измерения:

```css
.card-container {
  /* Определяем тип контейнера. Используем 'inline-size' для запросов ширины. */
  /* Также можно использовать 'size' для ширины и высоты, но это требует явного указания размеров */
  container-type: inline-size;
}
```

### 2. Применение стилей на основе размеров контейнера

Используем правило `@container` для применения стилей, когда контейнер достигает определенного размера:

```css
/* Дефолтные стили для небольших контейнеров (стек) */
.card {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

/* Стили для больших контейнеров (блоки располагаются в ряд). */
/* Это срабатывает, когда контейнер становится шире `400px` */
@container (min-width: 400px) {
  .card {
    flex-direction: row;
    align-items: center;
  }

  .card-image {
    width: 150px;
    height: 150px;
  }
}
```

### 3. Условное отображение контента

Запросы к контейнеру также можно использовать для скрытия или отображения дополнительных деталей, когда для этого достаточно места:

```css
.card-details {
  /* Скрываем дополнительные детали на маленьких экранах */
  display: none;
}

@container (min-width: 600px) {
  .card-details {
    /* Отображаем детали, когда для них достаточно места */
    display: block;
  }
}
```

### Поддержка и страховка

Запросы к контейнеру поддерживаются всеми основными браузерами с 14.02.2023.

Для браузеров без поддержки лучше предоставить безопасный дефолтный макет (такой как стековый вертикальный макет) и прогрессивно улучшать его с помощью медиа-запросов, когда общий размер экрана позволяет это:

```css
/* Дефолтный безопасный стековый макет */
.card {
  display: flex;
  flex-direction: column;
}

/* Резерв с помощью медиа-запросов для старых браузеров */
@media (min-width: 600px) {
  .card {
    flex-direction: row;
  }
}

/* Перезаписываем запросами к контейнеру при наличии их поддержки */
@supports (container-type: inline-size) {
  @media (min-width: 600px) {
    .card {
      /* Сбрасываем резерв медиа-запроса, если нужно, или позволяем запросу к контейнеру обработать это */
      flex-direction: column;
    }
  }

  @container (min-width: 400px) {
    .card {
      flex-direction: row;
    }
  }
}
```
