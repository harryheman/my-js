# Визуальная стабилизация резервного шрифта

После загрузки веб-шрифтов, они часто заменяют резервный шрифт, который имеет другие размеры, даже если обоим шрифтам установлено одинаковое значение свойства `font-size`. Это вызывает "сдвиг макета" (CLS) и может сделать текст неразборчивым, если буквы нижнего регистра (`x-height` - высота символа "x") резерва сильно отличаются от букв предпочтительного шрифта.

Свойство `font-size-adjust` решает эту проблему путем нормализации размера шрифта на основе конкретной метрики (обычно, `x-height`), гарантируя, что текст занимает одинаковое визуальное пространство, независимо от того, какой шрифт сейчас активен.

## Реализация

### 1. Определение соотношения сторон предпочтительного шрифта

Для нормализации резервного шрифта требуется "значение соотношения" (aspect value) (отношение высоты строчных букв к размеру шрифта) основного шрифта.

* Автоматическое определение (рекомендуется): используйте ключевое слово `from-font`, чтобы браузер мог извлечь отношение из основного шрифта.
* Ручное вычисление: если значение отношения известно (например, 0.545 для Verdana), его можно указать явно для более точного контроля.

### 2. Применение `font-size-adjust` к текстовому контейнеру

Примените свойство к элементу или его родительскому контейнеру. Это гарантирует, что при загрузке основного шрифта или ее провале, резервный шрифт будет масштабирован для совпадения с визуальным размером основного шрифта:

```css
.text-content {
  /* Определяем стек шрифтов как обычно */
  font-family: "MyWebFont", "Arial", sans-serif;
  font-size: 1rem;

  /* Нормализуем размер шрифта на основе `x-height` основного шрифта.
     Это гарантирует, что если 'Arial' используется в качестве резерва, он масштабируется
     для совпадения с `x-height` 'MyWebFont'. */
  font-size-adjust: from-font;
}
```

### 3. Корректировка с учетом конкретных показателей (опционально)

Хотя `x-height` - это дефолтное и рекомендуемое значение, нормализация может выполняться на основе других метрик, таких как `cap-height` (полезно для заголовков в верхнем регистре) или `ch-width` (полезно для моноширинных шрифтов):

```css
h1 {
  /* Нормализация на основе высоты заглавных букв */
  font-size-adjust: cap-height from-font;
}
```

### 4. Проверка визуальной стабильности

Убедитесь, что значение `font-size-adjust` правильно стабилизирует резерв. Это можно сделать путем временной блокировки основного шрифта или изменения объявления `font-family` в инструментах разработчика в браузере и проверки того, что макет текста остается стабильным.

## Поддержка и страховка

Свойство `font-size-adjust` поддерживается всеми основными браузерами с 25.07.2024.

В браузерах без поддержки шрифт будет рендериться в дефолтном масштабе. Это может привести к сдвигам макета или ухудшению читаемости во время смены шрифта.

Для уменьшения этих негативных эффектов без `font-size-adjust`, можно использовать дескрипторы `@font-face` - `size-adjust`, `ascent-override` и `descent-override` для ручной кастомизации резервных шрифтов, но их значения более сложны для вычисления, чем значение `font-size-adjust`.
