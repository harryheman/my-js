# Точное выравнивание текста

## Проблема

Браузеры автоматически добавляют дополнительные пробелы над и под текстовыми символами, чтобы учесть высоту строки и специфичные для шрифта показатели, такие как верхние и нижние элементы. Это "призрачное пространство" делает невозможным достижение идеального вертикального выравнивания с использованием стандартного CSS.

Распространенные проблемы:

- Невыровненные иконки: текст визуально находится ниже или выше соседней иконки, даже при использовании `align-items: center`.
- Неточный паддинг: кнопка с `padding: 12px` визуально имеет больше пространства сверху или снизу из-за внутреннего интерлиньяжа шрифта.
- Хаки выравнивания: верх заглавной буквы невозможно точно выровнять по верху контейнера или соседнего изображения без использования "магических значений" отрицательных отступов.

## Решение

Свойства `text-box-trim` и `text-box-edge` (сокращение `text-box`) позволяют обрезать этот внутренний интерлиньяж на основе конкретных показателей шрифта. Обрезая текстовое поле до `cap` (верхняя часть заглавных букв) и `alphabetic` (нижняя часть большинства букв), можно гарантировать, что ограничивающая рамка элемента соответствует его визуальному содержимому.

### Стратегия реализации

1. Применяем `text-box-trim: trim-both` (или сокращение `text-box`) к элементу, содержащему текст.
2. Определяем метрики, используемые для обрезки, с помощью `text-box-edge`. В большинстве случаев подойдет `cap alphabetic`.
3. Используем это для достижения визуального вертикального центрирования в контейнерах `flex` или `grid`.
4. Используем это для того, чтобы значения `padding` совпадали с визуальным отступом между текстом и краем контейнера.
5. Не используем это в длинном основном тексте, где для удобства чтения необходим традиционный межстрочный интервал. Это лучше всего подходит для заголовков, кнопок и подписей.

## Руководство по реализации

### Случай 1: обрезка внутреннего интерлиньяжа для баджей (badges)

Разные шрифты имеют разное количество встроенных пробелов над или под текстом. Это может вызывать несовпадения с дизайном или проблемы с визуальным центрированием текста в бадже. Для точного охвата текста паддингами контейнера следует использовать `text-box: trim-both cap alphabetic`. Это особенно полезно для плотных компонентов UI, таких как баджи или теги. Это позволяет паддингу начинаться сразу за краями текста со всех сторон:

```css
.badge {
  padding: 10px;
  background: hotpink;
  border-radius: 10px;
  /*
    Обрезаем верх до высоты заглавных букв, а
    низ - до базовой линии алфавита
  */
  text-box: trim-both cap alphabetic;
}
```

### Случай 2: центрирование текста с иконками

При использовании Flexbox для выравнивания текста и иконок, "призрачное пространство" часто заставляет текст выглядеть расположенным немного не по центру. Обрезка блока заставляет движок макета использовать реальную видимую высоту символов для выравнивания:

```css
.button {
  display: inline-flex;
  align-items: center;
  gap: 8px;
}
/*
  Свойство `text-box` не наследуется и должно применяться прямо на текстовом элементе
*/
.button-text{
  /*
    Гибкий контейнер теперь центрируется относительно
    видимых символов, а не относительно невидимого блока шрифта
  */
  text-box: trim-both cap alphabetic;
}
```

### Случай 3: выравнивание текста с верхними краями

Для выравнивания заголовка точно по верху соседнего изображения или декоративного элемента следует использовать `trim-start cap alphabetic`. Даже если низ не обрезается, нижний край все равно должен быть определен:

```css
.hero {
  display: flex;
  align-items: flex-start;
}

h1 {
  /* Нижний край должен быть определен, хотя обрезается только верх */
  text-box: trim-start cap alphabetic;
}
```

## Лучшие практики

- Используйте сокращение `text-box` для краткости: `text-box: <trim-direction> <edges>`.
- Всегда определяйте оба края, даже если обрезается только один (до тех пор, пока не используется дефолтный край `text`).
- Комбинируйте с `line-height` для управления пространством. Обрезка применяется к первой и последней строкам текста, но `line-height` по-прежнему влияет на расстояние между строками в многострочном тексте.
- Не применяйте к каждому элементу. Используйте только когда точное выравнивание является обязательным условием.

### Поддержка и страховка

`text-box` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 133+, Edge 133+ (с февраля 2025 года) и Safari 18.2+ (с декабря 2024 года). Не поддерживается в Firefox.

`text-box` - это прогрессивное улучшение. В браузерах без поддержки текст рендерится с дефолтным интерлиньяжем. Макет остается функциональным, хотя и менее точным. Специальный резерв не требуется, неподдерживаемые свойства просто игнорируются.

## Другие замечания

1. Метрики шрифта: разные шрифты имеют разные внутренние метрики. `cap alphabetic` это надежное дефолтное значение, но некоторые шрифты или случаи использования могут потребовать использования значения `ex` (высота символа "x") для лучшего выравнивания нижнего регистра.
2. Многострочный текст: обрезка применяется к первой и последней строкам блока. Внутренние строки остаются нетронутыми.
