# Уменьшение дублирования стилей с помощью функций CSS

Поддержка больших таблиц стилей часто приводит к повторяющейся логике, особенно при работе с токенами дизайн-системы, вроде градиентов или шаблонов отзывчивого макета.

CSS-правило `@function` позволяет инкапсулировать эту логику в повторно используемые, параметризованные функции, что делает CSS более поддерживаемым, согласованным и DRY (Don't Repeat Yourself - не повторяй себя).

## Синтаксис `@function`

Кастомная функция определяется с помощью правила `@function`, названия, начинающегося с двух дефисов, и списка параметров. Функция возвращает значение с помощью свойства `result`:

```css
@function --my-function(--input1 <length>, --input2: default-value) returns <length> {
  /* Логика */
  result: var(--input1);
}
```

### Основные концепции

- Параметры: должны начинаться с двойного дефиса (`--`).
- Значения по умолчанию: после двоеточия (`:`) можно указывать дефолтные значения.
- Результат: свойство `result` определяет значение, возвращаемое функцией. Побеждает последнее `result`, определенное в теле функции.
- Область видимости: параметры и переменные, определенные в функции, имеют локальную область видимости.
- Типы: типы параметров и возвращаемого значения можно определять с помощью треугольных скобок (например, `<color>`). Для определения нескольких возможных типов используется функция `type` (например, `type(<number> | <percentage>)`).

## Практические примеры

### 1. Токены дизайн-системы (градиенты)

Обеспечиваем согласованные цвета градиента во всем приложении путем инкапсуляции логики градиента. `--angle` имеет дефолтное значение, которое может быть перезаписано:

```css
@function --fancy-gradient(--start-color <color>, --end-color <color>, --angle: 98deg) returns <image>{
  result: linear-gradient(in oklab var(--angle), var(--start-color), var(--end-color) );
}

.card {
  background: --fancy-gradient(#ed73d7, #5d87e9);
}
```

### 2. Условная логика макета

Для возврата разных значений на основе окружения можно использовать `@media` или другие запросы прямо внутри функции. При использовании условной логики в функции, обратите внимание, что `@function` не "возвращается" на первом значении `result`, но следует по каскаду CSS и разрешается последним значением, совпадающим с размером экрана, размером контейнера или другим запросом:

```css
@function --grid-template(--count <number>){
  /* Сначала указываем дефолтное значение */
  result: 1fr; /* По умолчанию стек */
  @media (min-width: 800px) {
    result: repeat(var(--count), 1fr); /* Сетка на больших экранах */
  }
}

main {
  display: grid;
  grid-template-columns: --grid-template(2);
}
```

## Лучшие практики

- Используйте правильные префиксы: названия функций и параметры всегда должны начинаться с двойного дефиса `--`.
- Предоставляйте дефолтные значения: разумные значения по умолчанию делают функции более надежными.
- Сохраняйте простоту: используйте функции для повторяющейся или сложной логики. Не инкапсулируйте простые пары "свойство-значение".
- Используйте типы: обеспечьте, чтобы параметры и возвращаемые значения имели ожидаемые типы.
- Рассмотрите возможность предварительной компиляции: для функций, которые не зависят от пользовательского ввода, медиа-запросов или других клиентских вещей, рассмотрите возможность использования предварительной компиляции CSS во избежание выполнения ненужной работы на клиенте.

### Поддержка и страховка

Правило `@function` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 139+ и Edge 139+ (с августа 2025 года). Не поддерживается в Firefox и Safari.

В браузерах без поддержки, значения, установленные с помощью функций CSS, будут невалидными. Для поддержки таких браузеров сначала предоставьте резервное значение для свойства. Оно будет перезаписано в браузерах, поддерживающих функции CSS:

```css
.card {
  /* Резерв, чистый цвет */
  background: #5d87e9;
  background: --fancy-gradient(#ed73d7, #5d87e9);
}

main {
  /* Резерв, простой стек */
  grid-template-columns: 1fr;
  grid-template-columns: --grid-template(2);
}
```
