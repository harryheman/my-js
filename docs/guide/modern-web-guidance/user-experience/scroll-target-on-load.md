# Установка цели прокрутки при загрузке страницы

Свойство CSS `scroll-initial-target` предоставляет декларативный способ размещения определенного дочернего элемента в видимой области его прокручиваемого контейнера после рендеринга контейнера с помощью одного CSS. Раньше разработчики полагались на JavaScript (`Element.scrollIntoView()`) или идентификаторы фрагмента URL (`#item-id`), имеющие ограничения и сложные в реализации.

## Реализация

1. Целевой элемент должен находиться внутри прокручиваемого контейнера (элемент с переполнением (overflow), разрешающим прокрутку, таким как `overflow: auto`). Это может быть любой предок, включая корневой элемент `<html>`.
2. Примените `scroll-initial-target: nearest` к дочернему элементу, который должен быть прокручен в область просмотра.

## Пример кода: вертикальная новостная лента

В этом примере лента начинается прокрученной к определенному элементу, а не с самого начала списка:

```css
/**
 * Цель: элемент, который должен быть видим при начальной загрузке
 */
.item.target {
  scroll-initial-target: nearest;
}
```

## Стратегическая реализация и лучшие практики

- Используйте `scroll-initial-target` для опыта "начала с середины", такого как календарь, начинающийся с текущего дня, или галерея, начинающаяся с определенного изображения.
- Не путайте это с фокусом доступности. Это свойство перемещает только визуальную область просмотра; оно не перемещает клавиатурный фокус. Нужно вручную вызывать `element.focus()`, если цель должна быть начальной точкой для пользователей, просматривающих страницу с помощью клавиатуры.
- Не используйте это, если требуется плавная анимация прокрутки при загрузке страницы; это свойство является дискретным и устанавливает положение прокрутки мгновенно на стадии компоновки.
- Не устанавливайте `scroll-initial-target` на нескольких элементах в одном прокручиваемом контейнере. Если несколько элементов определяют `scroll-initial-target: nearest`, браузер выбирает первый в дереве DOM.
- Явно указывайте размеры медиа. Поскольку положение прокрутки вычисляется при начальной компоновке, убедитесь, что изображения или видео имеют `aspect-ratio` или фиксированную `height`/`width` во избежание смещения цели после загрузки медиа.
- Учитывайте иерархию приоритетов: фрагмент URL (например, `example.com/#top`) и свойство уровня контейнера `scroll-start` имеют приоритет над `scroll-initial-target`.

## Поддержка и страховка

`scroll-initial-target` имеет ограниченную поддержку браузеров. Поддерживается в Chrome 133+ и Edge 133+ (с февраля 2025 года). Не поддерживается в Firefox и Safari.

В браузерах без поддержки следует использовать резерв на JavaScript. Используйте событие `DOMContentLoaded` для обеспечения прокрутки элемента браузером сразу после завершения парсинга HTML, что обеспечивает более быстрый опыт, чем ожидание загрузки всех изображений и других ресурсов. В качестве альтернативы, скрипт можно разметить в конце элемента `<body>` - в этом случае обработчик событий не нужен:

```javascript
document.addEventListener("DOMContentLoaded", () => {
  // Проверяем нативную поддержку
  if (!CSS.supports("scroll-initial-target", "nearest")) {
    const feedTarget = document.querySelector(".item.target");
    if (feedTarget) {
      // 'block: center' обеспечивает выравнивание медиа по центру области просмотра
      feedTarget.scrollIntoView({ behavior: "instant", block: "center" });
    }
  }
});
```
