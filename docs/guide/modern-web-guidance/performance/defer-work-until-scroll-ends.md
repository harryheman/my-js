# Задержка выполнения работы до завершения прокрутки

Прокрутка должна быть плавной и отзывчивой. Выполнение тяжелых задач, таких как пересчеты макета, отправка аналитических данных (beacons) или обновление DOM в процессе прокрутки может подавлять основной поток, что приводит к потере кадров и засорению макета.

Исторически разработчики использовали задержку обработки события `scroll` с помощью `setTimeout()` для "угадывания" момента завершения прокрутки. Однако, такие функции заведомо ненадежны. Они могут сработать во время прокрутки.

Событие `scrollend` предлагает надежное производительное решение. Браузер вызывает событие `scrollend` точно в момент окончания прокрутки и всех переходов (transitions) и завершения жеста касания.

## Реализация

Для реализации шаблона отложенной работы:

1. Определите прокручиваемый контейнер: создайте контейнер с `overflow: auto` или `overflow: scroll`.
2. Обрабатывайте события `scroll`: используйте этот обработчик только для базовых динамических метрик и информативной стилизации макета. Не выполняйте тяжелую работу здесь.
3. Обрабатывайте события `scrollend`: зарегистрируйте коллбэк `scrollend` на прокручиваемом контейнере или самом документе.
4. Выполняйте дорогую работу в коллбэке: вот где можно безопасно запрашивать (fetch) динамичный контент или запускать сложные компоновки DOM.

## Пример кода

```css
.scroll-container {
  height: 300px;
  overflow-y: auto;
}
```

```javascript
const scroller = document.querySelector('.scroll-container');

// 1. Информативная обратная связь в процессе прокрутки
scroller.addEventListener('scroll', () => {
  // Избегайте выполнения тяжелых операций здесь
  console.log('Прокрутка... обновления отложены');
});

// 2. Коллбэк, запускаемый после завершения прокрутки
scroller.addEventListener('scrollend', () => {
  // Здесь можно спокойно выполнять дорогие операции
  const currentVisibleSection = findMostVisibleSection(scroller);
  fetchAdditionalData(currentVisibleSection);
});
```

## Стратегическая реализация и лучшие практики

- Используйте событие `scrollend`, вместо отложенных событий `scroll`, для отправки аналитических данных или получения нового контента динамически.
- Также используйте мгновенные (snap) взаимодействия `scrollSnapChange` или `scrollSnapChanging` при разработке каруселей или галерей слайдов.
- Не выполняйте динамические обновления макета внутри динамических визуальных коллбэков прокрутки.
- Событие `scrollend` вызывается не только прокруткой области просмотра, но и ее масштабированием.

## Поддержка и страховка

Событие `scrollend` поддерживается всеми основными браузерами с 12.12.2025.

Для старых браузеров можно использовать отложенное событие `scroll` с `setTimeout` для отправки (dispatch) кастомного события `scrollend`:

```javascript
function initializeDemo() {
  const scroller = document.querySelector('#scroller');
  scroller.addEventListener('scrollend', () => {
    // Безопасное выполнение тяжелой операции
  });
}

if ('onscrollend' in window) {
  initializeDemo();
} else {
  initializeDemo();
  const scroller = document.querySelector('#scroller');
  scroller.addEventListener('scroll', () => {
    clearTimeout(window.scrollendtimer);
    window.scrollendtimer = setTimeout(() => {
      scroller.dispatchEvent(new CustomEvent('scrollend'));
    }, 100);
  });
}
```
