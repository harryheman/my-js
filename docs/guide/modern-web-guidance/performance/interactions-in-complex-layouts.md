# Оптимизаций взаимодействия в сложных макетах

Поддерживайте высокую частоту кадров (60 кадров в секунду) и устраняйте задержки взаимодействий во время перетаскивания или тяжелых изменений в сложных макетах с несколькими столбцами, таких как доски Канбан или массивные сетки данных.

## Обзор

В сложных макетах незначительная модификация одного элемента, например, перетаскивание карточки или редактирование ячейки, может вызвать цепную реакцию вычислений стиля и макета, которая вынудит браузер перекомпоновать всю страницу. Это приводит к потере кадров и высокой задержке взаимодействия со следующей отрисовкой (INP).

Применение `content-visibility: auto` к автономным областям макета (таким как колонки в доске Канбан) позволяет изолировать рендеринг.

### Механизм для элементов на экране

Важно понимать, как `content-visibility: auto` влияет на элементы, видимые на экране:

* Браузер не пропускает рендеринг видимых элементов.
* Выигрыш в производительности достигается за счет ограничений (containments) CSS, которые это свойство автоматически обеспечивает (т.е. макета, стиля и отрисовки).
* Это сдерживание (containment) действует как граница. Если мутация происходит внутри контейнера с примененным сдерживанием, браузер знает, что изменения не могут повлиять на геометрию или стили элементов вне этого контейнера. Перекомпоновка страницы изолирована, что предотвращает глобальный пересчет макета.

## Реализация

### 1. Определение областей сдерживания

Примените `content-visibility: auto` к большим автономным контейнерам, представляющим изолированные единицы макета (например, колонки сетки, списки досок):

```css
.board-column {
  /* Применяем границы сдерживания */
  content-visibility: auto;

  /* Предоставляем заполнитель размера для предотвращения сдвигов макета.
     Для вертикальной колонки нужно определить разумную ширину и высоту.
     - 'auto' является опциональным и позволяет браузеру запоминать реальный размер элемента
       после рендеринга. Оно должно использоваться вместе со значением <length>
       для первого рендеринга.
     - '300px' - примерная ширина элемента. Это может быть любое валидное
      значение <length>.
     - '800px' - примерная высота элемента. Это может быть любое валидное
      значение <length>.
   */
  contain-intrinsic-size: auto 300px auto 800px;
}
```

### 2. Управление взаимодействиями

Убедитесь, что взаимодействия, происходящее внутри колонки, выигрывают от сдерживания:

```javascript
// Пример: перетаскивание элемента
function moveItemToColumn(itemId, columnId) {
  const item = document.getElementById(itemId);
  const column = document.getElementById(columnId);

  // Браузер перерисовывает только определенную колонку,
  // а не всю доску!
  column.appendChild(item);
}
```

### Поддержка и страховка

Свойство `content-visibility` поддерживается всеми основными браузерами с 15.09.2025.

В браузерах без поддержки:

* `content-visibility` игнорируется, мутации вызывают стандартную глобальную перекомпоновку.
* Для достижения похожей изоляции в старых браузерах можно применить сдерживание вручную:

```css
@supports not (content-visibility: auto) {
  .board-column {
    /* Резерв для сдерживания */
    contain: layout style paint;
  }
}
```
