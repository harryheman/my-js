# Оптимизация приоритета предварительной загрузки

Предварительная загрузка ресурсов с помощью `<link rel="preload">` сигнализирует браузеру, что ресурс скоро понадобится. Однако, предварительная загрузка наследует дефолтный приоритет загрузки типа ресурса, который для изображений, например, является низким. Использование `fetchpriority` позволяет переопределить этот дефолтный приоритет, обеспечивая, в частности, высокий приоритет загрузки изображения, позволяя загрузке начаться раньше и получить больше пропускной способности.

## Реализация

1. Определите кандидатов для предварительной загрузки: найдите ресурсы, которые поздно обнаруживаются браузером (например, постеры видео или фоновые изображения в CSS), но являются важными для начального вида страницы.
2. Повысьте приоритет важных предзагрузок изображений: для изображения LCP или других критических изображений, которые не приоритизированы по умолчанию, используйте `<link rel="preload" fetchpriority="high">` для повышения их приоритета среди прочих предзагрузок. Обратите внимание, что приоритет загрузки шрифтов высокий по умолчанию.
3. Понизьте приоритет незначительных предзагрузок: для ресурсов, которые не являются критичными для начального рендеринга (например, фоновое видео или второстепенный шрифт), используйте `<link rel="preload" fetchpriority="low">`.
4. Учитывайте тип ресурса: разные типы `as` имеют разные дефолтные приоритеты; используйте `fetchpriority` для их перезаписи при необходимости.

## Пример кода

```html
<!-- Повышаем приоритет постера видео, который является кандидатом на роль LCP -->
<link rel="preload" href="/images/video-poster.jpg" as="image" fetchpriority="high">

<!-- Повышаем приоритет критичного изображения LCP, "скрытого" в CSS -->
<link rel="preload" href="/images/hero-background.jpg" as="image" fetchpriority="high">

<!-- Понижаем приоритет вторичного шрифта во избежание сетевых конфликтов -->
<link rel="preload" href="/fonts/secondary-font.woff2" as="font" type="font/woff2" fetchpriority="low" crossorigin>
```

## Лучшие практики

- Используйте `fetchpriority="high"` только для предзагрузок 1-2 критичных изображений во избежание сетевых конфликтов и снижения приоритета.
- Используйте `fetchpriority="high"` для изображений, определяющих показатель отрисовки наибольшего контента (LCP).
- Ограничивайте количество предзагрузок на странице, максимум, 2 изображениями и 2-3 другими ресурсами во избежание пробем с полосой пропускания.
- Используйте `fetchpriority="low"` для предзагрузок, которые должны запускаться раньше, но не за счет критичных ресурсов (например, некритичных шрифтов).
- Правильно определяйте атрибут `as` для корректной работы предзагрузки.
- Старайтесь делать так, чтобы критичные ресурсы (такие как изображения LCP) можно было обнаружить в HTML через теги `<img>`, вместо предзагрузки фоновых изображений.
- Убедитесь, что предзагрузки шрифтов всегда содержат атрибут `crossorigin` (даже если они загружаются из того же источника).
- Не используйте устаревший атрибут `importance`. Он был заменен `fetchpriority` и не поддерживается ни одним браузером.

## Поддержка и страховка

Атрибут `fetchpriority` поддерживается всеми основными браузерами с 29.10.2024.

Атрибут `fetchpriority` на `<link rel="preload">` - это прогрессивное улучшение. Браузеры без его поддержки будут предварительно загружать ресурсы с дефолтным приоритетом для этого типа ресурса. Для обеспечения обратной совместимости всегда предоставляйте правильные атрибуты `as` и `type`.
