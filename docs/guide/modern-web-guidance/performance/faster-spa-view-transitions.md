# Ускорение переходов отображения SPA с помощью кэширования состояния

Реализуйте мгновенную навигацию между отображениями в одностраничном приложении (SPA) путем кэширования рендеринга неактивных отображений, вместо их уничтожения.

## Обзор

Как правило, при переключении пользователя между вкладками или отображениями в SPA, разработчики либо уничтожают старое отображение, либо скрывают его с помощью `display: none`. Оба подхода заставляют браузер повторно создавать или вычислять полный макет и отрисовывать его при возвращении пользователя к этому отображению.

При применении `content-visibility: hidden` к неактивным отображениям, браузер удаляет контент элемента из потока компоновки и перестает его рисовать, но сохраняет его кэшированное состояние рендеринга в памяти. Когда пользователь возвращается назад, отображение восстанавливается почти мгновенно.

### Компромисс между ЦП и ОЗУ

Хотя этот подход сильно улучшает производительность, он сопряжен с определенным компромиссом, который следует принимать во внимание:

* Экономия ЦП: большая. Браузер полностью пропускает стадии компоновки и отрисовки скрытых отображений.
* Расходы ОЗУ: высокие. Браузер сохраняет все узлы DOM, обработчики событий и состояние скрытых отображений в памяти.

#### Когда этот компромисс становится опасным

* Используйте эту стратегию для простых приложений с небольшим предсказуемым количеством отображений (например, интерфейс с 3-5 вкладками).
* Не используйте безусловное кэширование каждого отображения в высокодинамичных приложениях, генерирующих множество уникальных отображений или использующих бесконечные динамические роуты. Это в конечном итоге приведет к серьезному раздуванию памяти и может привести к сбою браузера на слабых устройствах.
* Если приложение высокодинамично, следует реализовать стратегию "выселения" (eviction) (такую как LRU) для уничтожения старых отображений при достижении порога памяти.

## Реализация

### 1. Настройка контейнеров отображения

Установите `content-visibility: hidden` на неактивных отображениях.

> Свойство `content-visibility: hidden` скрывает контент элемента, но сам элемент остается стилизованным и видимым. Его фон, границы и отступы по-прежнему рисуются браузером.

```css
.spa-view.inactive {
  /* Используем hidden для кэширования рендеринга неактивных отображений */
  content-visibility: hidden;

  /* Опционально: чтобы скрытые отображения не занимали физическое пространство в потоке компоновки */
  position: absolute;
}
```

### 2. Управление фокусом

При переключении отображений, убедитесь, что клавиатурный фокус работает ожидаемо для обеспечения доступности:

```javascript
function switchToView(viewId) {
  // Скрываем все отображения
  document.querySelectorAll('.spa-view').forEach(view => {
    view.classList.add('inactive');
    view.setAttribute('aria-hidden', 'true');
  });

  // Показываем целевое отображение
  const activeView = document.getElementById(viewId);
  activeView.classList.remove('inactive');
  activeView.setAttribute('aria-hidden', 'false');

  // Перемещаем фокус на новое отображения для обеспечения логичного порядка переключения фокуса
  activeView.focus();
}
```

### Поддержка и страховка

Свойство `content-visibility` поддерживается всеми основными браузерами с 15.09.2025.

В браузерах без поддержки:

* `content-visibility` игнорируется;
* для предотвращения одновременного рендеринга всех отображений в старых браузерах следует использовать такой резерв:

```css
@supports not (content-visibility: hidden) {
  .spa-view.inactive {
    display: none;
  }
}
```
