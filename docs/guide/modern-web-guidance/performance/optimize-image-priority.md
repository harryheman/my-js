# Оптимизация приоритета загрузки изображений

Браузеры используют различные эвристики для назначения приоритетов загрузки изображениям, но эти значения могут не всегда соответствовать отрисовке наибольшего контента (LCP) вашей страницы. Использование `fetchpriority` в элементе `<img>` позволяет явно сообщить браузеру о важности изображения, гарантируя, что такие изображения загружаются быстрее, а второстепенные не конкурируют за полосу пропускания.

## Реализация

1. Определите изображение LCP: определите, какое изображение является главным кандидатом на роль LCP (как правило, это основное (hero) изображение в начале/наверху страницы).
2. Повысьте приоритет LCP: добавьте `fetchpriority="high"` к элементу `<img>` - главному кандидату на роль LCP.
3. Понизьте приоритет второстепенных изображений: добавьте `fetchpriority="low"` к изображениям, которые являются частью второстепенного UI или появляются только после действия пользователя (многоуровневые меню, модальные окна, слайды карусели за пределами экрана).
4. Оптимизируйте ленивую загрузку: никогда не используйте `loading="lazy"` на изображении LCP. Для обычных изображений за пределами экрана `loading="lazy"` - эффективный способ отложить запрос, пока пользователь не прокрутит страницу близко к ним. Не добавляйте к таким изображениям `fetchpriority="low"`, поскольку они должны загружаться с нормальным приоритетом, когда пользователь до них докрутит. Используйте `fetchpriority="low"` для изображений, которые технически находятся на экране, но не видны изначально (например, скрытые слайды карусели или многоуровневые меню). Для таких изображений допустимо также использовать `loading="lazy"`; браузер выполнит запрос в нужное время, соблюдая низкий приоритет.
5. Старайтесь использовать дефолтные приоритеты: если изображение должно иметь нормальный приоритет загрузки, опустите атрибут `fetchpriority`, вместо установки его значения в `auto`. Это стилистическое соглашение для обеспечения более чистого HTML и использования нативных эвристик браузера.

## Пример кода

```html
<!-- Повышаем приоритет изображения LCP -->
<img src="/images/hero-lcp.jpg"
     alt="Main Banner"
     fetchpriority="high"
     width="800" height="400">

<!-- Понижаем приоритет скрытых изображений за пределами экрана -->
<img src="/images/gallery-alt.jpg"
     alt="Gallery Image"
     fetchpriority="low"
     width="400" height="300">

<!-- Понижаем приоритет изображений, появляющихся только после действия пользователя -->
<img src="/images/mega-menu-promo.jpg"
     alt="Special Offer"
     fetchpriority="low"
     width="300" height="150">

<!-- Используем только ленивую загрузку для обычных изображений за пределами экрана -->
<img src="/images/footer-logo.png"
     alt="Footer Logo"
     loading="lazy"
     width="120" height="60">

<!-- Опускаем fetchpriority для изображений со стандартным приоритетом -->
<img src="/images/standard-image.jpg"
     alt="Standard Image"
     width="400" height="300">
```

## Лучшие практики

- Всегда применяйте `fetchpriority="high"` к изображению LCP.
- Используйте `fetchpriority="high"` только для 1-2 критичных изображений во избежание сетевых конфликтов и снижения приоритета.
- Используйте `fetchpriority="low"` для изображений, которые технически находятся на экране, но скрыты изначально (например, скрытые слайды карусели, изображения в многоуровневых меню).
- Не используйте `fetchpriority="low"` для стандартных изображений за пределами экрана, которые используют `loading="lazy"`. Эти изображения должны загружаться с нормальным приоритетом при попадании в область видимости.
- Не рекомендуется использовать `fetchpriority="auto"`. Для дефолтного приоритета достаточно полностью опустить этот атрибут.
- Не объединяйте `fetchpriority="high"` с `loading="lazy"` для изображения LCP.
- Не используйте устаревший атрибут `importance`. Он был заменен `fetchpriority` и не поддерживается ни одним браузером.

## Поддержка и страховка

Атрибут `fetchpriority` поддерживается всеми основными браузерами с 29.10.2024.

`fetchpriority` - это прогрессивное улучшение для элемента `<img>`. Браузер просто игнорирует его при отсутствии поддержки и использует дефолтный приоритет.
