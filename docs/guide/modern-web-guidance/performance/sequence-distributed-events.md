# Упорядочение распределенных событий

Высокочастотное отслеживание и логирование событий в распределенных системах требует точных временных меток (timestamps) для обеспечения правильного причинно-следственного порядка. Стандартный метод JavaScript `Date.now()` предоставляет миллисекундную точность, что может привести к конфликтам временных меток, когда несколько событий возникает в одну и ту же миллисекунду.

`Temporal.Instant` предоставляет наносекундную точность, обеспечивая точную последовательность событий без коллизий.

## Реализация

Для упорядочения высокочастотных событий с помощью `Temporal`:

1. Захватите точную временную метку: используйте `Temporal.Now.instant()` для получения точного текущего времени в наносекундах.
2. Сортируйте события по хронологии: используйте `Temporal.Instant.compare(a, b)` для сортировки объектов событий. Этот метод выполняет сортировку на наносекундном уровне.
3. Вычисляйте задержку: используйте `Temporal.Instant.prototype.since(other)` для определения точной разницы между событиями.
4. Сериализуйте для передачи: используйте `Temporal.Instant.prototype.toString()` для преобразования временной метки в стандартную строку ISO-8601 для логирования или передачи по сети.

## Пример кода

```javascript
// 1. Захватываем временные метки для входящих событий
function recordEvent(eventType, nodeId) {
  return {
    nodeId,
    eventType,
    timestamp: Temporal.Now.instant() // Наносекундная точность
  };
}

// 2. Сортируем события по хронологии
function sequenceEvents(events) {
  // Всегда используйте Temporal.Instant.compare для сортировки мгновений
  return [...events].sort((a, b) => Temporal.Instant.compare(a.timestamp, b.timestamp));
}

// 3. Вычисляем задержку между событиями
function analyzeTelemetry(sortedEvents) {
  for (let i = 1; i < sortedEvents.length; i++) {
    const prev = sortedEvents[i - 1];
    const curr = sortedEvents[i];

    // Вычисляем разницу в нс
    const duration = curr.timestamp.since(prev.timestamp);
    const nsDiff = duration.total('nanoseconds');

    console.log(`Разница между событием ${prev.eventType} и событием ${curr.eventType}: ${nsDiff}ns`);
  }
}
```

## Стратегическая реализация и лучшие практики

- Используйте `Temporal.Now.instant()` для серверного отслеживания или клиентской телеметрии, когда миллисекундной точности недостаточно (например, микросекундное профилирование).
- Не используйте `Date.now()`, когда требуется стабильная сортировка событий, следующих одно за другим.
- Не используйте `Temporal.Instant` для отображения времени на часах для пользователя, если оно не связано с часовым поясом (используйте `Temporal.ZonedDateTime` для локализованного отображения).
- Убедитесь в поддержке `Temporal` окружением перед его использованием или предусмотрите страховку.

## Поддержка и страховка

Temporal имеет ограниченную поддержку браузеров. Поддерживается в Chrome 144+, Edge 144+ (с января 2026 года) и Firefox 139+ (с мая 2025 года). Не поддерживается в Safari.

Для сред без поддержки используйте соответствующий стандартам полифилл, такой как `@js-temporal/polyfill`. Загружайте его условно во избежание раздувания полезной нагрузки для современных клиентов. Обратите внимание, что `@js-temporal/polyfill` не устанавливает глобальный объект `Temporal` автоматически, это нужно делать вручную при необходимости.

```javascript
(async () => {
  // Проверяем нативную поддержку
  if (typeof Temporal === 'undefined') {
    // Динамически загружаем ESM-совместимый полифилл из CDN
    const module = await import('https://esm.sh/@js-temporal/polyfill');
    // Полифилл не устанавливается глобально автоматически,
    // делаем это вручную
    globalThis.Temporal = module.Temporal;
  }

  // Логика приложения
})();
```
