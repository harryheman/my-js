# Условные асинхронные зависимости

`await` верхнего уровня позволяет модулям действовать как асинхронные функции, т.е. выполнение модуля может быть приостановлено в ожидании промиса. Это очень полезно для условной загрузки асинхронных зависимостей, таких как полифиллы или тяжелые опциональные библиотеки.

Верхнеуровневый `await` позволяет инкапсулировать условную логику загрузки внутри одного модуля, эффективно предотвращая выполнение последующих модулей до тех пор, пока зависимость не будет полностью загружена и готова.

## Условная загрузка полифилла

Хотя верхнеуровневый `await` может быть использован для условной загрузки любой асинхронной зависимости, которая является модулем, хорошим случаем его использования является условная загрузка полифилла в браузерах, которые не поддерживают определенную возможность. Этот подход инкапсулирует определение поддержки и динамический импорт внутри одного модуля.

В следующем примере полифилл атрибута `popover` загружается условно, если он отсутствует в `HTMLElement.prototype`:

```javascript
// conditionally-load-polyfill.js

// Проверяем отсутствие поддержки.
// Лучше проверять HTMLElement.prototype, чем window или document,
// для определения поддержки глобального атрибута или свойства DOM,
// такого как popover.
if (!('popover' in HTMLElement.prototype)) {
  // Используем await верхнего уровня для приостановки выполнения любого модуля, который импортировал этот файл,
  // до завершения загрузки и выполнения полифилла.
  await import('/path/to/popover-polyfill.js');
}

// Экспортируем маркер, если нужно
export const polyfillLoaded = true;
```

```javascript
// main.js

// Поскольку в conditionally-load-polyfill.js используется верхнеуровневый await,
// этот импорт заблокирует выполнение main.js, пока полифилл не будет готов.
import './conditionally-load-polyfill.js';

// Теперь можно безопасно использовать возможность (например, отобразить поповер)
const myPopover = document.getElementById('my-popover');
if (myPopover) {
  myPopover.showPopover();
}
```

## Баг `await` верхнего уровня в Safari

Вы должны тщательно структурировать импорт, чтобы избежать ошибки, при которой `await` верхнего уровня не ведет себя ожидаемо в Webkit, что происходит, когда несколько модулей одновременно импортируют модуль, содержащий `await` верхнего уровня:

```javascript
// Не делайте так: одновременный импорт верхнеуровневого await из нескольких модулей
// одного уровня обрушит Safari.
//
// a.js: import './conditionally-load-polyfill.js';
// b.js: import './conditionally-load-polyfill.js';
// main.js: import './a.js'; import './b.js'; // Крах!

// Обеспечьте одну точку входа:
// импортируйте await верхнего уровня один раз в самом начале дерева модулей приложения.
import './conditionally-load-polyfill.js';

// Затем импортируйте остальной код приложения, после разрешения await
import './app.js';
```

## Поддержка

Верхнеуровневый `await` поддерживается в Chrome 89+, Edge 89+ (с марта 2021 года) и Firefox 89+ (с июня 2021 года). С Safari ситуация любопытная. По [данным Can I Use](https://caniuse.com/?search=top+level+await) Safari не поддерживает эту возможность, а согласно руководству от команды Google Chrome - поддерживает, надо только учитывать баг, описанный выше.
