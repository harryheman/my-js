# Понижение приоритета фоновых сетевых запросов

Когда на странице выполняется несколько одновременных запросов, они часто соревнуются за одну полосу пропускания. Незначительные данные, такие как аналитика, логирование или фоновая синхронизация должны иметь более низкий приоритет, чтобы инициированные пользователем или критичные запросы данных могли завершиться быстрее.

## Реализация

1. Определите фоновые запросы: определите, какие вызовы `fetch()` запрашивают незначительные данные, которые не влияют на непосредственный пользовательский опыт.
2. Примените приоритет запроса (fetch priority): добавьте настройку `priority: 'low'` в объект инициализации `fetch()`.

## Пример кода

```javascript
// Используйте высокий приоритет (по умолчанию) для критичных обновлений UI
const criticalData = await fetch('/api/data');

// Явная деприоритизация фоновой аналитики
fetch('/api/analytics', {
  method: 'POST',
  body: JSON.stringify(eventData),
  // Понижение приоритета во избежание сетевых конфликтов
  priority: 'low'
});
```

## Лучшие практики

- Используйте `priority: 'low'` для аналитики, логирования или телеметрии, которые не требуются для текущего отображения.
- Используйте `priority: 'low'` для "предварительного получения" данных, которые могут потребоваться пользователю позже.
- Не используйте `priority: 'low'` для запросов, критичных для опыта пользователя.
- Не используйте устаревший ключ `importance` в объекте настроек `fetch`. Правильный ключ - `priority`.

## Поддержка и страховка

Настройка `priority` поддерживается всеми основными браузерами с 29.10.2024.

`priority` - это прогрессивное улучшение. Браузеры без поддержки будут игнорировать ее и выполнять запросы с дефолтным приоритетом. Для основного функционала не требуется определение поддержки и резервная логика.
