# Отложенный рендеринг тяжелого контента

Веб-страницы с большим количеством содержимого, например, с бесконечной прокруткой, сложными информационными панелями или насыщенными статьями, могут страдать от медленного начального рендеринга и медленного взаимодействия. Современные веб-технологии позволяют отложить рендеринг контента, который не виден сразу, что значительно повышает производительность, не нарушая при этом доступности и ожиданий пользователей.

Для оптимизации рендеринга можно использовать свойство CSS `content-visibility` и атрибут HTML `hidden="until-found"`. Хотя оба предназначены для повышения производительности, они используются в разных случаях.

## Случаи использования

| Сценарий / Пример                                                                 | Возможность                      | Производительность / Выгода                                                                                                                                                            |
|:----------------------------------------------------------------------------------|:---------------------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1. Элемент находится за пределами области просмотра (задержка начальной загрузки) | **`content-visibility: auto`**   | Браузер откладывает компоновку/отрисовку такого элемента до прокрутки контейнера близко к области просмотра, что повышает скорость начальной загрузки страницы.                        |
| 2. Переключение состояния (быстрое переключение отображения)                      | **`content-visibility: hidden`** | Пропускает расчеты макета для скрытых элементов `div`, но сохраняет состояние стилей, позволяя мгновенно переключать отображение без структурных сдвигов (лучше, чем `display: none`). |
| 3. Доступный для поиска и отложенный рендеринг (раскрываемые элементы)            | **`hidden="until-found"`**       | Сочетание производительности рендеринга с возможностью поиска на странице.                                                                                                             |

## Реализация `content-visibility: auto`

### Выбор контента, находящегося за пределами экрана

Следует осторожно выбирать, к каким элементам применять `content-visibility: auto`.
- Это должны быть большие, автономные блоки, которые находятся строго за пределами области просмотра (например, карточки в бесконечной ленте, комментарии к комментариям или разделы с большим количеством дополнительных страниц).
- Не применяйте это свойство к элементам, находящимся в области просмотра при загрузке страницы. Это вынуждает браузер оценивать границы видимости перед рендерингом, что сильно снижает производительность загрузки страницы.
- Выбирайте элементы с глубокими или сложными внутренними структурами DOM для минимизации стоимости рендеринга.

### Шаги реализации

1. Определите тяжелые разделы, находящиеся за пределами экрана при начальной загрузке страницы.
2. Примените `content-visibility: auto` к каждому такому элементу.
3. Укажите примерный размер каждого элемента с помощью `contain-intrinsic-size`.

### Использование `contain-intrinsic-size`

Свойство `content-visibility: auto` должно использоваться вместе с `contain-intrinsic-size`. Если этого не сделать, браузер свернет элемент за пределами экрана до высоты `0px`, что приведет к серьезному смещению макета и перескакиванию полосы прокрутки при прокрутке страницы пользователем.

Свойство `contain-intrinsic-size` действует как заполнитель (placeholder) размеров. Использование ключевого слова `auto` позволяет браузеру "запоминать" точный размер после рендеринга элемента, используя этот размер вместо заполнителя, когда элемент снова выходит за пределы экрана.

### Пример кода

```css
/* Применяйте этот класс только к элементам за пределами начальной области просмотра */
.heavy-section-deferred {
  /* Пропускаем рендеринг элементов, находящихся за пределами экрана */
  content-visibility: auto;

  /* Предоставляем примерный размер для предотвращения сдвигов макета.
    - 'auto' является опциональным и позволяет браузеру запомнить реальный размер элемента
      после рендеринга. Он должен использоваться вместе со значением <length> для
      первого рендеринга.
    - 'none' указывает браузеру не применять внутреннюю ширину к этому элементу.
      Это может использоваться как для значения высоты, так и для значения ширины.
    - '150px' - примерная высота элемента. Может быть любым валидным
      значением <length>.
   */
  contain-intrinsic-size: auto none auto 150px;
}
```

## Реализация `content-visibility: hidden`

1. Определите тяжелые разделы: найдите блоки, находящиеся за пределами экрана (например, скрытые строки в большой таблице).
2. Примените CSS: добавьте `content-visibility: hidden` к каждому такому элементу.
3. Покажите элемент: когда элемент должен быть отображен, измените свойство `content-visibility` на значение `visible` или `auto`.

### Пример кода

```css
.cached-view {
  /* Скрываем контент, но кэшируем состояние рендеринга */
  content-visibility: hidden;
}

.cached-view.is-active {
  content-visibility: visible;
}
```

Поскольку `content-visibility: hidden` исключает элемент и его потомков из дерева доступности и поиска по странице, не используйте его для контента, который должен оставаться обнаруживаемым при скрытии. Если вы хотите, чтобы контент оставался доступным для нативного поиска по странице, используйте `hidden="until-found"`, как описано в руководстве "Поиск скрытого контента".

## Лучшие практики

- Используйте `contain-intrinsic-size` с `content-visibility: auto`. Невыполнение этого требования приводит к перерасчету высоты при прокрутке, что приводит к скачкам макета или визуальным сбоям.
- Не применяйте `content-visibility: auto` к элементам, находящимся в области просмотра при загрузке страницы, поскольку это замедляет рендеринг страницы.
- Доступность: при применении `content-visibility: auto`, убедитесь, что навигация с помощью клавиатуры работает ожидаемо. В некоторых вспомогательных технологиях узлы с `content-visibility: auto` могут быть исключены из дерева доступности или клавиатурной навигации до принудительной установки фокуса на них.

## Поддержка и страховка `content-visibility`

Свойство `content-visibility` поддерживается всеми основными браузерами с 15.09.2025.

Браузер просто игнорирует `content-visibility` при отсутствии поддержки. В большинстве случаев `content-visibility: auto` не нуждается в страховке, но без него производительность становится хуже. Браузеры без поддержки будут оставлять элементы с `content-visibility: hidden` полностью видимыми. Используйте определение поддержки для реализации страховки.

```css
/* Дефолтный стиль для всех */
.inactive {
  display: none;
}

/* Стили для современных браузеров */
@supports (content-visibility: hidden) {
 .inactive {
    display: block; /* Возвращает блочный макет */
    content-visibility: hidden;
  }
}
```
