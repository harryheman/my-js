# Этап 4. Текстовые редакторы: nano и vim

## Зачем нужен текстовый редактор в терминале

В предыдущих главах мы создавали файлы командами `touch` и `echo`, а для просмотра
использовали `cat` и `less`. Но что, если нужно **изменить** файл — исправить строку
в конфигурации, добавить параметр, убрать лишнее?

В Windows вы бы открыли файл в Блокноте. В Linux на сервере графического интерфейса
нет — есть только терминал. Поэтому нужен **текстовый редактор, работающий прямо
в командной строке**.

Таких редакторов существует несколько. Мы разберём два:

- **nano** — простой и понятный, подходит для повседневной работы;
- **vim** — мощный, но со своеобразным управлением. Мы изучим только самый минимум,
  чтобы вы не растерялись, если vim откроется неожиданно.

> **Привязка к проекту.** Совсем скоро (в главе 11) мы будем настраивать nginx
> и PHP-FPM. Для этого потребуется редактировать конфигурационные файлы прямо
> на сервере. Навык работы с текстовым редактором в терминале — обязательный
> инструмент системного администратора.

## nano — основной редактор

### Установка

В Debian 13 nano обычно уже установлен. Проверить можно так:

```bash
nano --version
```

Если команда выведет версию — всё в порядке. Если нет — установите nano командой:

```bash
sudo apt install nano
```

(Команду `sudo` и установку пакетов мы подробно разберём в главах 07 и 08.
Пока просто введите эту команду, если nano не найден.)

### Открытие файла

Чтобы открыть файл для редактирования:

```bash
nano notes.txt
```

Если файл `notes.txt` существует — он откроется для редактирования. Если не
существует — nano создаст новый пустой файл с этим именем (при сохранении).

### Интерфейс nano

Когда вы откроете файл, экран будет выглядеть примерно так:

```
  GNU nano 8.3                    notes.txt

Здесь находится содержимое файла.
Вы можете просто печатать текст.


                         [ Новый файл ]
^G Помощь    ^O Записать  ^W Поиск     ^K Вырезать
^X Выход     ^R Вставить  ^\ Замена    ^U Вставить
```

Экран разделён на три части:

- **верхняя строка** — имя редактора и имя открытого файла;
- **основная область** — содержимое файла, здесь вы редактируете текст;
- **нижние две строки** — подсказки по основным командам.

#### Что означает символ `^`

В подсказках внизу вы видите записи вроде `^X Выход` или `^O Записать`. Символ
`^` означает клавишу **Ctrl**. То есть:

| Запись | Что нажать |
|--------|------------|
| `^X` | `Ctrl+X` |
| `^O` | `Ctrl+O` |
| `^W` | `Ctrl+W` |
| `M-U` | `Alt+U` |

Запись `M-` (Meta) означает клавишу **Alt**.

### Навигация

nano — обычный текстовый редактор, похожий на Блокнот. Перемещаться по тексту
можно привычными способами:

| Клавиши | Действие |
|---------|----------|
| `←` `→` `↑` `↓` | Перемещение по символам и строкам |
| `Home` / `End` | Начало / конец строки |
| `Page Up` / `Page Down` | Страница вверх / вниз |
| `Ctrl+_` (подчёркивание) | Перейти к строке по номеру |
| `Ctrl+C` | Показать текущую позицию (строка, столбец) |

> **Совет.** Комбинация `Ctrl+_` полезна, когда вы видите ошибку вроде
> «ошибка в строке 42» — можно быстро перейти к нужной строке по номеру.

### Редактирование текста

В nano вы просто печатаете текст — как в обычном редакторе. Никаких специальных
режимов нет. Курсор стоит в нужном месте — вводите текст, он появляется на экране.

| Клавиши | Действие |
|---------|----------|
| Любые символы | Ввод текста в позиции курсора |
| `Backspace` | Удалить символ перед курсором |
| `Delete` | Удалить символ под курсором |
| `Ctrl+K` | Вырезать (удалить) текущую строку целиком |
| `Ctrl+U` | Вставить последнюю вырезанную строку |
| `Alt+U` | Отменить последнее действие (undo) |
| `Alt+E` | Повторить отменённое действие (redo) |

Комбинация `Ctrl+K` / `Ctrl+U` работает как «вырезать / вставить» для целых строк.
Это удобно для перемещения строк: вырезаете строку в одном месте, перемещаете
курсор в другое и вставляете.

### Поиск и замена

Для поиска текста:

1. Нажмите `Ctrl+W` (Where is — «где находится»).
2. Введите искомый текст.
3. Нажмите `Enter` — курсор перейдёт к первому совпадению.
4. Чтобы найти следующее совпадение, снова нажмите `Ctrl+W` и `Enter`.

Для поиска и замены:

1. Нажмите `Ctrl+\`.
2. Введите текст, который хотите найти, и нажмите `Enter`.
3. Введите текст для замены и нажмите `Enter`.
4. nano спросит, заменять ли каждое совпадение: `Y` — да, `N` — нет, `A` — заменить все.

### Сохранение и выход

| Клавиши | Действие |
|---------|----------|
| `Ctrl+O` | Сохранить файл (nano спросит имя — нажмите `Enter` для подтверждения) |
| `Ctrl+X` | Выйти из nano |

Если вы изменили файл и нажали `Ctrl+X`, nano спросит:

```
Сохранить изменённый буфер?  Y Да  N Нет  ^C Отмена
```

- `Y` — сохранить и выйти;
- `N` — выйти без сохранения;
- `Ctrl+C` — отменить выход и продолжить редактирование.

На практике самая частая последовательность — **`Ctrl+X`, затем `Y`, затем `Enter`** —
сохранить файл и выйти.

### Полезные приёмы

#### Номера строк

По умолчанию nano не показывает номера строк. Можно включить их при запуске:

```bash
nano -l notes.txt
```

Или прямо во время работы — нажмите `Alt+N` для переключения отображения
номеров строк.

#### Открытие файла на определённой строке

Если вы знаете, что ошибка в строке 42:

```bash
nano +42 config.txt
```

nano откроет файл и сразу поставит курсор на строку 42.

#### Подсветка синтаксиса

nano умеет подсвечивать синтаксис для многих форматов: конфигурационных файлов,
скриптов, HTML и других. В Debian подсветка обычно включена по умолчанию.
Если нет — можно включить, добавив в файл `~/.nanorc` строку:

```
include "/usr/share/nano/*.nanorc"
```

(Как отредактировать этот файл? Конечно же, в nano: `nano ~/.nanorc`.)

#### Шпаргалка по nano

| Комбинация | Действие |
|------------|----------|
| `Ctrl+O` | Сохранить |
| `Ctrl+X` | Выйти |
| `Ctrl+K` | Вырезать строку |
| `Ctrl+U` | Вставить строку |
| `Ctrl+W` | Найти текст |
| `Ctrl+\` | Найти и заменить |
| `Ctrl+_` | Перейти к строке |
| `Ctrl+C` | Показать позицию курсора |
| `Alt+U` | Отменить действие |
| `Alt+N` | Вкл/выкл номера строк |

## Практика: редактируем конфигурацию проекта

Применим знания nano на нашем сквозном проекте. Откроем файл конфигурации nginx
и наполним его содержимым.

```bash
nano wordpress-project/config/nginx.conf
```

Введите следующий текст (просто печатайте — nano работает как обычный редактор):

```nginx
server {
    listen 80;
    server_name localhost;

    root /var/www/wordpress;
    index index.php index.html;

    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

Не беспокойтесь, если не понимаете значение каждой строки — мы подробно разберём
этот конфигурационный файл в главе 11. Сейчас важно попрактиковаться в работе
с редактором.

Сохраните файл: `Ctrl+X`, затем `Y`, затем `Enter`.

Проверим, что файл сохранился:

```bash
cat wordpress-project/config/nginx.conf
```

Теперь потренируемся в редактировании. Откройте файл снова:

```bash
nano wordpress-project/config/nginx.conf
```

Выполните действия:

1. Перейдите к строке 2 (`Ctrl+_`, введите `2`, `Enter`).
2. Замените `80` на `8080` — поставьте курсор на `8`, удалите `80` и напечатайте `8080`.
3. Найдите слово `localhost` (`Ctrl+W`, введите `localhost`, `Enter`).
4. Отмените последнее действие (`Alt+U`), убедитесь, что изменение откатилось.
5. Выйдите без сохранения (`Ctrl+X`, затем `N`).

## vim — обзор

### Зачем знать vim

vim (Vi Improved) — один из старейших и самых мощных текстовых редакторов в мире
Unix/Linux. Он установлен практически на любой Linux-системе — даже на минимальных
серверах, где нет nano.

Знать vim полезно по нескольким причинам:

- **vim может оказаться единственным редактором** на сервере, к которому вы
  подключились;
- **некоторые программы открывают vim автоматически** — например, `git commit`
  (глава 13) или `crontab -e` по умолчанию запускают vim для редактирования;
- если vim открылся неожиданно и вы не знаете, как из него выйти, это может
  вызвать панику (это настолько частая ситуация, что стала мемом в IT).

Наша цель — не изучить vim в совершенстве, а научиться **открыть файл,
внести правку, сохранить и выйти**. Этого достаточно для начала.

### Два режима vim

Главное отличие vim от nano: **в vim нельзя просто начать печатать**. У vim есть
несколько режимов работы, но для начала важны два:

- **нормальный режим** (Normal mode) — режим по умолчанию. Клавиши выполняют
  команды, а не вводят текст. Например, `dd` удаляет строку, `x` удаляет символ;
- **режим вставки** (Insert mode) — режим ввода текста, похожий на обычный
  редактор.

```
    ┌────────────────────┐
    │  Нормальный режим  │ ← vim запускается в этом режиме
    │  (команды)         │
    └────────┬───────────┘
             │ i — войти
             ▼
    ┌────────────────────┐
    │  Режим вставки     │
    │  (ввод текста)     │
    └────────┬───────────┘
             │ Esc — вернуться
             ▼
    ┌────────────────────┐
    │  Нормальный режим  │
    └────────────────────┘
```

**Правило:** если что-то пошло не так — **нажмите `Esc`**. Вы всегда вернётесь
в нормальный режим, откуда можно сохранить файл или выйти.

### Минимальный набор действий

#### Открыть файл

```bash
vim notes.txt
```

vim откроется в **нормальном режиме**. Символ `~` (тильда) в начале строк означает,
что эти строки не существуют — файл короче, чем экран.

#### Войти в режим вставки

Нажмите `i` (insert). Внизу экрана появится надпись `-- INSERT --` (или
`-- ВСТАВКА --`). Теперь можно печатать текст как обычно.

#### Вернуться в нормальный режим

Нажмите `Esc`. Надпись `-- INSERT --` исчезнет.

#### Сохранить и выйти

Из **нормального режима** (нажмите `Esc`, если не уверены) введите команду.
Команды в vim начинаются с двоеточия:

| Команда | Действие |
|---------|----------|
| `:w` | Сохранить файл (write) |
| `:q` | Выйти (quit) |
| `:wq` | Сохранить и выйти |
| `:q!` | Выйти без сохранения (принудительно) |

При вводе команды двоеточие и текст появляются в нижней строке экрана. После
ввода команды нажмите `Enter`.

> **Не можете выйти из vim?** Нажмите `Esc`, затем введите `:q!` и нажмите
> `Enter`. Это закроет vim без сохранения — самый надёжный способ выбраться.

### Шпаргалка по vim

| Действие | Клавиша / команда |
|----------|-------------------|
| Войти в режим вставки | `i` |
| Вернуться в нормальный режим | `Esc` |
| Сохранить | `:w` + `Enter` |
| Выйти | `:q` + `Enter` |
| Сохранить и выйти | `:wq` + `Enter` |
| Выйти без сохранения | `:q!` + `Enter` |
| Удалить символ (нормальный режим) | `x` |
| Удалить строку (нормальный режим) | `dd` |
| Отмена действия | `u` |

### Как сменить редактор по умолчанию

Если программы (вроде `git` или `crontab`) открывают vim, а вам удобнее nano,
можно изменить редактор по умолчанию:

```bash
export EDITOR=nano
```

Чтобы это сохранилось между сессиями, добавьте строку в файл `~/.bashrc`:

```bash
echo 'export EDITOR=nano' >> ~/.bashrc
```

> **Что здесь происходит?** Символ `>>` — это **перенаправление вывода** с добавлением
> в конец файла. Команда `echo` выводит текст, а `>>` записывает его в файл
> `~/.bashrc`, не затирая то, что там было. Подробно перенаправление ввода-вывода
> мы разберём в следующей главе (05).

О переменных окружения и файле `~/.bashrc` мы подробно поговорим в главе 06.

## Практическое задание

1. Откройте в nano файл `~/wordpress-project/config/php.ini`. Введите
   следующие строки:
   ```ini
   upload_max_filesize = 64M
   post_max_size = 64M
   max_execution_time = 300
   memory_limit = 256M
   ```
   Сохраните файл и выйдите.

2. Откройте этот же файл снова. С помощью поиска (`Ctrl+W`) найдите
   слово `256M`. Замените его на `512M`. Сохраните и выйдите.

3. Откройте в nano новый файл `~/wordpress-project/site/index.html`.
   Введите:
   ```html
   <!DOCTYPE html>
   <html>
   <head>
       <title>WordPress скоро будет здесь</title>
   </head>
   <body>
       <h1>Сайт в процессе настройки</h1>
   </body>
   </html>
   ```
   Сохраните файл. Проверьте содержимое командой `cat`.

4. Откройте любой файл в vim (`vim ~/wordpress-project/site/index.html`).
   Попрактикуйтесь:
   - войдите в режим вставки (`i`);
   - добавьте пустую строку;
   - вернитесь в нормальный режим (`Esc`);
   - выйдите без сохранения (`:q!`).

5. Снова откройте файл в vim. Войдите в режим вставки, добавьте
   комментарий `<!-- Настроено вручную -->` перед `</body>`.
   Сохраните и выйдите (`:wq`). Проверьте результат командой `cat`.

6. Переведите nano в режим отображения номеров строк. Откройте файл
   `nginx.conf` и перейдите к строке 5 с помощью `Ctrl+_`. На какой
   строке находится директива `index`?
