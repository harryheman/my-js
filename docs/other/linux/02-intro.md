# Этап 1. Первое знакомство с Linux

## Что такое Linux

Когда говорят «Linux», обычно имеют в виду операционную систему — программу, которая
управляет компьютером: распределяет память, работает с дисками, обеспечивает запуск
других программ. Если точнее, Linux — это **ядро** операционной системы. Само по себе
ядро бесполезно для пользователя, поэтому вокруг него собирают набор программ, утилит
и настроек. Такой готовый набор называется **дистрибутивом**.

Дистрибутивов существуют десятки: Ubuntu, Fedora, Arch, CentOS и другие. В этом курсе
мы используем **Debian 13 (trixie)**. Debian — один из старейших дистрибутивов, известный
своей стабильностью. Именно на его основе построен Ubuntu — самый популярный дистрибутив
для настольных компьютеров. На серверах Debian используется повсеместно.

Linux работает не только на серверах. Ядро Linux лежит в основе Android, его используют
на суперкомпьютерах, в маршрутизаторах и умных устройствах. Но для нас главное — это
серверы. Подавляющее большинство веб-сайтов, баз данных и облачных сервисов работают
именно на Linux.

## Подключение по SSH

В этом курсе мы работаем с Linux через командную строку — без графического интерфейса.
Это именно тот режим, в котором работают системные администраторы и DevOps-инженеры.

Для подключения к серверу используется **SSH** (Secure Shell) — протокол безопасного
удалённого доступа. Данные, включая пароль, передаются в зашифрованном виде.

### Подключаемся из Windows

В Windows 10 и 11 SSH-клиент уже встроен. Откройте **Windows Terminal** (или PowerShell)
и введите команду:

```
ssh student@192.168.1.100
```

Здесь `student` — имя пользователя на сервере, а `192.168.1.100` — IP-адрес сервера.
Оба значения вам сообщит преподаватель.

> **Адрес и имя пользователя в примерах.** Во всех примерах этого курса мы используем
> `student@192.168.1.100`. В вашем случае адрес и имя пользователя могут отличаться —
> подставляйте свои.

### Первое подключение

При первом подключении к серверу SSH спросит, доверяете ли вы этому хосту:

```
The authenticity of host '192.168.1.100' can't be established.
ED25519 key fingerprint is SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxx.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Введите `yes` и нажмите Enter. Это нормальное поведение — SSH запоминает сервер, и при
следующих подключениях этот вопрос больше не появится.

После этого введите пароль. **При вводе пароля символы не отображаются** — ни звёздочки,
ни точки. Это не ошибка, просто вводите пароль и нажимайте Enter.

### Отключение

Чтобы завершить сеанс и отключиться от сервера, введите:

```bash
exit
```

Или нажмите `Ctrl+D`.

## Командная строка: первый взгляд

После успешного подключения вы увидите **приглашение командной строки** (prompt) —
примерно такое:

```
student@debian:~$
```

Разберём его по частям:

| Часть | Значение |
|-------|----------|
| `student` | имя пользователя, под которым вы вошли в систему |
| `@` | разделитель |
| `debian` | имя компьютера (хоста) |
| `:` | разделитель |
| `~` | текущий каталог (`~` означает «домашний каталог пользователя») |
| `$` | признак обычного пользователя |

Символ `$` в конце приглашения означает, что вы работаете как обычный пользователь.
У администратора (root) вместо `$` стоит `#`. Пока запомните: **видите `$` — всё
нормально**. Работа от имени root рассматривается в главе 08.

## Первые команды

Пришло время попробовать. Вводите команды после символа `$` и нажимайте Enter.

### whoami — кто я

```bash
whoami
```

```
student
```

Команда выводит имя текущего пользователя. Полезно, когда вы не уверены, под какой
учётной записью работаете.

### hostname — имя компьютера

```bash
hostname
```

```
debian
```

Показывает сетевое имя машины. На сервере это может быть что-то вроде `web-server-01`
или `db-prod-03`.

### pwd — где я нахожусь

```bash
pwd
```

```
/home/student
```

**pwd** расшифровывается как *print working directory* — «показать рабочий каталог».
В Linux нет дисков `C:` и `D:`, как в Windows. Вся файловая система начинается с
корня `/`, а каталоги разделяются символом `/` (прямой слеш, а не обратный `\`, как
в Windows).

`/home/student` — это ваш **домашний каталог**, личное пространство. Именно сюда вы
попадаете при входе в систему.

### ls — что здесь лежит

```bash
ls
```

Команда выводит содержимое текущего каталога. Если каталог пуст, команда не выведет
ничего — это нормально. В свежей системе домашний каталог обычно пустой.

Попробуйте посмотреть, что лежит в корне файловой системы:

```bash
ls /
```

```
bin  boot  dev  etc  home  lib  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var
```

Здесь вы видите основные системные каталоги. Подробно мы разберём их в следующей главе.

### cd — перейти в другой каталог

```bash
cd /tmp
```

Команда **cd** (*change directory*) меняет текущий каталог. После выполнения обратите
внимание — приглашение командной строки изменилось:

```
student@debian:/tmp$
```

Вместо `~` теперь показывается `/tmp`. Убедитесь командой `pwd`:

```bash
pwd
```

```
/tmp
```

Чтобы вернуться в домашний каталог, введите `cd` без аргументов:

```bash
cd
```

```
student@debian:~$
```

### clear — очистить экран

```bash
clear
```

Экран очищается. Команды и их вывод никуда не пропадают — просто экран прокручивается
вверх. Можно прокрутить обратно, чтобы увидеть историю.

### exit — выйти

```bash
exit
```

Завершает текущий сеанс. Если вы подключены по SSH — соединение закроется.

## Как получить помощь

Запомнить все команды и их параметры невозможно — и не нужно. Важнее уметь быстро
найти нужную информацию.

### man — встроенное руководство

В Linux есть встроенная система справки — **man** (от слова *manual*). Чтобы прочитать
руководство по любой команде, введите:

```bash
man ls
```

Откроется страница с подробным описанием команды `ls`: что она делает, какие параметры
принимает, примеры использования.

**Навигация по man-странице:**

| Клавиша | Действие |
|---------|----------|
| `Пробел` или `Page Down` | следующая страница |
| `b` или `Page Up` | предыдущая страница |
| стрелки `вверх` / `вниз` | прокрутка по строкам |
| `/текст` + Enter | поиск текста на странице |
| `n` | следующее совпадение поиска |
| `q` | выход из man |

Самое важное: **`q` — выход**. Новички часто пугаются, попав в man, потому что не
знают, как выйти. Просто нажмите `q`.

### --help — краткая справка

Большинство команд поддерживают флаг `--help`, который выводит краткую справку прямо
в терминал:

```bash
ls --help
```

Это удобнее, чем man, когда нужно быстро вспомнить нужный параметр.

> **Совет.** Не пытайтесь запомнить всё сразу. Даже опытные администраторы регулярно
> заглядывают в man-страницы и ищут информацию в интернете. Главный навык — умение
> находить ответ, а не держать всё в голове.

## Практическое задание

1. Подключитесь к серверу по SSH из Windows Terminal.
2. Выполните команды `whoami`, `hostname`, `pwd` и запишите результат.
3. Выполните `ls /` и посмотрите, какие каталоги есть в корне системы.
4. Перейдите в каталог `/var` командой `cd /var`, убедитесь командой `pwd`.
5. Посмотрите содержимое `/var` командой `ls`.
6. Вернитесь в домашний каталог (`cd` без аргументов), проверьте с помощью `pwd`.
7. Откройте man-страницу для команды `hostname` (`man hostname`), найдите описание, выйдите клавишей `q`.
8. Отключитесь от сервера командой `exit`.
